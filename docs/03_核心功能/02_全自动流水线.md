## 一、流水线全景


## 二、并发防重机制

```python
# 1. 非阻塞获取锁，失败立即丢弃冗余请求
# -> 2. 检查 is_running 内存标记（双重防护）
# -> 3. finally 块无条件释放锁（系统自愈）
if not _scan_entry_lock.acquire(blocking=False):
    return
```

## 三、STEP 1：物理扫描

```
1. 加载路径白名单 + inode 指纹集合
-> 2. 先扫 library（建立 inode 指纹库）
-> 3. 再扫 download（inode 熔断过滤硬链接做种文件）
-> 4. 路径去重 → 文件名去重 → 路径霸权类型修正
-> 5. 新文件入库（library→archived，download→pending）
-> 6. library 文件立刻流转至 media_archive 冷表
```

**路径霸权**：路径明确配置为 `movie`/`tv` 时强制覆盖正则猜测。

**失忆救援**：从父目录名 `Title (Year)` 提取元数据，自动绑定本地海报，跳过刮削流程。

## 四、STEP 2：全量刮削（v1.0.1-Enhanced）

**完整业务链路**：

```
pending/failed 任务
  └─► 获取防重锁（非阻塞，blocking=False）
        └─► 校验 TMDB API Key
              └─► 获取待刮削任务（双表联查：tasks + media_archive）
                    └─► 逐个处理任务
                          ├─► NFO 短路拦截（本地 NFO 存在）
                          │     └─► 1. 查找本地 NFO 文件 -> 
                          │         2. 解析 NFO 获取 TMDB ID / IMDb ID -> 
                          │         3. 若成功则跳过 AI 提炼和 TMDB 搜索 -> 
                          │         4. 直接更新 DB 并归档 -> 
                          │         5. 若失败则降级走正常刮削流程
                          │
                          ├─► AI 提炼逻辑（NFO 不存在）
                          │     └─► 1. 正则清洗文件名 -> 
                          │         2. 调用 AI Agent 分析 -> 
                          │         3. 提取查询词、年份、类型 -> 
                          │         4. 路径权威优先（不被 AI 覆盖）-> 
                          │         5. 年份物理验证（防止 AI 推断错误年份）
                          │
                          ├─► TMDB 搜索与防重拦截
                          │     └─► 1. 根据类型调用 TMDB 搜索 -> 
                          │         2. 若剧集搜索失败则二次搜索（去集号）-> 
                          │         3. 精确匹配或宽松匹配候选结果 -> 
                          │         4. 提取 TMDB ID、标题、年份
                          │
                          ├─► IMDb 金标准防重检测
                          │     └─► 1. 获得 IMDb ID -> 
                          │         2. 检查 check_media_exists() -> 
                          │         3. 若重复则标记 ignored + 继承同源海报 -> 
                          │         4. 跳过物理归档
                          │
                          ├─► 就地补录（文件在 library 路径）
                          │     └─► 1. 检查文件是否来自 library 路径 -> 
                          │         2. 若是则进入就地补录模式 -> 
                          │         3. 计算元数据目录（剧集需上升一级）-> 
                          │         4. 写入 NFO 和海报 -> 
                          │         5. 更新 DB 元数据（不移动文件）
                          │
                          ├─► 归档全链路（文件在下载目录）
                          │     └─► 1. 获取媒体库根路径 -> 
                          │         2. 构建目标目录结构 -> 
                          │         3. SmartLink 创建链接（硬链接 > 软链接 > 复制）-> 
                          │         4. 生成 NFO 文件 -> 
                          │         5. 下载海报和 Fanart -> 
                          │         6. 更新数据库状态
                          │
                          └─► 字幕白嫖（本地字幕检测）
                                └─► 1. 获取字幕检测路径 -> 
                                    2. 调用本地字幕检测函数 -> 
                                    3. 若发现本地字幕则标记为 success -> 
                                    4. 否则标记为 pending（等待搜索）
```

**关键设计**：
- **NFO 短路拦截**：本地 NFO 存在时直接解析，跳过 AI 和 TMDB 搜索，节省 Token
- **AI 提炼**：调用 `AIAgent.ai_identify_media()` 进行文件名语义分析
- **防重拦截**：`check_media_exists()` 精确到季集号，防止同剧不同集被误杀
- **就地补录**：文件已在媒体库，仅补充元数据，不移动文件
- **归档全链路**：文件在下载目录，执行 SmartLink、重命名、写入元数据
- **字幕白嫖**：归档完成后立即检测本地字幕，若有则标记 `success`，零 API 消耗

**季号补充**：归档时从路径 `parts` 提取真实季号，回写数据库。

**孤儿救援**：刮削完成后扫描仍处于 `pending` 状态超时的任务，标记为 `failed`。

## 五、STEP 3：字幕查找

```
arcived 任务（sub_status=pending/missing/failed）
  └─► 本地字幕检测（.srt/.ass/.vtt → 跳过，零 API 消耗）
        └─► OpenSubtitles API 搜索
              └─► 语言评分排序（zh-CN=100, zh-TW=50, en=30）
                    └─► 下载 + 重命名为 {stem}.ai.zh-CN.srt
                          └─► db.update_archive_sub_status('scraped')
```

**频率限制**：
```
1. 获取待字幕任务 -> 2. 本地白嫖（已有字幕直接跳过）-> 3. 调用 API
-> 4. AI 命名规范重命名 -> 5. 写回数据库 -> 6. 休眠 5 秒防封号
```

## 七、核级重构（Nuclear Rebuild）— v1.0.1-Enhanced

**触发场景**：用户在前台点击「执行核级重构」按钮，对已归档任务进行元数据补全或修正。

**完整业务链路**：

```
RebuildDialog.tsx（前端弹窗）
  └─► 1. 验证 tmdb_id 非空 -> 
      2. 构建请求参数（task_id, is_archive, nuclear_reset=true）-> 
      3. 调用 api.rebuildTask()
        └─► POST /tasks/manual_rebuild
              └─► manual_rebuild() 后端处理
                    ├─► 1. 读取任务记录（根据 is_archive 选择热表/冷表）-> 
                    ├─► 2. 校验 target_path 有效性 -> 
                    ├─► 3. 初始化 TMDB 适配器 -> 
                    ├─► 4. 金标准防重预检（检查本地 NFO 中的 IMDb ID）-> 
                    ├─► 5. 核级清理（_nuclear_clean_directory）-> 
                    ├─► 6. 视频文件重命名 -> 
                    ├─► 7. 文件夹土木工程 -> 
                    ├─► 8. 调用 update_any_task_metadata()（更新元数据）-> 
                    ├─► 9. 调用 update_task_status()（触发归档流程）-> 
                    ├─► 10. NFO 生成与写入 -> 
                    ├─► 11. 海报下载与存储 -> 
                    ├─► 12. 字幕检测（本地白嫖）-> 
                    └─► 返回重建结果 {success, rebuilt: {nfo, poster, subtitle, nuclear}}
                          └─► 前端显示成功提示 + 刷新列表
```

**开门重置器（防止幽灵记忆）**：

```typescript
// RebuildDialog.tsx: useEffect
// 业务链路：1. 监听 open 状态与 task.id 变化 -> 
//          2. 触发弹窗展示 -> 
//          3. 彻底清空上一轮的幽灵记忆 (keyword, results, selected) -> 
//          4. 重新灌入当前 task 的基础参数

useEffect(() => {
  if (open) {
    setMediaType(task.media_type || 'movie');
    setKeyword(task.title || '');
    setResults([]);
    setSelected(null);
    setSearchErr(null);
    setSeason(task.season ?? 1);
    setEpisode(task.episode ?? '');
  }
}, [open, task.id]);
```

**防止 Dialog 幽灵记忆陷阱**：`if (!open) return null` 只隐藏不卸载，导致状态被永久冻结。此 useEffect 确保每次打开新任务时，状态都被重置。

**核级清理（_nuclear_clean_directory）**：
- 删除旧的 NFO 文件
- 删除旧的海报文件
- 删除旧的 Fanart 文件
- 确保新元数据写入时无冲突
