## 一、流水线全景

```
[触发] 手动按钮 / AI 指令 / 定时巡逻
   ├─► STEP 1: 物理扫描 → 去重入库
   ├─► STEP 2: 全量刮削 → NFO/海报/归档
   └─► STEP 3: 字幕查找 → 本地检测/API下载
```

## 二、并发防重机制

```python
# 1. 非阻塞获取锁，失败立即丢弃冗余请求
# -> 2. 检查 is_running 内存标记（双重防护）
# -> 3. finally 块无条件释放锁（系统自愈）
if not _scan_entry_lock.acquire(blocking=False):
    return
```

## 三、STEP 1：物理扫描

```
1. 加载路径白名单 + inode 指纹集合
-> 2. 先扫 library（建立 inode 指纹库）
-> 3. 再扫 download（inode 熔断过滤硬链接做种文件）
-> 4. 路径去重 → 文件名去重 → 路径霸权类型修正
-> 5. 新文件入库（library→archived，download→pending）
-> 6. library 文件立刻流转至 media_archive 冷表
```

**路径霸权**：路径明确配置为 `movie`/`tv` 时强制覆盖正则猜测。

**失忆救援**：从父目录名 `Title (Year)` 提取元数据，自动绑定本地海报，跳过刮削流程。

## 四、STEP 2：全量刮削（V2.0 流水线编排器架构）

> **[v1.0.0 重构]** 单体上帝函数已原子化拆解为 **4 个独立步骤函数 + 1 个纯粹编排器**。每个步骤独立容错，单步异常不崩溃全链路。全局网络层由 `http_utils.py` 提供统一的重试与 Timeout 防护。

### 6 段式执行逻辑（V2.0）

```
pending/archived(缺imdb_id) 任务
  └─► 编排器 _process_single_task()
        │
        ├─► [Step 1] _step_nfo_shortcut()          NFO 短路拦截
        │     业务链路：查找本地 NFO → 解析 TMDB/IMDb ID → 直接归档
        │     容错：解析失败自动降级，绝对不调用 AI 和 TMDB
        │     短路成功 → return True, False  （跳过后续所有步骤）
        │
        ├─► [极致省流] 存量字幕检测
        │     业务链路：archived 且缺 imdb_id 的任务，先检测本地字幕
        │     若已有字幕 → 跳过 AI 刮削，零 Token 消耗
        │     → return True, False
        │
        ├─► [Step 2] _step_ai_extraction()         AI 提炼（含原子识别协议）
        │     业务链路：
        │       0. 🛡️ 保护性前置提取（v1.0.0-Atomic）：在任何正则删除之前，
        │           先对 raw_filename 调用 extract_season_episode()，
        │           将结果缓存为 locked_season / locked_episode，
        │           写回 task dict，后续所有步骤不可覆盖。
        │       1. MediaCleaner 正则物理去噪（[EXTRACT] 规则隔离，季集号不被删除）
        │       2. AIAgent.ai_identify_media(
        │              cleaned_name, full_path,
        │              locked_season=locked_season,
        │              locked_episode=locked_episode
        │          ) 语义识别
        │       3. 🔒 季集锁定护盾：AI 返回后，若正则已锁定 S/E，
        │           强制覆盖 AI 输出，AI 无权修改或丢弃集号。
        │       4. AI 失败 → 降级使用正则清洗名（零中断，locked S/E 仍保留）
        │       5. 路径权威优先：路径已定 type 不被 AI 覆盖
        │       6. YEAR_MIRROR 镜像校验：AI 提交 filename_year（物理提取）
        │           和 knowledge_year（知识库权威）两份证据；
        │           Python 后端执行比对裁决：
        │           - movie 两证据一致 → 使用该年份精确过滤
        │           - movie 两证据冲突 → 置空，执行模糊搜索
        │           - tv → 强制使用 knowledge_year（第一季首播年）
        │     返回：(refined_query, refined_year, refined_type)
        │
        ├─► [Step 3] _step_tmdb_search_and_dup_check()  搜索与防重
        │     业务链路：
        │       1. 按 refined_type 调用 TMDB 搜索
        │       2. 剧集匹配失败 → 三梯队降级搜索策略
        │          第一梯队（精确）: Title + Year
        │          第二梯队（容错）: Title完整片名 + 无Year（隔离AI幻觉年份）
        │          第三梯队（模糊）: 截断首词 + 无Year（最后手段）
        │       3. 精确匹配（original_name/name == query）优先
        │       4. 宽松匹配（去集号后匹配）兜底
        │       5. 获取 IMDb ID（get_external_ids）
        │       6. 🛡️ IMDb+S+E 三元组防重熔断（v1.0.0-Atomic）：
        │             type='tv' 且 episode=None → 跳过防重，进入待定状态
        │             check_media_exists(imdb_id, type, season, episode) → True
        │             → mark_task_as_ignored_and_inherit()  🚨 原子操作
        │             → return None  （主流程 return False, True）
        │     返回：tmdb_data dict | None
        │
        ├─► [Step 4] _step_archive_and_metadata()  物理归档与元数据
        │     业务链路 A（就地补录 — 文件已在 library）：
        │       1. 检测文件是否在 library 路径或 status=="archived"
        │       2. 计算元数据目录（剧集 Season 文件夹上升一级）
        │       3. 路径防重拦截（check_task_exists_by_path）
        │       4. 写入 NFO（generate_nfo，NFO已存在→直接复用，失败不中断）
        │       5. 下载海报（download_poster，海报已存在→直接复用，失败不中断）
        │       6. 下载 Fanart（download_fanart，Fanart已存在→直接复用，失败不中断）
        │       7. 更新 DB 元数据（不移动文件）
        │     业务链路 B（归档全链路 — 文件在下载目录）：
        │       1. 路径防穿越安全校验（Path.resolve().relative_to()）
        │       2. SmartLink.create_link()（硬链接 > 软链接 > 复制）
        │       3. 写入 NFO + 下载海报 + 下载 Fanart（各步独立容错，存在性短路）
        │       4. 更新 DB 状态为 archived
        │     返回：target_path str | None
        │
        └─► [字幕白嫖] _check_local_subtitles()
              业务链路：检测 target_path 同级字幕文件
              发现字幕 → update_task_sub_status("scraped")  零 API 消耗
              未发现   → update_task_sub_status("pending")
              返回：True, False
```

### 网络层防护（http_utils.py）

所有外部 HTTP 调用（TMDB API / 海报下载 / OpenSubtitles）均受 `http_utils.py` 全局防护：

| 防护机制 | 配置 | 说明 |
|----------|------|------|
| **连接超时** | `connect=10s` | 防止 DNS 解析或 TCP 握手永久挂起 |
| **读取超时** | `read=30s` | 防止服务器响应超时导致线程僵死 |
| **自动重试** | 最多 3 次，指数退避 | 网络抖动自愈，不暴露给业务层 |
| **429 限流** | 等待 `Retry-After` | TMDB / OpenSubtitles 限流自动冷却 |

```python

client = get_http_client()   # 已预配置 timeout + retry
```

### 步骤独立容错矩阵

| 步骤 | 失败行为 | 主流程影响 |
|------|----------|------------|
| `_step_nfo_shortcut` | 解析异常 → `return False`（降级）| 继续执行 Step 2 |
| `_step_ai_extraction` | AI 异常 → 降级为正则清洗名 | 零中断，继续 Step 3 |
| `_step_tmdb_search_and_dup_check` | 无结果 → `db.update_task_status(failed)` → `return None` | `return False, True` |
| `_step_archive_and_metadata` NFO | NFO 写入异常 → `logger.warning` 跳过 | 海报/Fanart 继续 |
| `_step_archive_and_metadata` 海报 | 下载异常 → `logger.warning` 跳过 | DB 写入不受影响 |
| `_step_archive_and_metadata` SmartLink | 链接失败 → `target_path=None` | `return False, True` |

**完整业务链路**：

```
pending/failed 任务
  └─► 获取防重锁（非阻塞，blocking=False）
        └─► 校验 TMDB API Key
              └─► 获取待刮削任务（双表联查：tasks + media_archive）
                    └─► 逐个处理任务
                          ├─► NFO 短路拦截（本地 NFO 存在）
                          │     └─► 1. 查找本地 NFO 文件 -> 
                          │         2. 解析 NFO 获取 TMDB ID / IMDb ID -> 
                          │         3. 若成功则跳过 AI 提炼和 TMDB 搜索 -> 
                          │         4. 直接更新 DB 并归档 -> 
                          │         5. 若失败则降级走正常刮削流程
                          │
                          ├─► AI 提炼逻辑（NFO 不存在）
                          │     └─► 1. 正则清洗文件名 -> 
                          │         2. 调用 AI Agent 分析 -> 
                          │         3. 提取查询词、类型（含 filename_year/knowledge_year 年份证据）-> 
                          │         4. 路径权威优先（不被 AI 覆盖）-> 
                          │         5. YEAR_MIRROR 镜像校验（Python 后端比对两份年份证据裁决 final_year）
                          │
                          ├─► TMDB 搜索与防重拦截
                          │     └─► 1. 根据类型调用 TMDB 搜索 -> 
                          │         2. 若剧集搜索失败则二次搜索（去集号）-> 
                          │         3. 精确匹配或宽松匹配候选结果 -> 
                          │         4. 提取 TMDB ID、标题、年份
                          │
                          ├─► IMDb 金标准防重检测
                          │     └─► 1. 获得 IMDb ID -> 
                          │         2. 检查 check_media_exists() -> 
                          │         3. 若重复则标记 ignored + 继承同源海报 -> 
                          │         4. 跳过物理归档
                          │
                          ├─► 就地补录（文件在 library 路径）
                          │     └─► 1. 检查文件是否来自 library 路径 -> 
                          │         2. 若是则进入就地补录模式 -> 
                          │         3. 计算元数据目录（剧集需上升一级）-> 
                          │         4. 写入 NFO 和海报 -> 
                          │         5. 更新 DB 元数据（不移动文件）
                          │
                          ├─► 归档全链路（文件在下载目录）
                          │     └─► 1. 获取媒体库根路径 -> 
                          │         2. 构建目标目录结构 -> 
                          │         3. SmartLink 创建链接（硬链接 > 软链接 > 复制）-> 
                          │         4. 生成 NFO 文件 -> 
                          │         5. 下载海报和 Fanart -> 
                          │         6. 更新数据库状态
                          │
                          └─► 字幕白嫖（本地字幕检测）
                                └─► 1. 获取字幕检测路径 -> 
                                    2. 调用本地字幕检测函数 -> 
                                    3. 若发现本地字幕则标记为 scraped -> 
                                    4. 否则标记为 pending（等待搜索）
```

**关键设计**：
- **NFO 短路拦截**：本地 NFO 存在时直接解析，跳过 AI 和 TMDB 搜索，节省 Token
- **🛡️ 原子识别协议（v1.0.0-Atomic）**：保护性前置提取 S/E → `[EXTRACT]` 规则隔离（季集号不被删除）→ 季集锁定护盾（AI 输出被正则锁定值强制覆盖）
- **AI 提炼**：调用 `AIAgent.ai_identify_media(locked_season, locked_episode)` 进行文件名语义分析，失败自动降级（集号仍保留）
- **三元组防重**：`check_media_exists(imdb_id, type, season, episode)` 精确到季集号；`type='tv' 且 episode=None` 时跳过检测防止误拦截；`mark_task_as_ignored_and_inherit()` 原子操作继承海报
- **就地补录**：文件已在媒体库，仅补充元数据，不移动文件，剧集元数据目录自动上升一级
- **归档全链路**：文件在下载目录，执行 SmartLink、写入元数据（各步独立容错）
- **字幕白嫖**：归档完成后立即检测本地字幕，若有则标记 `scraped`，零 API 消耗
- **网络防护**：`http_utils.py` 为所有外部 HTTP 调用提供统一 Timeout + 自动重试，业务层透明

**季号补充**：归档时从路径 `parts` 提取真实季号，回写数据库。

**孤儿救援**：刮削完成后扫描仍处于 `pending` 状态超时的任务，标记为 `failed`。

### 补录逻辑“物理感知护盾”（残缺状态自动补领）

> **[v1.0.0]** `manual_rebuild()` 在进入核级清理前，会对元数据目录执行**物理存在性二次校验**，以避免“逻辑正确但物理残缺”的假阳性短路。

**判定模型**：
- **有 NFO**：`tvshow.nfo` / `movie.nfo` 存在
- **有海报**：`poster.jpg/png/webp` 任意存在
- **ID 一致**：NFO 内 `imdbid`（或 `<uniqueid type="imdb">`）与当前重建目标 `new_imdb_id` 完全一致

**护盾规则**：
- 若同时满足 **ID 一致 + NFO 存在 + 海报存在**，则允许开启 `skip_metadata` 护盾，跳过后续 NFO/海报补录（避免重复 I/O 和网络）。
- 若满足 **ID 一致 + NFO 存在**，但 **海报物理缺失**，则**强制解除护盾**，即使金标准校验通过也要触发 `refix_poster` 补领，修复“有 NFO 但无海报”的残缺状态。

**开门重置器（防止幽灵记忆）**：

```typescript
// RebuildDialog.tsx: useEffect
// 业务链路：1. 监听 open 状态与 task.id 变化 -> 
//          2. 触发弹窗展示 -> 
//          3. 彻底清空上一轮的幽灵记忆 (keyword, results, selected) -> 
//          4. 重新灌入当前 task 的基础参数

useEffect(() => {
  if (open) {
    setMediaType(task.media_type || 'movie');
    setKeyword(task.title || '');
    setResults([]);
    setSelected(null);
    setSearchErr(null);
    setSeason(task.season ?? 1);
    setEpisode(task.episode ?? '');
  }
}, [open, task.id]);
```

**防止 Dialog 幽灵记忆陷阱**：`if (!open) return null` 只隐藏不卸载，导致状态被永久冻结。此 useEffect 确保每次打开新任务时，状态都被重置。

**与自动流水线并列的手动链路**：`POST /tasks/manual_rebuild` 由 `rebuilder` 包执行——`nuclear_reset=false` 走 **AssetPatchEngine（补录）**；`nuclear_reset=true` 走 **NuclearEngine（全量重建）**，与刮削编排器 `_process_single_task` 解耦。详见 [API 规范与鉴权](../02_数据契约/02_API规范与鉴权.md) 第四节「手动补录与全量重建」。
