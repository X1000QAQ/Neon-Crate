## 二、物理 inode 熔断

**场景**：qBittorrent 下载到 `/downloads`，硬链接归档到 `/media`，两路径共享同一 inode。

**业务链路**：
```
1. 先扫 library，收集 (inode, size) 指纹集合 -> 
2. 再扫 download，对比 inode 指纹 -> 
3. 匹配 → 硬链接做种文件，跳过入库
```

**实现**：
```python
# scan_engine.py
known_inodes_set = set()  # {(st_ino, st_size), ...}

# 第一步：扫 library，收集指纹
for file in scan_directory(library_path):
    stat = os.stat(file)
    known_inodes_set.add((stat.st_ino, stat.st_size))

# 第二步：扫 download，过滤硬链接
for file in scan_directory(download_path):
    stat = os.stat(file)
    if (stat.st_ino, stat.st_size) in known_inodes_set:
        logger.info(f"[INODE] 硬链接做种文件，跳过入库: {file}")
        continue
    # 新文件，入库
```

## 三、路径白名单拦截（双重哈希过滤）

**业务链路**：
```
1. 第一重：路径白名单（已知任务的 path + target_path）
2. 第二重：物理 inode 指纹（防止路径映射偏移）
```

**实现**：
```python
# 第一重：路径白名单
known_paths_set = {str(Path(p).resolve())
    for task in all_tasks
    for p in [task.get("path"), task.get("target_path")] if p}

# 第二重：物理 inode 指纹
known_inodes_set = set()  # {(st_ino, st_size), ...}

# 扫描时过滤
for file in scan_directory(download_path):
    file_norm = str(Path(file).resolve())
    if file_norm in known_paths_set:
        logger.info(f"[PATH] 路径已知，跳过入库: {file}")
        continue
    
    stat = os.stat(file)
    if (stat.st_ino, stat.st_size) in known_inodes_set:
        logger.info(f"[INODE] 硬链接做种文件，跳过入库: {file}")
        continue
    
    # 新文件，入库
```

## 四、原子操作：mark_task_as_ignored_and_inherit（v1.0.0）

```
🚨 架构级原子操作 [DO NOT SPLIT OR INLINE]

触发场景：文件被 IMDb 重复检测判定为物理副本（PT 做种重复文件）

业务链路：
1. 跨表穿透检索：冷表优先从 media_archive 检索同源海报路径 -> 
2. 热表兜底：冷表无结果时从 tasks 热表查找同 imdb_id 的海报路径 -> 
3. 状态原子写入：一次性写入 ignored + local_poster_path + imdb_id + tmdb_id -> 
4. 视觉防护：确保前端获得 local_poster_path，触发 VHS TAPE ERROR 特效
```

**为什么不可拆分**：
- 若 `local_poster_path` 为空，ignored 状态的视觉标识不会触发
- 前端出现白板破图
- 四步必须永远绑定在一起

**实现**（task_repo.py）：
```python
def mark_task_as_ignored_and_inherit(
    self,
    task_id: int,
    is_archive: bool,
    imdb_id: Optional[str] = None,
    tmdb_id: Optional[str] = None
) -> bool:
    """
    业务链路：
    1. 跨表检索同源海报路径（冷表 → 热表）-> 
    2. 原子写入：一次性更新 status + local_poster_path + imdb_id + tmdb_id -> 
    3. 提交事务 -> 4. 返回操作结果
    """
    with self.db_lock:
        conn = self._get_conn()
        try:
            conn.execute("BEGIN")
            
            # Step 1: 跨表检索同源海报路径
            inherited_poster = None
            if imdb_id:
                # 从 media_archive 冷表查找同源文件的海报路径
                cursor = conn.execute(
                    "SELECT local_poster_path FROM media_archive WHERE imdb_id = ? AND local_poster_path IS NOT NULL LIMIT 1",
                    (imdb_id,)
                )
                row = cursor.fetchone()
                inherited_poster = row[0] if row else None
                
                # 如果冷表没找到，从热表 tasks 查找
                if not inherited_poster:
                    cursor = conn.execute(
                        "SELECT local_poster_path FROM tasks WHERE imdb_id = ? AND local_poster_path IS NOT NULL LIMIT 1",
                        (imdb_id,)
                    )
                    row = cursor.fetchone()
                    inherited_poster = row[0] if row else None
            
            # Step 2: 原子写入
            table = "media_archive" if is_archive else "tasks"
            pk_field = "original_task_id" if is_archive else "id"
            
            updates = ["status = 'ignored'"]
            params = []
            
            if inherited_poster:
                updates.append("local_poster_path = ?")
                params.append(inherited_poster)
            
            if imdb_id:
                updates.append("imdb_id = ?")
                params.append(imdb_id)
            
            if tmdb_id:
                updates.append("tmdb_id = ?")
                params.append(tmdb_id)
            
            params.append(task_id)
            
            # Step 3: 执行原子级 UPDATE
            cursor = conn.execute(
                f"UPDATE {table} SET {', '.join(updates)} WHERE {pk_field} = ?",
                params
            )
            
            # Step 4: 提交事务
            conn.commit()
            return cursor.rowcount > 0
        except Exception as e:
            conn.rollback()
            logger.error(f"[TaskRepo] mark_task_as_ignored_and_inherit 失败: {e}", exc_info=True)
            raise
```

## 五、防重拦截（IMDb 金标准检测）

**业务链路**：
```
1. 获得 IMDb ID -> 
2. 检查 check_media_exists() -> 
3. 若重复则标记 ignored + 继承同源海报 -> 
4. 跳过物理归档
```

**关键设计**：
- **精确到季集号**：防止同剧不同集被误杀
- **跨表查询**：同时检查热表和冷表
- **原子操作**：标记 ignored 和继承海报必须一起完成

**实现**（task_repo.py）：
```python
def check_media_exists(self, imdb_id: str, media_type: str, season: int = None, episode: int = None) -> bool:
    """
    业务链路：
    1. 若为电影，仅查 IMDb ID + type -> 
    2. 若为剧集，精确到 season + episode -> 
    3. 双表查询（冷表优先）
    """
    if not imdb_id or not str(imdb_id).strip():
        return False
    
    _imdb = str(imdb_id).strip()
    with self.db_lock:
        conn = self._get_conn()
        if media_type == "movie":
            # 电影：仅查 IMDb ID + type
            if conn.execute(
                "SELECT 1 FROM media_archive WHERE imdb_id = ? AND type = 'movie' LIMIT 1",
                (_imdb,)
            ).fetchone():
                return True
            if conn.execute(
                "SELECT 1 FROM tasks WHERE imdb_id = ? AND type = 'movie' AND status IN ('archived', 'scraped') LIMIT 1",
                (_imdb,)
            ).fetchone():
                return True
        elif media_type == "tv":
            # 剧集：必须精确到 season + episode，防止同剧不同集被误杀
            if season is None or episode is None:
                return False
            if conn.execute(
                "SELECT 1 FROM media_archive WHERE imdb_id = ? AND type = 'tv' AND season = ? AND episode = ? LIMIT 1",
                (_imdb, season, episode)
            ).fetchone():
                return True
            if conn.execute(
                "SELECT 1 FROM tasks WHERE imdb_id = ? AND type = 'tv' AND season = ? AND episode = ? AND status IN ('archived', 'scraped') LIMIT 1",
                (_imdb, season, episode)
            ).fetchone():
                return True
        return False

**调用约束**：
- 仅在 IMDb 重复检测确认为物理副本后调用
- `is_archive=True` 时操作 `media_archive` 冷表（`pk_field = 'original_task_id'`）
- `is_archive=False` 时操作 `tasks` 热表（`pk_field = 'id'`）
- 严禁在此方法外单独更新 `status='ignored'`（会导致 VHS 特效白板）

## 五、双表删除机制

```
delete_task(task_id):
  1. 先查 tasks 中该 id 对应的 path
  -> 2. DELETE FROM tasks WHERE id = ?
  -> 3. DELETE FROM media_archive WHERE path = ?         （按路径，最准确）
  -> 4. DELETE FROM media_archive WHERE original_task_id = ?  （兜底）
```

⚠️ `tasks.id` 与 `media_archive.id` 各自独立自增，禁止用 id 交叉对删。

## 六、失忆救援机制（Amnesia Recovery）

**触发条件**：扫描到 library 目录的已有文件（数据库被误删后重新扫描）

```
1. 从父目录名提取片名和年份（格式：Title (Year)）
-> 2. 剧集层级修正：父目录是 Season XX 时向上一级寻找根目录
-> 3. 自动绑定本地海报（poster.jpg / poster.png）
-> 4. target_path = 当前文件路径（就地归档）
-> 5. 立刻流转至 media_archive 冷表，跳过刮削流程
```
