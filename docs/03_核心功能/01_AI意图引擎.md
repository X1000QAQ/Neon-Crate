## 一、系统定位

AI 大脑是 Neon Crate 的总控神经，将用户自然语言指令转化为系统操作。核心链路：

```
用户输入 → LLM 意图识别 → 触发系统任务 / 调用 Servarr 下载 / 普通对话
```

所有配置（人格、路由规则、归档规则、LLM 参数）均从数据库**动态读取**，WebUI 修改后无需重启即时生效。

## 二、AIAgent.process_message() 全流程

```
1. 候选等待状态拦截
   ├─ 读取 DB _pending_candidates
   ├─ 输入匹配候选（纯数字/序号格式/含片名前4字）
   │     → ServarrClient(tmdb_id) 直接下载，清除状态
   └─ 不匹配 → 清除状态，放行至意图识别

2. LLMClient.call_llm() 意图识别
   → master_router_rules 作为 System Prompt
   → 要求 JSON 输出：{intent, query, year, type}
   → 解析失败 → 关键词规则引擎兜底

3. _generate_llm_response() 生成响应
   ├─ SYSTEM_STATUS → 注入真实数据库统计（防 LLM 编造）
   ├─ DOWNLOAD（模糊）→ _tmdb_search_candidates()
   │     → 前5候选写入 _pending_candidates
   │     → 响应末尾附加 __CANDIDATES__[...]
   ├─ DOWNLOAD（精准）→ ServarrClient 直接下载
   └─ CHAT → 注入系统快报 + ai_persona，LLM 生成回复

4. 响应含 __CANDIDATES__ → 强制 action_code=None
5. 返回 (response_text, action_code)
```

## 三、意图常量

| 常量 | 触发动作 |
|------|----------|
| `ACTION_SCAN` | `POST /tasks/scan` |
| `ACTION_SCRAPE` | `POST /tasks/scrape_all` |
| `ACTION_SUBTITLE` | `POST /tasks/find_subtitles` |
| `DOWNLOAD` | ServarrClient（agent 内部完成）|
| `LOCAL_SEARCH` | 本地媒体库搜索 |
| `SYSTEM_STATUS` | 查询并注入真实统计数据 |
| `CHAT` | 普通对话 |

## 四、LLMClient

- 兼容所有 OpenAI 格式 API（DeepSeek / Together / Ollama）
- 三连击指数退避重试：1s → 2s → 4s
- `"enable_thinking": False` 关闭 Qwen3 思考模式
- 从数据库动态读取 `llm_provider`、URL、Key、Model

## 五、候选白名单反转逻辑

只有输入**明确像**候选选择才保留状态：
- 纯数字（`1` / `2`）
- `序号.` 格式（`1.` / `2.`）
- 含候选片名前 4 字

其余一律清除状态，避免黑名单覆盖不全问题。

## 六、前端 AI 舱（AiSidebar.tsx）

### AbortController 全链路
```
1. 注入信号：每次发送创建新 AbortController，signal 传入 fetch
-> 2. 物理掐断：连续发送时 abort() 旧请求，浏览器切断连接
-> 3. 后端回收：Uvicorn 注入 CancelledError，释放 LLM 推理资源
```

### 候选封死机制
- 只有最后一条含候选消息的按钮处于激活状态
- 任何新消息发送立即封死所有旧候选（`disabled` + 变灰）
- 双重保险：候选按钮点击 + `handleSendText` 封死逻辑
