## 一、元数据工厂全景

```
TMDBAdapter（搜索 + 详情 + 外部 ID）
  └─► MetadataManager
        ├─► generate_nfo()      → Jellyfin/Emby/Plex 标准 NFO
        ├─► download_poster()   → poster.jpg（原图质量）
        └─► download_fanart()   → fanart.jpg（w1280）
```

各步骤独立容错：海报下载失败不影响 NFO 生成。

## 二、TMDBAdapter

### 搜索方法

| 方法 | 说明 |
|------|------|
| `search_movie(query, year)` | 搜索电影 |
| `search_tv(query, year)` | 搜索剧集 |
| `get_movie_details(tmdb_id)` | 电影详情（含演职员）|
| `get_tv_details(tmdb_id)` | 剧集详情 |
| `get_external_ids(tmdb_id, media_type)` | 获取 IMDB ID 等外部 ID |

### 精确匹配策略

```
results = search_tv(query, year)
  ├─ 优先：original_name == query（严格）
  ├─ 其次：name == query（严格）
  ├─ 宽松：去末尾集号后匹配（"The Boys 02" → "The Boys"）
  └─ 兜底：results[0]
```

### 限流处理

429 自动重试：最多 3 次，指数退避 2s → 4s → 8s。

## 三、MetadataManager（v1.0.1-Enhanced）

### NFO 生成（generate_nfo）

**业务链路**：
1. 根据 media_type 确定 NFO 文件名（movie.nfo 或 tvshow.nfo）
2. 调用 TMDB API 获取详细信息
3. 构建 XML 结构（符合 Jellyfin / Emby / Plex 标准）
4. 写入文件（经 `_validate_path()` 防御路径穿越）

**输出内容**：
- 标题、原始标题、年份、简介、评分
- 演职员列表（actor/director）
- TMDB ID、IMDB ID
- 剧集额外字段：季号、集号、首播日期

**防御机制**：
```python
# 所有文件写入经 _validate_path() 防御路径穿越
def _validate_path(self, output_path: str) -> bool:
    """
    业务链路：1. 解析目标路径 -> 2. 检查是否在允许目录内 -> 
             3. 防止 ../ 等穿越攻击
    """
    resolved = Path(output_path).resolve()
    allowed = Path(self.output_dir).resolve()
    try:
        resolved.relative_to(allowed)
        return True
    except ValueError:
        return False
```

### 海报下载（download_poster）

**业务链路**：
1. 调用 TMDB API 获取海报 URL
2. 下载原图质量（无压缩）
3. 保存为 poster.jpg
4. 返回本地路径

**关键设计**：
- 原图质量：无压缩，保证清晰度
- 文件名固定：poster.jpg（Jellyfin/Emby/Plex 标准）
- 容错处理：下载失败不影响 NFO 生成

### Fanart 下载（download_fanart）

**业务链路**：
1. 调用 TMDB API 获取 Fanart URL
2. 下载 w1280 分辨率（平衡质量与大小）
3. 保存为 fanart.jpg
4. 返回本地路径

**关键设计**：
- 分辨率：w1280（1280px 宽度，平衡质量与大小）
- 文件名固定：fanart.jpg（Jellyfin/Emby/Plex 标准）
- 容错处理：下载失败不影响其他步骤

## 四、SmartLink 归档组织器

### 三级降级策略

```
1. 硬链接 os.link()      同分区，零额外空间（首选）
-> 2. 软链接 os.symlink()  跨分区，依赖源文件存在
-> 3. 文件复制 shutil.copy2 兜底，占用双倍空间
```

### 归档路径规范

```
电影：{library_root}/{title} ({year})/{filename}
剧集：{library_root}/{title} ({year})/Season {season:02d}/{filename}
```

### 季号补充机制

当 `season=1` 时，从原始路径 `parts` 搜索 `Season XX` 目录名补充真实季号，并通过 `update_task_title_year(season=N)` 回写数据库。

---

## 五、1+1 路径约束

保存设置时强制校验：
- 电影媒体库（type=library, category=movie）启用数 ≤ 1
- 剧集媒体库（type=library, category=tv）启用数 ≤ 1
- 有媒体库时两种类型必须同时存在

违反时返回 HTTP 400，前端展示错误提示。
