### 1.2 冷启动注入机制现状

`_inject_ai_defaults`（`config_repo.py`）**仅注入 5 个 AI 相关键**：

```python
for key in ["ai_name", "ai_persona", "expert_archive_rules",
            "master_router_rules", "filename_clean_regex"]:
    if key in defaults and not self.get_config(key, "").strip():
        self.set_config(key, defaults[key])
```

`supported_video_exts` 和 `supported_subtitle_exts` **不在注入列表中**，因此即使加入 `DEFAULT_CONFIG`，冷启动时也不会自动写入数据库。

### 1.3 `get_config` 兜底机制

```python
# config_repo.py get_config()
value = settings.get(key, None)
if value is None or (isinstance(value, str) and not value.strip()):
    return DEFAULT_CONFIG.get(key, default)  # ← 自动兜底 DEFAULT_CONFIG
```

这意味着：**只要 `DEFAULT_CONFIG` 中有该键，`get_config()` 会在未配置时自动返回默认值，无需强制写入数据库。** Phase 3 的「冷启动注入」实际上已被 `get_config` 的兜底机制覆盖。

## 二、流程示意图

### 2.1 当前扩展名读取流程（硬编码模式）

```
扫描阶段（ScanEngine）
  └─ 从 constants.VIDEO_EXTS_EXTENDED 读取扩展名（硬编码）
       ↓ 无法用户自定义

核级清理（_nuclear_clean_directory）
  └─ 从 _VIDEO_EXTENSIONS（scrape_task.py line 922）读取（硬编码）
       ↓ 无法用户自定义，且与 constants.py 不同步（多了 .rmvb .iso .vob）

字幕检测（_check_local_subtitles）
  └─ valid_exts = {'.srt', '.ass', '.vtt', '.sub', '.idx'}（局部硬编码）
       ↓ 无法用户自定义
```

### 2.2 目标扩展名读取流程（动态配置模式）

```
DEFAULT_CONFIG
  ├─ supported_video_exts: ".mkv,.mp4,.avi,..."  ← 新增
  └─ supported_subtitle_exts: ".srt,.ass,.vtt,..."  ← 新增
       ↓ get_config() 自动兜底

扫描阶段（ScanEngine）
  └─ db.get_config("supported_video_exts") → 解析为 set
       ↓ 用户可自定义

核级清理（_nuclear_clean_directory）
  └─ 接收 video_exts 参数（由调用方从 DB 读取传入）
       ↓ 消除硬编码常量

字幕检测（_check_local_subtitles）
  └─ 接收 sub_exts 参数 或 从 db 读取
       ↓ 统一来源

前端 BasicSettings
  └─ 「格式控制」区域 → POST /tasks/settings → DB 持久化
```

### Phase 3 — 冷启动注入（已有兜底，仅需扩展注入列表）
