## 二、根因分析

### 2.1 后端 status 枚举的真实值

后端 `domain_media.py` 中的 status 字段定义：

```python
status: str = Field(
    default="pending",
    description="任务状态: pending | archived | failed | ignored | match failed | scraped"
)
```

**关键发现**：`"match failed"` 中间有一个**空格**，不是下划线！

### 2.2 当前动态映射的工作原理

`MediaTable.tsx` ~179 行的当前实现：

```tsx
t(('status_' + task.status.replace(' ', '_').toLowerCase()) as ...)
```

`.replace(' ', '_')` 将空格替换为下划线，因此：
- `"match failed"` → `"status_match_failed"` → `t('status_match_failed')` → ✅ 正确映射

### 2.3 Gemini 方案的致命错误

Gemini 要求删除 `status_match_failed` 并只保留 `status_failed`，但：

```
后端枚举值        replace(' ','_')    i18n key            结果
"failed"      →  "status_failed"   →  ✅ 存在
"match failed" →  "status_match_failed" →  ❌ 若被删除则 fallback 到原始值 "match failed"
```

**删除 `status_match_failed` 后，所有状态为 `match failed` 的任务将在 UI 上直接显示英文 `"match failed"`，违反 i18n 原则。**

## 三、流程示意图

### 3.1 当前映射流程（正确）

```
task.status = "match failed"
    │
    ▼
.replace(' ', '_') → "match_failed"
    │
    ▼
'status_' + "match_failed" = "status_match_failed"
    │
    ▼
t('status_match_failed') = '匹配失败'  ← ✅ 正确显示
```

### 3.2 若按 Gemini 方案删除后（错误）

```
task.status = "match failed"
    │
    ▼
.replace(' ', '_') → "match_failed"
    │
    ▼
'status_' + "match_failed" = "status_match_failed"
    │
    ▼
t('status_match_failed') = 'status_match_failed'  ← key 不存在，返回 key 本身
    │
    ▼
|| task.status → "match failed"  ← ❌ 显示原始英文枚举值！
```

### 3.3 正确的后端枚举 → i18n key 映射表

| 后端 status 值 | `.replace(' ','_')` | i18n key | 是否需要保留 |
|--------------|-------------------|----------|------------|
| `pending` | `pending` | `status_pending` | ✅ 必须保留 |
| `archived` | `archived` | `status_archived` | ✅ 必须保留 |
| `failed` | `failed` | `status_failed` | ✅ 必须保留 |
| `ignored` | `ignored` | `status_ignored` | ✅ 必须保留 |
| `match failed` | `match_failed` | `status_match_failed` | ✅ **必须保留！** |
| `scraped` | `scraped` | `status_scraped` | ✅ 必须保留 |
| `processing` | `processing` | `status_processing` | ✅ 必须保留 |

## 四、正确的清洗方案

Gemini 方案中唯一正确的部分：**5 个标准 Key 的语义是对的**，但它遗漏了 `match failed` 这个真实存在的后端枚举值。

**正确的清洗动作（最小化）**：

1. **保留** `status_match_failed`（对应后端 `"match failed"`）
2. **无需修改** `MediaTable.tsx`（当前映射逻辑已正确）
3. **无需全局替换**（`status_match_failed` 只出现在 `i18n.ts` 字典中，不在组件代码里）

**实际上需要的清洗仅一项**：
- 确认 `status_scraped` 已存在（上轮已加入）
- 确认 `status_processing` 已存在（上轮已加入）
- 字典已经处于正确状态，**无需任何修改**
