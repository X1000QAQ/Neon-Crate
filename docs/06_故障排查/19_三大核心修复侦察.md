## 一、Phase 1 — 文件夹土木工程

**插入位置**：`scrape_task.py` Step 2（重命名）和 Step 3（DB 同步）之间

**需要**：在文件顶部 `import shutil`

**TV 季搬运逻辑**：
```python
if body.media_type == "tv" and task_season is not None:
    target_season_dir = Path(metadata_dir) / f"Season {task_season}"
    target_season_dir.mkdir(parents=True, exist_ok=True)
    current_video = Path(new_video_path)
    if current_video.parent.resolve() != target_season_dir.resolve():
        dest = target_season_dir / current_video.name
        if not dest.exists():
            shutil.move(str(current_video), str(dest))
            new_video_path = str(dest)
```

**电影根目录重命名逻辑**：
```python
if body.media_type == "movie" and new_title:
    expected_dir_name = f"{new_title} ({new_year})" if new_year else new_title
    current_root = Path(new_video_path).parent.resolve()
    if current_root.name != expected_dir_name:
```

## 二、Phase 2 — IMDb ID 强制显影

**后端**：`search_tmdb` endpoint 搜索结果已有 `imdb_id: None`（上轮添加）
- TMDB `/search/movie` 和 `/search/tv` 的原始结果不含 `imdb_id`
- 需要为每个结果额外调用 `get_external_ids(tmdb_id, media_type)` 拼接
- 但这会产生 N 次额外 HTTP 请求（10 结果 = 10 次），影响性能
- **方案**：在 `search_tmdb` 中，只对前 5 条结果批量获取 external_ids

**前端**：`RebuildDialog.tsx` 已显示 `IMDb ttxxxxxxx` 或 `TMDB #xxx`（上轮已修）
- 需要确认显示逻辑正确：`r.imdb_id ? \`IMDb ${r.imdb_id}\` : \`TMDB #${r.tmdb_id}\``

## 三、Phase 3 — 格式设置空白修复

**根因**：`get_all_config()` 直接返回 `config.json` 内容，若 `supported_video_exts` 从未被写入 `config.json`，返回空字符串。

**修复位置**：`get_all_config()` 返回前，遍历 `DEFAULT_CONFIG` 补全空值：
```python
settings = config.setdefault("settings", {})
for k, v in DEFAULT_CONFIG.items():
    if k not in settings or settings[k] is None or settings[k] == "":
        settings[k] = v
```
