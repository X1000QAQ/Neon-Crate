## 防线 2：冷热双表与前端 Task 接口对齐

### 检查项 2.1 [HIGH]：热表 (tasks) 字段映射

**后端热表 tasks 字段**（`db_manager.py`）：
```sql
CREATE TABLE IF NOT EXISTS tasks (
    id                INTEGER PRIMARY KEY AUTOINCREMENT,
    path              TEXT UNIQUE NOT NULL,
    file_name         TEXT,
    clean_name        TEXT,
    type              TEXT DEFAULT 'movie',
    status            TEXT DEFAULT 'pending',
    tmdb_id           TEXT,
    imdb_id           TEXT,
    title             TEXT,
    year              TEXT,
    target_path       TEXT,
    sub_status        TEXT DEFAULT 'pending',
    last_sub_check    TEXT,
    created_at        TEXT DEFAULT CURRENT_TIMESTAMP,
    poster_path       TEXT,
    local_poster_path TEXT,
    season            INTEGER,
    episode           INTEGER,
    is_active         INTEGER DEFAULT 1
)
```

**前端 Task 接口**（`frontend/types/index.ts:1-24`）：
```typescript
export interface Task {
  id: number;
  file_path: string;
  file_name?: string;
  clean_name?: string;
  media_type: 'movie' | 'tv';
  status: 'pending' | 'archived' | 'failed' | 'ignored' | 'match failed' | 'scraped';
  tmdb_id?: number;
  imdb_id?: string;
  title?: string;
  year?: number;
  poster_path?: string;
  local_poster_path?: string;
  target_path?: string;
  sub_status?: 'pending' | 'failed' | 'missing' | 'scraped' | 'found';
  season?: number | null;
  episode?: number | null;
  created_at: string;
}
```

**对齐分析**：

| 后端字段 | 后端类型 | 前端字段 | 前端类型 | 状态 |
|---------|---------|---------|---------|------|
| id | INTEGER | id | number | ✅ |
| path | TEXT | file_path | string | ✅ |
| file_name | TEXT | file_name? | string? | ✅ |
| clean_name | TEXT | clean_name? | string? | ✅ |
| type | TEXT | media_type | 'movie' \| 'tv' | ⚠️ 类型值不同 |
| status | TEXT | status | 枚举 | ⚠️ 枚举值不同 |
| tmdb_id | TEXT | tmdb_id? | number? | ⚠️ 类型不匹配 |
| imdb_id | TEXT | imdb_id? | string? | ✅ |
| title | TEXT | title? | string? | ✅ |
| year | TEXT | year? | number? | ⚠️ 类型不匹配 |
| target_path | TEXT | target_path? | string? | ✅ |
| sub_status | TEXT | sub_status? | 枚举? | ⚠️ 枚举值不同 |
| last_sub_check | TEXT | ❌ 缺失 | — | ❌ 缺失 |
| created_at | TEXT | created_at | string | ✅ |
| poster_path | TEXT | poster_path? | string? | ✅ |
| local_poster_path | TEXT | local_poster_path? | string? | ✅ |
| season | INTEGER | season? | number \| null? | ✅ |
| episode | INTEGER | episode? | number \| null? | ✅ |
| is_active | INTEGER | ❌ 缺失 | — | ❌ 缺失 |

**问题 2.1a [MEDIUM]：后端 `type` 为 TEXT，前端期望 'movie' \| 'tv'**
- 后端存储：`'movie'` 或 `'tv'` 字符串
- 前端期望：严格的 `'movie' | 'tv'` 联合类型
- **风险**：若后端返回其他值（如 `'mixed'`），前端类型检查会失败
- **建议**：前端应改为 `media_type: string` 或添加运行时验证

**问题 2.1b [MEDIUM]：后端 `tmdb_id` 为 TEXT，前端期望 number**
- 后端存储：`'12345'` 字符串
- 前端期望：`12345` 数字
- **风险**：类型不匹配，可能导致 API 调用时参数错误
- **建议**：后端应返回 `number` 或前端应进行类型转换

**问题 2.1c [MEDIUM]：后端 `year` 为 TEXT，前端期望 number**
- 后端存储：`'2024'` 字符串
- 前端期望：`2024` 数字
- **风险**：同上
- **建议**：后端应返回 `number` 或前端应进行类型转换

**问题 2.1d [MEDIUM]：后端 `status` 枚举值不同**
- 后端可能值：`'pending'`, `'scraped'`, `'failed'`, `'ignored'` 等
- 前端期望值：`'pending' | 'archived' | 'failed' | 'ignored' | 'match failed' | 'scraped'`
- **风险**：若后端返回 `'archived'`，前端无法识别
- **建议**：确认后端实际返回的枚举值集合

**问题 2.1e [MEDIUM]：后端 `sub_status` 枚举值不同**
- 后端存储：`'pending'`, `'scraped'` 等
- 前端期望：`'pending' | 'failed' | 'missing' | 'scraped' | 'found'`
- **风险**：同上
- **建议**：确认后端实际返回的枚举值集合

**问题 2.1f [HIGH]：后端 `last_sub_check` 字段在前端 Task 中缺失**
- 后端存储：`last_sub_check TEXT`
- 前端定义：无此字段
- **风险**：若后端返回此字段，前端会忽略；若前端需要此字段，会得到 `undefined`
- **建议**：确认前端是否需要此字段，若需要则添加到 Task 接口

**问题 2.1g [HIGH]：后端 `is_active` 字段在前端 Task 中缺失**
- 后端存储：`is_active INTEGER DEFAULT 1`
- 前端定义：无此字段
- **风险**：同上
- **建议**：确认前端是否需要此字段
