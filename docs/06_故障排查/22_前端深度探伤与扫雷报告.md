### 问题 1.2 [MEDIUM] 错误处理不一致 — 部分 API 调用使用 `catch(() => ({}))`，部分直接抛出

**位置**：`frontend/lib/api.ts` 多处

**当前代码**：
```typescript
// 第 83 行
const body = await res.json().catch(() => ({}));

// 第 109 行
const errData = await res.json().catch(() => ({}));

// 第 330 行
const err = await res.json().catch(() => ({}));
```

**问题**：
- 当响应体不是 JSON 时，`catch(() => ({}))` 会返回空对象
- 调用方无法区分"真实的空响应"和"JSON 解析失败"
- 错误信息可能丢失

**建议**：
```typescript
const errData = await res.json().catch((e) => {
  logger.debug(`[API] JSON 解析失败: ${e}`);
  return { detail: 'Invalid JSON response' };
});
```

**影响范围**：中（错误诊断困难）

## 防线 2：React 状态与生命周期排雷

### 问题 2.1 [HIGH] Hook 依赖陷阱 — `refreshSettings` 在 `useEffect` 依赖中可能导致无限循环

**位置**：`frontend/context/SettingsContext.tsx:62-64`

**当前代码**：
```typescript
  const refreshSettings = useCallback(async () => {
    try {
      setIsLoading(true);
      const data = await api.getSettings();
      setConfigState(data);
    } catch (error) {
      // 静默降级
    } finally {
      setIsLoading(false);
    }
  }, []);  // ✅ 空依赖数组，正确

  useEffect(() => {
    void refreshSettings();
  }, [refreshSettings]);  // ⚠️ refreshSettings 在依赖中
```

**问题分析**：
- `refreshSettings` 使用 `useCallback` 且依赖数组为空 `[]`，函数引用稳定
- `useEffect` 依赖 `refreshSettings`，但由于 `useCallback` 的空依赖，不会触发无限循环
- **实际上这是正确的实现**，但代码审查时容易误判

**建议**：
- 保持现状（已正确实现）
- 或添加注释说明：
```typescript
  // refreshSettings 使用 useCallback([]) 保证引用稳定，不会触发重复调用
  useEffect(() => {
    void refreshSettings();
  }, [refreshSettings]);
```

**结论**：✅ 无问题，但建议添加注释防止误判。

### 问题 2.3 [MEDIUM] 异步 setState 在组件卸载后执行 — 潜在内存泄漏

**位置**：`frontend/components/media/RebuildDialog.tsx` 和其他异步组件

**问题**：
- 组件中有多个异步操作（`api.searchTmdb`, `api.rebuildTask`）
- 若组件在异步操作完成前卸载，`setState` 仍会执行
- 触发 React Warning: "Can't perform a React state update on an unmounted component"

**当前代码示例**：
```typescript
  const handleSearch = useCallback(async () => {
    setSearching(true);
    setSearchErr(null);
    try {
      const res = await api.searchTmdb(kw.trim(), mt);
      setResults(res);  // ⚠️ 组件可能已卸载
      setSelected(null);
    } catch (err: any) {
      setSearchErr(err.message || 'Search failed');  // ⚠️ 组件可能已卸载
    } finally {
      setSearching(false);  // ⚠️ 组件可能已卸载
    }
  }, [mediaType, keyword]);
```

**建议**：
```typescript
  useEffect(() => {
    let isMounted = true;
    
    const handleSearch = async () => {
      setSearching(true);
      try {
        const res = await api.searchTmdb(kw.trim(), mt);
        if (isMounted) {
          setResults(res);
          setSelected(null);
        }
      } catch (err: any) {
        if (isMounted) {
          setSearchErr(err.message || 'Search failed');
        }
      } finally {
        if (isMounted) {
          setSearching(false);
        }
      }
    };
    
    return () => {
      isMounted = false;
    };
  }, []);
```

**影响范围**：中（内存泄漏 + 控制台警告）

### 问题 2.4 [MEDIUM] 状态初始化依赖 Props — `season` 和 `episode` 使用 `task.season` 初始化但未监听变化

**位置**：`frontend/components/media/RebuildDialog.tsx:50-51`

**当前代码**：
```typescript
  const [season, setSeason]   = useState<number | ''>(task.season ?? 1);
  const [episode, setEpisode] = useState<number | ''>(task.episode ?? '');
```

**问题**：
- 若父组件传入的 `task` prop 发生变化，`season` 和 `episode` 状态不会更新
- 这是 React 的经典陷阱：`useState` 的初始值只在组件首次渲染时生效

**建议**：
```typescript
  // 方案 1：使用 useEffect 同步 props 变化
  useEffect(() => {
    setSeason(task.season ?? 1);
    setEpisode(task.episode ?? '');
  }, [task.season, task.episode]);

  // 方案 2：直接使用 props（若不需要本地编辑）
  const season = task.season ?? 1;
  const episode = task.episode ?? '';
```

**影响范围**：中（UI 状态不同步）
