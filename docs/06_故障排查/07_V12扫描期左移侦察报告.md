## 三、关键风险分析

### 风险 1：白嫖逻辑1 删除的副作用

- **有 NFO 的文件**：NFO 数据已覆盖目录名，白嫖逻辑1 完全冗余，删除安全 ✅
- **无 NFO 的 library 文件**：删除后将丢失从目录名提取的 `clean_name`/`year`，
  只能依赖 ScanEngine 从文件名清洗出的结果
- **结论**：可接受，ScanEngine 的文件名清洗已足够可靠

### 风险 2（关键）：archive_task 触发条件失效

当前代码：
```python
if default_status == "archived" and task_id:
    db.archive_task(task_id)
```

若 download 文件（`default_status="pending"`）因 NFO 越级为 `archived`，
`default_status` 仍为 `"pending"`，`archive_task` **不会被调用**！
文件无法流转至 `media_archive` 冷表，数据库状态不一致。

**必须修改为**：
```python
if task_data.get("status") == "archived" and task_id:
    db.archive_task(task_id)
```

### 风险 3：Gemini Phase 3 的 add_task 修改无效

`scan_task.py` 使用的是 `insert_task`（Dict版），不是 `add_task`：
```python
# scan_task.py 的实际调用链
db.insert_task(task_data)  ← 这里
# add_task 从未在 scan_task.py 中被调用
```

**Gemini Phase 3 可完全跳过**，`insert_task` 上轮已扩展完毕。

try:

    _nfo_path = find_nfo(file_path)
    if _nfo_path:

            task_data["target_path"] = file_path
