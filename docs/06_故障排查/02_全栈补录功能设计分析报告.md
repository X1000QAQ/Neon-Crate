## 二、物理路径分析（AIO 硬链接归档防冲突）

**结论：手动补录不会产生冗余文件，前提是走就地补录模式（In-Place）。**

### 归档流程回顾

```
下载目录文件
    → SmartLink.create_link(src, dst)
        ├─ os.link(src, dst)          硬链接（首选，零空间占用）
        ├─ os.symlink(src, dst)       软链接（跨分区 EXDEV 降级）
        └─ already_exists             幂等返回（目标已存在不报错）
    → MetadataManager.generate_nfo()    写 movie.nfo / tvshow.nfo
    → MetadataManager.download_poster() 写 poster.jpg（已存在则跳过）
    → db.update_task_status(status="archived")
    → db.archive_task(task_id)           tasks → media_archive 冷迁移
```

### 冗余文件风险矩阵

| 场景 | 风险 | 防护机制 |
|------|------|--------|
| 重复触发 manual_rebuild | ❌ 无冗余 | `SmartLink.create_link` 幂等；`download_poster` 已存在即跳过 |
| 指定错误 tmdb_id 后再次补录 | ⚠️ NFO 被覆盖 | NFO 是覆盖写，正常预期行为 |
| 指定错误 keyword 导致标题变更 | ⚠️ 旧硬链接孤存 | manual_rebuild 只更新元数据，不移动物理文件 |

### 就地补录模式选择依据

`manual_rebuild` 必须走 `_is_library_file=True` 分支（就地补录模式）：
- 任务已被 `archive_task` 迁移到 `media_archive`，物理文件已在 library 路径
- 重新移动文件会破坏硬链接 inode，导致做种失败
- 就地补录仅重写 NFO + 海报，不触碰物理文件位置

## 三、数据流分析（tasks ↔ media_archive）

### 状态机与表关系

```
tasks 表（热存储）                     media_archive 表（冷存储）
─────────────────────────────          ──────────────────────────────
status=pending
  ↓ perform_scrape_all_task_sync
status=archived
  ↓ archive_task()
  [DELETE FROM tasks]  ──────────→    INSERT INTO media_archive
                                       (original_task_id=旧id, ...)
```

### manual_rebuild 写路径设计

补录目标是已在 `media_archive` 的记录（`is_archive=True`），写路径：

```python
# Step 1: 重新刮削 TMDB（用新 keyword 或 tmdb_id）
# Step 2: 就地重写 NFO（写入 target_path 所在目录）
# Step 3: 就地重下载海报（覆盖写，已存在检查已在 download_poster 内）
# Step 4: 更新 media_archive 记录
db.update_any_task_metadata(
    task_id=archive_id,
    is_archive=True,           # ← 关键：写入 media_archive，非 tasks
    imdb_id=new_imdb_id,
    tmdb_id=new_tmdb_id,
    title=new_title,
    year=new_year,
    sub_status="pending"       # 重置字幕状态，触发重搜
)
```

## 四、AI Prompt 回归测试（Chinese-First Strategy）

### 问题复现：《飞驰人生 Pegasus》识别偏差

| 文件名 | 旧行为（双语保留策略） | 根因 |
|--------|----------------------|------|
| `飞驰人生 Pegasus (2019).mkv` | query=`飞驰人生 Pegasus`（双语混合） | TMDB 中文端点收到英文后缀，匹配率下降 |
| `飞驰人生 Pegasus (2019).mkv` | query=`Pegasus`（极端简化） | 直接命中西班牙电影《Pegasus》 |

### 新策略：Chinese-First（中文优先）

**核心规则**（写入 `ai_identify_media` 的 `user_prompt`）：

> 中文优先铁律：
> - 若文件名中存在中文字符（CJK Unicode：\u4e00-\u9fff），query **只输出中文标题**（如 `飞驰人生`）
> - 严禁在 query 中保留英文副标题、英文译名、拼音，以确保 TMDB 中文搜索端点命中率
> - 只有文件名完全不含中文时，才使用英文标题作为 query

**回归测试对比**：

| 文件名 | 旧 query | 新 query | TMDB 结果 |
|--------|---------|---------|----------|
| `飞驰人生 Pegasus (2019).mkv` | `飞驰人生 Pegasus` | `飞驰人生` | ✅ 精确命中 (tt8236282) |
| `Pegasus (2021).mkv` | `Pegasus` | `Pegasus` | ✅ 无变化 |
| `The Boys S03E01.mkv` | `The Boys` | `The Boys` | ✅ 无变化 |
| `速度与激情 Fast & Furious 9.mkv` | `Fast & Furious 9` | `速度与激情` | ✅ 精确命中 |

## 五、副作用审计：高并发死锁分析

### 锁路径对比

| 操作 | 锁类型 | 持锁粒度 | 并发安全 |
|------|--------|---------|--------|
| `perform_scrape_all_task_sync` | `threading.RLock` (db_lock) | 单条 SQL (~1ms) | ✅ |
| `manual_rebuild` (新增) | `threading.RLock` (db_lock) | 单条 SQL (~1ms) | ✅ |
| `_scrape_entry_lock` | `threading.Lock` | 整个刮削任务生命周期 | ✅ 防重入 |

**结论**：`manual_rebuild` 使用独立 HTTP 请求线程，与 `_scrape_entry_lock` 无交集，不阻塞全量刮削任务。`db_lock` 是 `RLock`（可重入），两操作竞争时最差情况是毫秒级等待，**不产生死锁**。

### Task 2：`backend/app/api/v1/endpoints/tasks/scrape_task.py`

**新增接口**：`POST /api/v1/tasks/manual_rebuild`

#### Request Body

```python
class ManualRebuildRequest(BaseModel):
    task_id: int                    # media_archive.id（is_archive=True）或 tasks.id
    is_archive: bool = True         # 默认操作 media_archive 表
    tmdb_id: Optional[int] = None   # 直接指定 TMDB ID（跳过搜索，最精确）
    keyword: Optional[str] = None   # 手动指定搜索词（tmdb_id 未提供时使用）
    media_type: str = "movie"       # movie / tv
    refix_nfo: bool = True          # 是否重生成 NFO（覆盖写）
    refix_poster: bool = True       # 是否重下载 poster.jpg（覆盖写）
    refix_subtitle: bool = True     # 是否重置 sub_status=pending（触发字幕重搜）
```

#### 执行链路（复用现有模块，零新增依赖）

```
POST /tasks/manual_rebuild
  ├─ 读取 target_path（从 media_archive 或 tasks 表）
  ├─ 解析 metadata_dir = os.path.dirname(target_path)
  │    └─ 剧集目录修正：若末级为 Season N，上移一层到剧集根目录
  │
  ├─ [refix_nfo=True]
  │    ├─ 若提供 tmdb_id：TMDBAdapter.get_movie_details / get_tv_details
  │    └─ 若提供 keyword：TMDBAdapter.search_media(keyword) → 取第一条
  │         → MetadataManager.generate_nfo(tmdb_id, media_type, nfo_path)  覆盖写
  │
  ├─ [refix_poster=True]
  │    └─ MetadataManager.download_poster(tmdb_id, media_type, metadata_dir)
  │         ⚠️ 注意：download_poster 内部有「已存在即跳过」逻辑
  │         需先 os.remove(poster.jpg) 再调用，实现强制覆盖
  │
  ├─ [refix_subtitle=True]
  │    └─ db.update_any_task_metadata(task_id, is_archive, sub_status="pending")
  │         字幕巡逻 cron 会在下次扫描时自动触发 perform_find_subtitles_task_sync
  │
  └─ db.update_any_task_metadata(
         task_id, is_archive,
         tmdb_id=new_tmdb_id, imdb_id=new_imdb_id,
         title=new_title, year=new_year
     )
```

#### Response

```json
{
  "success": true,
  "task_id": 42,
  "rebuilt": {
    "nfo": true,
    "poster": true,
    "subtitle_reset": true
  },
  "title": "飞驰人生",
  "tmdb_id": 12345,
  "message": "补录完成：NFO ✅  海报 ✅  字幕已重置 ✅"
}
```

#### 路由注册（`router.py`）

