## 防线 1：后端核心链路排雷

### 问题 1.1 [CRITICAL] 异常吞噬 — 无日志记录的 Exception 捕获

**位置**：`backend/app/api/v1/endpoints/tasks/scrape_task.py:54`

```python
except Exception:
    pass  # ❌ 无任何日志记录，异常被完全吞噬
```

**影响**：调用方无法诊断失败原因，导致隐性故障。

**建议**：
```python
except Exception as e:
    logger.error(f"[ERROR] 操作失败: {e}", exc_info=True)
    raise  # 或根据业务逻辑决定是否继续
```

**扫描结果**：共 15 处 `except Exception` 捕获，其中 1 处无日志（第 54 行），其余均有日志记录。

**位置**：`backend/app/api/v1/endpoints/tasks/scrape_task.py:975-976`（`_nuclear_clean_directory` 函数内）

```python
for f in resolved_meta.iterdir():

    _protected_names = {"tvshow.nfo", "movie.nfo", ...}  # ❌ 每次循环都重新创建
```

```python
_protected_names = {"tvshow.nfo", "movie.nfo", "poster.jpg", ...}
for f in resolved_meta.iterdir():
    ...
```

### 问题 1.6 [MEDIUM] 变量作用域污染 — 临时变量使用下划线前缀但未隔离

**位置**：`backend/app/api/v1/endpoints/tasks/scrape_task.py:1175-1185`

```python
_tree = ET.parse(...)  # 临时变量
_root = _tree.getroot()
_existing_imdb = ...
_uid = ...
```

**影响**：虽然使用了下划线前缀表示临时变量，但在函数作用域内仍可能与其他代码冲突。

**建议**：使用更具体的前缀或将逻辑提取为独立函数：
```python
def _check_nfo_imdb_id(nfo_path: Path, target_imdb_id: str) -> bool:
    """检查 NFO 中的 IMDb ID 是否与目标一致"""
    try:
        tree = ET.parse(nfo_path)
        root = tree.getroot()
        existing_id = (root.findtext("imdbid") or "").strip()
        if not existing_id:
            for uid in root.findall("uniqueid"):
                if uid.get("type") == "imdb":
                    existing_id = (uid.text or "").strip()
                    break
        return existing_id == target_imdb_id.strip()
    except Exception as e:
        logger.warning(f"NFO 解析失败: {e}")
        return False
```

## 防线 3：数据库与模型一致性

### 问题 3.1 [MEDIUM] DRY 原则破坏 — 格式后缀在两处定义

**位置**：
- `backend/app/infra/database/default_config.py:236-238`
- `backend/app/models/domain_system.py:132-133`

```python
# default_config.py
"supported_video_exts": ".mkv, .mp4, .avi, ...",

# domain_system.py
supported_video_exts: str = ""  # ❌ 空值，但应该从 DEFAULT_CONFIG 读取
```

**影响**：虽然已通过 `get_all_config()` 合并逻辑规避了硬编码重复，但 Pydantic 模型中的空值仍然是"谎言"。

**建议**：在 Pydantic 模型中添加 Field 默认值工厂或文档说明：
```python
from pydantic import Field

class SystemSettings(BaseModel):
    supported_video_exts: str = Field(
        default="",
        description="支持的视频格式后缀，由后端 get_all_config 从 DEFAULT_CONFIG 合并"
    )
    supported_subtitle_exts: str = Field(
        default="",
        description="支持的字幕格式后缀，由后端 get_all_config 从 DEFAULT_CONFIG 合并"
    )
```

**位置**：多处

- `task_repo.py` 中：`clean_name` 字段
- `scrape_task.py` 中：`new_title` 变量

```python
# Step 3.4
```
