## 一、现状全景

### 1.1 前端调用链

```
MediaTable
  └─ RebuildDialog（NFO 模式）
       └─ handleNuclearExecute()
            └─ onConfirm({ tmdb_id, media_type, nuclear_reset: true })
                 ↓
       MediaTable.handleRebuildConfirm(params)
            └─ onRebuild({
                 task_id, is_archive, media_type,
                 refix_nfo/poster/subtitle,
                 tmdb_id,
                 nuclear_reset: true
               })
                 ↓
       api.manualRebuild({ ...params })
            └─ POST /tasks/manual_rebuild
```

**关键发现**：`handleRebuildConfirm` 的参数类型目前是：
```ts
params: { tmdb_id?: number; media_type: string; nuclear_reset: boolean }
```

**无 `season` / `episode` 字段**，无法从前端传递季集信息。

### 1.2 Task 类型定义（types/index.ts）

```ts
export interface Task {
  season?: number | null;
  episode?: number | null;
  media_type: 'movie' | 'tv';
  // ...其他字段
}
```

`Task` 已有 `season`/`episode`，但在 `RebuildDialogProps.onConfirm` 和 `ManualRebuildRequest` 均未透传用户输入的覆盖值。

### 1.3 后端 ManualRebuildRequest（scrape_task.py:880）

```python
class ManualRebuildRequest(BaseModel):
    task_id: int
    is_archive: bool = True
    tmdb_id: Optional[int] = None
    keyword_hint: Optional[str] = None
    media_type: str = "movie"
    refix_nfo: bool = True
    refix_poster: bool = True
    refix_subtitle: bool = True
    nuclear_reset: bool = False
    # ❌ 无 season / episode 字段
```

### 1.4 核级重置触发条件（scrape_task.py:1150）

```python
if body.nuclear_reset and new_tmdb_id:
    video_files = [f for f in Path(metadata_dir).iterdir()  # ← 仅一层，不递归
                   if f.is_file() and f.suffix.lower() in _db_video_exts]
    if not video_files:
        raise HTTPException(status_code=400, detail="[NUCLEAR] 目录内未找到视频本体文件")  # ← 400 根因
```

**TV 剧集 400 根因**：TV 剧集的视频通常在 `Series/Season X/ep.mkv` 子目录，`Path.iterdir()` 只扫描一层，无法找到视频文件，触发 400 熔断。

### 1.5 重命名函数现状（scrape_task.py:~1000）

```python
def _rename_video_file(video_path, title, year, media_type,
                       season=None, episode=None) -> str:
```

已支持 `season`/`episode` 参数，但当前从数据库读取：
```python
task_season  = task_record.get("season")
task_episode = task_record.get("episode")
```

**缺陷**：若 DB 中 season/episode 错误或缺失，无法通过前端输入覆盖。

## 三、修改方案（待点火）

### Phase 1 — 后端：ManualRebuildRequest 扩展

```python
class ManualRebuildRequest(BaseModel):
    # ... 现有字段 ...
    season: Optional[int] = None   # 新增：用户强制指定季数
    episode: Optional[int] = None  # 新增：用户强制指定集数
```

### Phase 2 — 后端：递归视频探测（TV 400 根治）

```python
# 原来（只扫一层）:
video_files = [f for f in Path(metadata_dir).iterdir()
               if f.is_file() and f.suffix.lower() in _db_video_exts]

# 修改后（先扫一层，TV 任务无结果时递归 rglob）:
video_files = [f for f in Path(metadata_dir).iterdir()
               if f.is_file() and f.suffix.lower() in _db_video_exts]
if not video_files and body.media_type == 'tv':
    video_files = [f for f in Path(metadata_dir).rglob('*')
                   if f.is_file() and f.suffix.lower() in _db_video_exts]
```

### Phase 3 — 后端：season/episode 强制覆盖

```python
# 原来（从 DB 读取）:
task_season  = task_record.get("season")
task_episode = task_record.get("episode")

# 修改后（请求值优先，DB 值兜底）:
task_season  = body.season  if body.season  is not None else task_record.get("season")
task_episode = body.episode if body.episode is not None else task_record.get("episode")
```

### Phase 4 — 前端：RebuildDialogProps 扩展

```ts
// onConfirm 签名扩展
onConfirm: (params: {
  tmdb_id?: number;
  media_type: string;
  nuclear_reset: boolean;
  season?: number;    // 新增
  episode?: number;   // 新增
}) => void;
```

### Phase 5 — 前端：RebuildDialog TV 模式新增输入框

当 `mediaType === 'tv'` 时，在弹窗搜索区下方显示：
- **Season 输入框**（数字，默认 `task.season ?? 1`）
- **Episode 输入框**（数字，可选，默认 `task.episode ?? ''`）

### Phase 6 — 前端：MediaTable.handleRebuildConfirm 透传

```ts
const handleRebuildConfirm = useCallback(async (
  params: { tmdb_id?: number; media_type: string; nuclear_reset: boolean;
            season?: number; episode?: number }
) => {
  await onRebuild({
    ...,
    season: params.season,
    episode: params.episode,
  });
}, [...]);
```

### Phase 7 — 前端：api.manualRebuild 透传

```ts
// api.ts manualRebuild params 类型补充 season/episode
body: JSON.stringify({
  ...,
  season: params.season ?? undefined,
  episode: params.episode ?? undefined,
})
```

## 五、风险评估

| 风险 | 等级 | 应对 |
|------|------|------|
| `rglob` 扫出多个视频（多集目录）| LOW | 取第一个，与现有 `iterdir` 逻辑一致 |
| season=None 时 `_rename_video_file` 行为 | ZERO | 函数已有 None 兜底 |
| 电影模式误传 season | ZERO | `body.media_type != 'tv'` 时不触发递归 |
| api.ts 类型不兼容 | LOW | 扩展 params 类型即可 |
