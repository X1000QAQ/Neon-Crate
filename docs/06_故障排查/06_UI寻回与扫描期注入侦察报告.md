## 二、侦察任务 1：UI 寻回与裂图追溯

### 2.1 TMDB/IMDb ID 标签现状

| 字段 | Task 类型 | 渲染代码 | 实际展示 |
|------|----------|---------|----------|
| `tmdb_id` | `number?` ✅ | 条件渲染 TMDB 链接 ✅ | 有值时显示 ✅ |
| `imdb_id` | `string?` ✅ | **完全没有** ❌ | **永远不显示** ❌ |

`TaskRow` 外链区代码（`MediaTable.tsx` ~175 行）：
```tsx
<div className="flex items-center gap-1 flex-shrink-0">
  {task.tmdb_id && (
    <a href={`https://www.themoviedb.org/.../${task.tmdb_id}`}>TMDB</a>
  )}
  {/* ⚠️ 无任何 IMDb 渲染代码 */}
</div>
```

**根因**：重构时 IMDb 外链按钮被删除未恢复。

### 2.2 裂图根因分析

**根因（最可能）：`/placeholder-poster.jpg` 不存在于 `frontend/public/`**

- Pending 任务：`local_poster_path=null`, `poster_path=null`
- `getPosterUrl` 返回 `'/placeholder-poster.jpg'`
- `SecureImage` Rule 1：直接 `setBlobUrl('/placeholder-poster.jpg')`
- 原生 `<img src="/placeholder-poster.jpg">`
- 若 `frontend/public/placeholder-poster.jpg` 不存在 → 浏览器 404 → **裂图**

**其他可能根因**：
- `local_poster_path` 指向的文件已被删除 → 后端 404 → `setError(true)` → `fallback` 渲染（TaskRow 有 fallback，显示图标而非裂图）
- `poster_path` 为 TMDB 外部 URL 且 CDN 不可达 → fetch 失败 → `setError(true)` → fallback

**验证方法**：
```bash
ls /home/x1000qaq/test/Neon-Crate/frontend/public/placeholder-poster.jpg
```

## 三、侦察任务 2：扫描期注入可行性分析

### 3.1 现有 NFO 相关代码

`scan_task.py` 「失忆救援+白嫖逻辑」（~346 行）已有：
- ✅ 季目录回溯（`Season X` 正则）
- ✅ 从父目录名提取 `clean_name`/`year`
- ✅ 绑定本地 `poster.jpg`
- ❌ **不读 NFO**，不提取 `tmdb_id`/`imdb_id`

### 3.2 数据库兼容性分析（关键障碍）

`task_repo.insert_task` 的 INSERT SQL **不含 `tmdb_id`/`imdb_id` 列**：

```sql
-- 当前 SQL（task_repo.py ~78 行）
INSERT INTO tasks
(path, file_name, clean_name, type, status, year,
 season, episode, local_poster_path, target_path)
VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?, ?)
-- ⚠️ tmdb_id 和 imdb_id 完全不在列表中！
```

`tasks` 表 schema 中 `tmdb_id TEXT` 和 `imdb_id TEXT` 列**已存在**（`db_manager.py` ~140 行），只是 INSERT 语句没写进去。

**结论**：无需修改表结构，只需扩展 `insert_task` 的 SQL 语句，加入 `tmdb_id`/`imdb_id` 两列即可。

### 3.3 性能风险分析

| 风险 | 等级 | 说明 |
|------|------|------|
| 磁盘 I/O（find_nfo 寻址） | 低 | `Path.is_file()` 调用，~2 次文件系统 stat，微秒级 |
| XML 解析（parse_nfo） | 低 | `xml.etree.ElementTree` 解析小文件（<500KB），毫秒级 |
| 扫描整体耗时影响 | 低 | 只在 `default_status=="archived"` 的 library 文件上触发，download 文件不受影响 |
| NFO 不存在时的影响 | 无 | `find_nfo` 返回 `None` 时直接跳过，降级为白嫖逻辑 |
| 并发风险 | 无 | `perform_scan_task_sync` 已有 `_scan_entry_lock` 串行化保护 |

**总结**：性能影响可忽略，每个文件额外开销 < 5ms。

### 3.4 POC 伪代码

```python
# scan_task.py — 失忆救援块中注入 NFO 解析（Shift-Left POC）
# 位置：if default_status == "archived": 块内，白嫖逻辑之后

if default_status == "archived":
    # ... 现有白嫖逻辑（目录名提取 + 海报绑定）...

    # ── NFO Shift-Left 注入 ───────────────────────────────
    # 在白嫖逻辑之后尝试读取 NFO，优先级：NFO > 目录名
    try:
        from app.services.metadata.nfo_parser import find_nfo, parse_nfo
        _nfo_path = find_nfo(file_path)  # 同级 + Season 回溯
        if _nfo_path:
            _nfo = parse_nfo(_nfo_path)
            if _nfo.get("tmdb_id"):
                task_data["tmdb_id"] = _nfo["tmdb_id"]   # str，insert_task 接受 str
                task_data["imdb_id"] = _nfo.get("imdb_id") or None
                # NFO 的 title/year 优先级高于目录名
                if _nfo.get("title"):
                    task_data["clean_name"] = _nfo["title"]
                if _nfo.get("year"):
                    task_data["year"] = _nfo["year"]
                logger.info(
                    f"[SCAN] [NFO Shift-Left] 提前获取元数据: "
                    f"tmdb={task_data['tmdb_id']}, title={task_data['clean_name']}"
                )
    except Exception as _nfo_scan_err:
        logger.debug(f"[SCAN] [NFO Shift-Left] 跳过（{_nfo_scan_err}）")
    # ─────────────────────────────────────────────────────

# insert_task 需同步扩展 SQL（task_repo.py）：
# INSERT INTO tasks
# (path, file_name, clean_name, type, status, year,
#  season, episode, local_poster_path, target_path, tmdb_id, imdb_id)
# VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?)
```
