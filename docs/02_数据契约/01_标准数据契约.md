## 一、核心数据流转

```
原始文件名
  ↓ MediaCleaner.clean_and_extract()
  └─ {clean_name, year, season, episode, is_tv, is_ad}
  ↓ ScanEngine 入库
  └─ Task: {path, file_name, clean_name, year, season, episode, ...}
  ↓ 刮削流程（TMDBAdapter + MetadataManager + SmartLink）
  └─ Task: {title, year, season, tmdb_id, status='archived', ...}
```

```python
{

    'file_name': str,
    'size': int,            # 字节
    'clean_name': str,
    'year': int | None,
    'is_tv': bool,
    'season': int | None,
    'episode': int | None

```

### filename_clean_regex 默认 15 条规则

| # | 用途 |
|---|------|
| 1 | 分辨率（1080p/4K/BluRay）|
| 2 | 编码格式（x264/HEVC/AAC）|
| 3 | 方括号技术标签 |
| 4 | 花括号标签 |
| 5 | 广告词 |
| 6 | 音视频特性（HDR/IMAX）|
| 7 | 语言标签（CHT/Cantonese）|
| 8 | 制作组后缀 |
| 9 | 年份清除 |
| 10-14 | 季集标签（5条）|
| 15 | 动漫番剧集数格式 |

用户可在 RegexLab 增删，`POST /tasks/settings/reset {target: 'regex'}` 一键重置。

## 三、前端类型定义契约

### Task 接口（types/index.ts）

```typescript
export interface Task {
  id: number;
  file_path: string;
  file_name?: string;
  media_type: 'movie' | 'tv';          // 对应后端 type 字段
  status: 'pending' | 'scraped' | 'failed' | 'archived' | 'ignored';
  tmdb_id?: number | string;
  imdb_id?: string;
  title?: string;
  year?: number | string;              // 宽松类型，前端防御
  poster_path?: string;
  local_poster_path?: string;
  target_path?: string;
  sub_status?: 'pending' | 'failed' | 'missing' | 'scraped' | 'found';  // ⚠️ 严禁使用 'success'
  season?: number | null;              // 整数类型，非字符串
  episode?: number | null;
  created_at: string;                  // 后端 NULL 兜底为空字符串
}
```

**关键约定**：
- `status` 枚举无 `'success'`，严禁使用模糊终态
- `sub_status` 枚举永久移除 `'success'`，合法终态仅为 `'scraped'`（刮削完成）和 `'found'`（本地发现）
- `season` / `episode` 必须为整数，不是字符串
- `created_at` 后端 media_router.py 已在查询层将 NULL 兜底为空字符串，防止前端 `.substring(null)` 崩溃

> 🚨 **[2026-03-15 契约变更]** `sub_status` 中的 `'success'` 已被永久移除。
> 
> **变更原因**：`'success'` 语义模糊，违反 Neon Crate 命名规范（终态必须描述具体完成方式）。
> **替代方案**：
> - `'scraped'` — 通过 API 刮削完成（远程获取）
> - `'found'` — 本地发现已有字幕文件（本地白嫖）
> 
> **进度条影响**：`MediaTable.tsx` 的 `getProgress()` 已同步更新，100% 条件为 `ss === 'scraped' || ss === 'found'`。

## 五、API 数据类型约定

| 字段 | 后端 DB | 前端 | 约定 |
|------|---------|------|------|
| 媒体类型 | `type` | `media_type` | `'movie'` 或 `'tv'` |
| 任务状态 | `status` | `status` | 无 `'success'` |
| 字幕状态 | `sub_status` | `sub_status` | 合法终态：`'scraped'` / `'found'`，永久移除 `'success'` |
| 季/集号 | INTEGER | number \| null | 整数，非字符串 |
| 年份 | TEXT | number \| string | 前端宽松兼容 |
| created_at | TEXT（可NULL）| string（非空）| 后端查询层兜底空字符串 |

- [ ] 季号补充逻辑在归档时执行并回写数据库
