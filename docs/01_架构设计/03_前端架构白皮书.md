## 一、系统定位与设计哲学

Neon Crate 前端是「数据容器编排引擎的工业级霓虹控制面板」。

| 原则 | 具体体现 |
|------|----------|
| **类型即契约** | `types/index.ts` 唯一类型来源，与后端 Pydantic 严格对齐 |
| **单一数据入口** | 所有 HTTP 调用只能经过 `lib/api.ts`，禁止裸 `fetch` |
| **国际化优先** | 所有 UI 字符串通过 `t(key)` 获取，严禁硬编码 |
| **错误防线纵深** | `apiError.ts` → `NeuralLinkAlert` → `error.tsx` 三层递进 |
| **前端不存业务规则** | 正则/AI 规则来自后端数据库，前端只展示与提交 |

## 五、核级重构弹窗（RebuildDialog.tsx）— v1.0.1-Enhanced

### 开门重置器（防止幽灵记忆）

**业务链路**：
```
1. 监听 open 状态与 task.id 变化 -> 
2. 触发弹窗展示 -> 
3. 彻底清空上一轮的幽灵记忆 (keyword, results, selected) -> 
4. 重新灌入当前 task 的基础参数
```

**实现**：
```typescript
useEffect(() => {
  if (open) {
    setMediaType(task.media_type || 'movie');
    setKeyword(task.title || '');
    setResults([]);
    setSelected(null);
    setSearchErr(null);
    setSeason(task.season ?? 1);
    setEpisode(task.episode ?? '');
  }
}, [open, task.id]);
```

**防止 Dialog 幽灵记忆陷阱**：
- `if (!open) return null` 只隐藏不卸载，导致状态被永久冻结
- 此 useEffect 确保每次打开新任务时，状态都被重置

### 核级重构执行流程

**业务链路**：
```
1. 用户点击「执行核级重构」按钮 -> 
2. 验证 tmdb_id 非空 -> 
3. 构建请求参数（task_id, is_archive, nuclear_reset=true）-> 
4. 调用 api.rebuildTask()
  └─► POST /tasks/manual_rebuild
        └─► 后端处理（见 @docs/02_数据契约/02_API规范与鉴权.md）
              └─► 返回重建结果 {success, rebuilt: {nfo, poster, subtitle, nuclear}}
                    └─► 前端显示成功提示 + 刷新列表
```

### 搜索业务链路

**业务链路**：
```
1. 用户输入关键词 -> 
2. 防抖处理（500ms）-> 
3. 调用 api.searchTmdb()
  └─► POST /tasks/search_tmdb
        └─► 返回候选列表 [{tmdb_id, title, year, ...}, ...]
              └─► 前端展示候选列表
                    └─► 用户选择一个候选 -> 
                        更新 selected 状态 -> 
                        显示「执行核级重构」按钮
```

## 六、双轨路径显示（MediaTable.tsx）— v1.0.1-Enhanced

### 路径设计

**前世（SRC）**：`file_path`
- 含义：文件在下载源中的原始路径
- 示例：`/downloads/The.Matrix.1999.1080p.BluRay.x264.mkv`
- 用途：追溯文件来源、重新扫描、调试

**今生（DST）**：`target_path`
- 含义：文件入库后的目标路径（已重命名、已移动）
- 示例：`/media/movies/The Matrix (1999)/The Matrix (1999).mkv`
- 用途：媒体库寻址、播放器调用、元数据关联

### 前端 UI 展示

**业务链路**：
```
1. 检查 task.target_path 是否存在 -> 
2. 若存在则显示为"入库路径"（置于上方）-> 
3. 使用 t('path_dst') 获取多语言标签 -> 
4. 显示 task.file_path（下载源中的原始路径）-> 
5. 使用 t('path_src') 获取多语言标签 -> 
6. 用于追溯文件来源
```

**实现**：
```typescript
// 🚀 今生：目标入库路径 (置于上方)
{task.target_path && (
  <p className="text-cyber-cyan/30 text-xs truncate mt-1" title={task.target_path}>
    <span className="text-cyber-cyan/50 font-mono mr-1">{t('path_dst')}:</span>
    {task.target_path}
  </p>
)}

// 前世：原始绝对路径 (置于下方)
<p className="text-cyber-cyan/40 text-xs truncate mt-0.5" title={task.file_path}>
  <span className="text-cyber-cyan/60 font-mono mr-1">{t('path_src')}:</span>
  {task.file_path}
</p>
```

## 七、性能优化（useMemo）— v1.0.1-Enhanced

### MediaTable 性能优化

**业务链路**：
```
1. 使用 useMemo 缓存计算结果 -> 
2. 避免不必要的重新渲染 -> 
3. 减少 DOM 更新频率 -> 
4. 提升列表滚动性能
```

**实现**：
```typescript
const displayTitle = useMemo(() => {
  if (task.title && task.year) {
    return `${task.title} (${task.year})`;
  }
  return task.clean_name || task.file_name || 'Unknown';
}, [task.title, task.year, task.clean_name, task.file_name]);
```

### NetworkContext 状态管理

**业务链路**：
```
1. 监听 window.online/offline 事件 -> 
2. 更新 isOnline 状态 -> 
3. 触发 UI 断链通知 -> 
4. 自动重试机制（指数退避）
```

// 1. 注册追踪：setTimeout ID 集中存入 useRef 数组
// -> 2. 静默清理：useEffect cleanup 遍历 clearTimeout
// -> 3. 预防报错：防止卸载后更新已不存在的 state
const timerRefs = useRef<ReturnType<typeof setTimeout>[]>([]);
useEffect(() => { return () => { timerRefs.current.forEach(clearTimeout); }; }, []);
```

## 九、数据获取层（lib/api.ts）

### secureFetch 核心拦截器

```
// 异步链路治理：
// 1. 注入超时 AbortSignal：timeoutMs 后自动 abort
// -> 2. 超时物理掐断：AbortError → REQUEST_TIMEOUT（区分超时与用户主动取消）
// -> 3. 复航信号：非 5xx 自动解除全局网络警告横幅
```

### api.chat 的 AbortController 全链路

```
前端 AiSidebar：
  1. 注入信号：创建新 AbortController，signal 传入 fetch
  -> 2. 物理掐断：连续发送时 abort() 旧请求，浏览器切断连接
  -> 3. 静默处理：catch AbortError 直接 return，不展示错误气泡

后端 agent.py：
  异步链路治理步骤 3：Uvicorn 感知连接断开注入 CancelledError
  → 释放 LLM 推理资源，必须 re-raise
```

## 十、状态管理

### NetworkContext — 全局断链状态

```
NetworkProvider 挂载 → 注册 window.__setLinkDown = setIsLinkDown

## 十一、AI 交互前端舱（AiSidebar.tsx）

### 候选列表解析

后端 DOWNLOAD 意图返回模糊结果时，响应末尾附加：
```
引导语 __CANDIDATES__["片名 (年份)", ...]
```
`parseCandidates()` 以 `__CANDIDATES__` 为分隔符切割，渲染为快捷选择按钮。

### 候选封死机制

- 只有**最后一条含候选的消息**按钮处于激活状态
- 任何新消息发送时立即封死所有旧候选（`disabled` + 变灰）
- 候选按钮点击 + `handleSendText` 双重保险，防止重复触发

## 十二、错误处理体系

```
第一层：lib/apiError.ts
  → ApiError（code + status）+ notifyLinkDown()

第二层：NeuralLinkAlert
  → 捕获 5xx / 超时 / 断网 → 全屏弹窗
  → 「重连」→ setLinkDown(false) + reload()

第三层：app/error.tsx
  → Next.js Error Boundary → 赛博朋克风格错误页
```

| ApiError code | 触发场景 | status |
|---------------|----------|--------|
| `REQUEST_TIMEOUT` | 15s 超时 | 408 |
| `NETWORK_ERROR` | 网络不可达 | 0 |
| `UNAUTHORIZED` | 401 响应 | 401 |
| `SERVER_ERROR` | 5xx 响应 | 500+ |

## 十四、架构约束与红线

| 约束 | 说明 |
|------|------|
| 禁止裸 fetch | 所有 HTTP 调用必须经过 `lib/api.ts` |
| 禁止硬编码类型 | Task.status / sub_status 必须从 `types/index.ts` 引用 |
| 禁止硬编码正则 | 清洗规则来自后端 `filename_clean_regex` |
| 禁止硬编码字符串 | 所有 UI 文本通过 `t(key)` 获取 |
| 禁止离散色值 | 颜色统一使用 CSS 变量 |
| 1+1 媒体库约束 | 电影媒体库 ≤ 1，剧集媒体库 ≤ 1 |
| 图片路径必须代理 | 本地图片经 `SecureImage` 组件代理 |
| i18n key 必须双语 | zh 和 en 字典必须同步添加 |
