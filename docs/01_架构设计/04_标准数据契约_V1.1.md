## 文档概述

本文档定义 Neon Crate 系统中所有模块必须遵守的标准数据契约，确保扫描、清洗、AI 识别、TMDB 刮削、数据库存储等各环节的参数命名和语义完全一致。

**核心原则**:
- 数据库是唯一真相源（`filename_clean_regex`、`expert_archive_rules` 等）
- 前后端双端共同遵守本契约，任何一端不得以"兼容性"为由放宽约束
- 所有过滤规则必须来自数据库，严禁硬编码
- 新增功能必须遵循本契约，如需扩展字段必须在本文档中完成评审与更新

## 一、核心数据流转

```
原始文件名
  ↓
MediaCleaner.clean_and_extract()
  ├─ 数据库 filename_clean_regex（15条默认规则）
  ├─ 固定内置正则（年份/季集提取，不参与过滤）
  └─ 返回：{clean_name, year, season, episode, is_tv, is_ad}
  ↓
ScanEngine 入库
  └─ Task: {path, file_name, clean_name, year, season, episode, is_tv, ...}
  ↓
刮削流程
  ├─ AI 识别（可选）
  ├─ TMDB 搜索（使用 clean_name）
  └─ 归档路径构建（season 从路径补充校正）
  ↓
数据库最终存储
  └─ Task: {title, year, season, episode, tmdb_id, status, ...}
```

## 二、扫描阶段数据契约

### 2.1 `MediaCleaner.clean_and_extract()` 返回值

```python
{
    'clean_name': str,      # 纯净名称（无年份、无技术标签）
    'year': int | None,     # 年份（1900-2099）
    'season': int | None,   # 季数（仅剧集）
    'episode': int | None,  # 集数（仅剧集）
    'is_tv': bool,          # 是否为剧集
    'is_ad': bool           # 是否为广告文件
}
```

**字段说明**:

| 字段 | 类型 | 说明 | 示例 |
|---|---|---|---|
| `clean_name` | string | 最通用、最纯净的作品名称 | `"Dune Part Two"` |
| `year` | int\|None | 发行年份 | `2024` |
| `season` | int\|None | 季数（仅剧集） | `3` |
| `episode` | int\|None | 集数（仅剧集） | `10` |
| `is_tv` | bool | 是否为剧集 | `true` |
| `is_ad` | bool | 是否为广告/废片 | `false` |

### 2.2 `ScanEngine.scan_directory()` 返回值

```python
{
    'path': str,            # 文件完整路径
    'file_name': str,       # 原始文件名
    'size': int,            # 文件大小（字节）
    'clean_name': str,      # 清洗后名称
    'year': int | None,     # 年份
    'is_tv': bool,          # 是否为剧集
    'season': int | None,   # 季数
    'episode': int | None   # 集数
}
```

### 2.3 数据库 `filename_clean_regex` 规则格式

**15 条默认规则**（每行一条，`#` 开头为注释）：

| # | 用途 | 示例 |
|---|---|---|
| 1 | 分辨率过滤 | `1080p`, `4K`, `BluRay` |
| 2 | 编码格式过滤 | `x264`, `HEVC`, `AAC` |
| 3 | 方括号技术标签 | `[HbT]`, `[SubsPlease]` |
| 4 | 花括号标签 | `{tag}` |
| 5 | 广告词过滤 | 广告相关词汇 |
| 6 | 音视频特性 | `HDR`, `10bit`, `IMAX` |
| 7 | 语言标签 | `中英`, `CHT`, `Cantonese` |
| 8 | 制作组后缀 | `-TEAM` 末尾 |
| 9 | 年份过滤 | 片名中的年份 |
| 10-14 | 季集标签 | `S01E01`, `Season`, `1x01`, `EP01`, `第X集` |
| 15 | 动漫番剧集数 | `- 28 [` 格式 |

**用户可在前端 RegexLab 自由增删，重启后端生效。**

## 三、刮削阶段数据契约

### 3.1 TMDB 搜索参数

**搜索关键字来源**：`clean_name`（来自 MediaCleaner 或 AI 识别）

```python
# 后端 tasks.py 中的刮削流程
search_query = ai_result.get("clean_name", clean_name)  # ✅ 标准做法
year_param = ai_result.get("year", "")                  # 可选辅助参数
final_type = ai_result.get("type", type_hint)           # movie / tv
```

**禁止事项**:
- ❌ 使用 `query`、`media_name` 等旧变量名
- ❌ 将原始文件名透传给 TMDB
- ❌ 在 `clean_name` 中包含年份

### 3.2 TMDB 搜索结果匹配策略

```
results = scraper.search_tv(query, year)
  ├─ 优先：original_name == query（严格）
  ├─ 其次：name == query（严格）
  ├─ 宽松：去末尾集号后再匹配（"The Boys 02" → "The Boys"）
  └─ 兜底：results[0]
```

### 3.3 季号补充与回写

```python
# 归档构建路径时
season_num = task.get("season") or 1
if season_num == 1:
    # 从 Path(file_path).parts 中搜索 Season XX 目录名
    # 若找到非1的季号 → 覆盖 season_num

# 写入数据库
db.update_task_title_year(task_id, title, year, season=season_num)
```

## 四、前端类型定义契约

### 4.1 `Task` 接口（`frontend/types/index.ts`）

```typescript
export interface Task {
  id: number;
  file_path: string;
  file_name?: string;
  media_type: 'movie' | 'tv';
  status: 'pending' | 'scraped' | 'failed' | 'archived' | 'ignored';
  tmdb_id?: number | string;
  imdb_id?: string;
  title?: string;
  year?: number | string;
  poster_path?: string;
  local_poster_path?: string;
  target_path?: string;
  sub_status?: 'pending' | 'scraped' | 'failed' | 'missing';
  season?: number | null;
  episode?: number | null;
  created_at: string;
}
```

**字段说明**:

| 字段 | 类型 | 说明 |
|---|---|---|
| `media_type` | `'movie'\|'tv'` | 媒体类型（对应后端 `type`）|
| `status` | 枚举 | 任务状态（已修正，无 `'success'`）|
| `sub_status` | 枚举 | 字幕状态（已修正，无 `'success'`）|
| `season` | `number\|null` | 季数（整数类型）|
| `episode` | `number\|null` | 集数（整数类型）|
| `year` | `number\|string` | 年份（宽松类型，前端防御）|

## 六、API 契约

### 6.1 任务查询

**请求**: `GET /tasks?page=1&page_size=20&status=all&media_type=all&search=keyword`

**响应**:
```json
{
  "tasks": [Task],
  "total": 156,
  "page": 1,
  "page_size": 20
}
```

### 6.2 设置读写

**读取**: `GET /tasks/settings` → `SettingsConfig`

**保存**: `POST /tasks/settings` + `SettingsConfig` 请求体

**重置**: `POST /tasks/settings/reset` + `{"target": "ai" | "regex"}`

### 6.3 触发任务

| 方法 | 接口 | 说明 |
|---|---|---|
| `triggerScan()` | `POST /tasks/scan` | 物理扫描 |
| `triggerScrapeAll()` | `POST /tasks/scrape_all` | 全量刮削 |
| `triggerFindSubtitles()` | `POST /tasks/find_subtitles` | 字幕查找 |

## 七、国际化契约

所有 UI 文本必须使用 i18n key，严禁硬编码字符串。详见 `frontend/lib/i18n.ts` 和 `docs/04_前端模块手册/03_lib/i18n.md`。

**关键 key**:

| key | 中文 | 英文 |
|---|---|---|
| `ai_scan_triggered` | 扫描任务已在后台启动 | Scan task started in background |

| `ai_subtitle_triggered` | 字幕查找任务已在后台启动 | Subtitle search task started in background |
| `op_deleting` | 删除中... | Deleting... |

## 九、合规性检查清单

在提交代码前，请确认以下事项：

- [ ] 所有过滤正则来自数据库 `filename_clean_regex`，无硬编码
- [ ] TMDB 搜索使用 `clean_name`，无 `query`/`media_name`
- [ ] 前端 `Task` 接口使用标准字段，`season`/`episode` 为整数
- [ ] 所有 UI 文本使用 i18n key，无硬编码字符串
- [ ] 后端 `expert_archive_rules` 正确注入 System Prompt
- [ ] 季号补充逻辑在归档时执行，并回写数据库
- [ ] 年份不包含在 `clean_name` 中
- [ ] 剧集使用剧集名而非单集名
