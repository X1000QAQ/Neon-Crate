## 图表 2：单飞高并发防御阵列（Singleflight Concurrency Shield）

> 场景：前端 **500 个并发**请求 `/public/image` 拉取海报。目标：在 TTL 内存缓存失效时，**仅允许 1 个协程**访问 DB，其余请求优先走 **stale-while-revalidate**（用旧缓存先放行）。

```mermaid
sequenceDiagram
  autonumber
  participant FE as Frontend (500 并发)
  participant API as FastAPI /public/image (async)
  participant Cache as TTL Cache (managed_paths)
  participant Lock as asyncio.Lock (singleflight)
  participant DB as SQLite (managed_paths)
  participant FS as FileResponse

  FE->>API: GET /public/image?path=...
  API->>Cache: 读取 managed_paths 缓存（检查 TTL）
  alt TTL 命中（fresh）
    Cache-->>API: 返回 allowed_paths（fresh）
    API->>FS: Path 校验通过 → 直接 FileResponse
    FS-->>FE: 200 image bytes
  else TTL 过期（stale）
    Cache-->>API: 返回 allowed_paths（stale / 可能为空）
    par 499 个请求（stale-while-revalidate）
      API->>FS: 使用 stale allowlist 先放行（或降级策略）
      FS-->>FE: 200 image bytes（低延迟）
    and 1 个请求触发 Singleflight
      API->>Lock: 尝试 acquire()
      Lock-->>API: ✅ 获得锁（唯一放行）
      API->>DB: to_thread.run_sync 查询 managed_paths
      DB-->>API: 返回最新 allowed_paths
      API->>Cache: 更新缓存值 + 写入 cache_at（刷新 TTL）
      API->>Lock: release()
      API->>FS: 继续 Path 校验 → FileResponse
      FS-->>FE: 200 image bytes
    end
  end
```
