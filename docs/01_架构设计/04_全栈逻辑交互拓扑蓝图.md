## 图表 2：全量刮削引擎核心流程（决策树）

**场景**：`perform_scrape_all_task_sync` 的内部运转机制。

```mermaid
flowchart TD
    Start([🚀 开始全量刮削]) --> LockCheck{获取防重锁<br/>blocking=False}
    LockCheck -->|锁被占用| Reject["⚠️ 拦截并发请求<br/>直接返回"]
    Reject --> End1([结束])
    
    LockCheck -->|获取成功| APICheck{检查 TMDB<br/>API Key}
    APICheck -->|缺失| APIFail["❌ 记录错误<br/>返回"]
    APIFail --> End2([结束])
    
    APICheck -->|存在| GetTasks["📋 获取待刮削任务列表<br/>双表联查 tasks + media_archive"]
    GetTasks --> TaskLoop["🔄 逐个处理任务"]
    
    TaskLoop --> NFOCheck{查找本地<br/>NFO 文件}
    NFOCheck -->|找到| NFOParse["✅ NFO 短路拦截<br/>1. 解析 NFO<br/>2. 提取 TMDB/IMDb ID<br/>3. 跳过 AI 和 TMDB 搜索"]
    NFOParse --> NFOUpdate["📝 更新 DB 元数据"]
    NFOUpdate --> SubCheck1["🎁 字幕白嫖"]
    SubCheck1 --> NextTask1["➡️ 处理下一个任务"]
    
    NFOCheck -->|未找到| AIExtract["🧠 AI 提炼逻辑<br/>1. 正则清洗文件名<br/>2. 调用 AI Agent 分析<br/>3. 提取查询词/年份/类型"]
    AIExtract --> AIFallback{AI 返回<br/>有效结果?}
    AIFallback -->|失败| AIFallbackUse["⚠️ 降级使用<br/>正则清洗名"]
    AIFallback -->|成功| AIRefine["✅ 路径权威优先<br/>年份物理验证"]
    AIFallbackUse --> AIRefine
    
    AIRefine --> TMDBSearch["🔍 TMDB 搜索<br/>1. 根据类型调用搜索<br/>2. 剧集失败二次搜索<br/>3. 精确/宽松匹配"]
    TMDBSearch --> TMDBResult{搜索结果<br/>有效?}
    TMDBResult -->|无结果| SearchFail["❌ 搜索失败<br/>标记为 failed"]
    SearchFail --> NextTask2["➡️ 处理下一个任务"]
    
    TMDBResult -->|有结果| ExtractMeta["📊 提取元数据<br/>TMDB ID / IMDb ID"]
    ExtractMeta --> DupCheck{IMDb ID<br/>防重检测}
    DupCheck -->|重复| MarkIgnored["🚨 标记为 ignored<br/>继承同源海报"]
    MarkIgnored --> NextTask3["➡️ 处理下一个任务"]
    
    DupCheck -->|不重复| LibCheck{文件在<br/>library 路径?}
    LibCheck -->|是| InPlace["🏥 就地补录<br/>补充 NFO/海报<br/>不移动文件"]
    InPlace --> SubCheck2["🎁 字幕白嫖"]
    SubCheck2 --> NextTask4["➡️ 处理下一个任务"]
    
    LibCheck -->|否| Archive["🏠 归档全链路<br/>SmartLink → 移动文件<br/>写入元数据"]
    Archive --> SubCheck3["🎁 字幕白嫖"]
    SubCheck3 --> NextTask5["➡️ 处理下一个任务"]
    
    NextTask1 --> LoopEnd{所有任务<br/>处理完?}
    NextTask2 --> LoopEnd
    NextTask3 --> LoopEnd
    NextTask4 --> LoopEnd
    NextTask5 --> LoopEnd
    
    LoopEnd -->|否| TaskLoop
    LoopEnd -->|是| UpdateStats["📊 更新统计信息"]
    UpdateStats --> ReleaseLock["🔓 释放防重锁"]
    ReleaseLock --> End3([✅ 全量刮削完成])
```

## 图表 3：冷热双表数据流转（状态图）

**场景**：一个媒体文件从下载完成到最终入库的状态流转。

```mermaid
stateDiagram-v2
    [*] --> Pending: 文件下载完成<br/>扫描入库
    
    Pending --> Pending: 等待刮削<br/>status='pending'
    
    Pending --> Processing: 触发全量刮削
    
    Processing --> NFOShortcut: NFO 短路拦截?
    NFOShortcut --> Archived: ✅ 是<br/>直接归档
    NFOShortcut --> AIExtract: ❌ 否<br/>继续流程
    
    AIExtract --> TMDBSearch: AI 提炼完成
    
    TMDBSearch --> DupDetect: TMDB 搜索完成
    
    DupDetect --> Ignored: 🚨 IMDb 防重<br/>检测到重复
    
    DupDetect --> InPlace: 文件在 library?
    InPlace --> Archived: ✅ 是<br/>补充元数据
    
    DupDetect --> Archive: ❌ 否<br/>归档全链路
    Archive --> Archived: 移动文件<br/>写入元数据
    
    Archived --> SubtitleCheck: 字幕白嫖<br/>检测本地字幕
    SubtitleCheck --> SubSuccess: ✅ 发现本地字幕
    SubtitleCheck --> SubPending: ❌ 无本地字幕
    
    SubSuccess --> HotTable: 🔥 热表 tasks
    SubPending --> HotTable: 🔥 热表 tasks
    
    HotTable --> ColdArchive: 触发冷表归档
    
    ColdArchive --> ColdTable: ❄️ 冷表 media_archive
    
    Ignored --> IgnoredState: 🚨 热表 tasks<br/>status='ignored'
    
    Processing --> Failed: 🔴 处理失败
    Failed --> Retry: 用户手动重试
    Retry --> Processing
    
    ColdTable --> [*]: ✅ 最终状态<br/>媒体库入库完成
    IgnoredState --> [*]: 🚨 最终状态<br/>重复文件标记
```

## 🔄 关键业务流程总结表

| 流程 | 触发条件 | 关键操作 | 输出状态 |
|------|---------|---------|---------|
| **NFO 短路** | 本地存在 NFO 文件 | 解析 NFO → 提取元数据 → 跳过 AI/TMDB | archived |
| **AI 提炼** | NFO 不存在 | 正则清洗 → AI 分析 → 提取查询词 | processing |
| **TMDB 搜索** | AI 提炼完成 | 调用 TMDB API → 精确/宽松匹配 | 获得 TMDB/IMDb ID |
| **防重拦截** | 获得 IMDb ID | 检查 check_media_exists → 若重复标记 ignored | ignored |
| **就地补录** | 文件在 library 路径 | 补充 NFO/海报 → 不移动文件 | archived |
| **归档全链路** | 文件在下载目录 | SmartLink → 移动文件 → 写入元数据 | archived |
| **字幕白嫖** | 归档完成 | 检测本地字幕 → 若有标记 success | success/pending |
