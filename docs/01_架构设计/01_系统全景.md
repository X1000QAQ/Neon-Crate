## 一、系统定位

Neon Crate 是面向 NAS / 家庭媒体服务器场景的**全自动影音库管理引擎**。核心能力：

- **AI 总控神经**：自然语言指令 → LLM 意图路由 → 自动触发扫描/刮削/下载/字幕任务
- **物理文件编排**：扫描下载目录 → 清洗文件名 → TMDB 刮削元数据 → SmartLink 硬链接归档
- **数据持久化**：SQLite WAL 模式，管理所有媒体任务状态、配置、归档记录
- **安全网关**：JWT 无状态鉴权 + Fernet 加密配置
- **AIO 静态服务**：生产模式单端口、单进程托管前后端

## 四、核心数据流

### 4.1 全量归档链路

```
POST /scan
  └─► ScanEngine（目录遍历 + 格式/大小过滤）
        └─► MediaCleaner（数据库驱动正则清洗）
              └─► db.insert_task()（path UNIQUE 防重入库）

POST /scrape_all
  └─► TMDBAdapter.search_movie/tv()（精确匹配优先）
        └─► MetadataManager（NFO + 海报 + Fanart 生成）
              └─► SmartLink.create_link()（硬链接归档）
                    └─► db.archive_task()（写入 media_archive 冷表）
                          └─► _update_library_counts()（刷新统计缓存）

POST /find_subtitles
  └─► SubtitleEngine
        ├─► 本地字幕检测（已有则跳过，零 API 消耗）
        └─► OpenSubtitles API → {视频名}.ai.zh-CN.srt
```

### 4.2 AI 对话链路

```
AiSidebar.tsx
  └─► abortControllerRef.current.abort()  （掐断旧请求）
  └─► api.chat(message, signal)
        └─► POST /agent/chat
              └─► AIAgent.process_message()
                    ├─► LLMClient（意图识别，JSON 输出）
                    ├─► DOWNLOAD → ServarrClient（TMDB 侦察 + 精准下载）
                    └─► ACTION_SCAN/SCRAPE/SUBTITLE → 触发对应后台任务
```

### 4.3 认证链路

```
浏览器访问任意页面
  └─► AuthGuard.tsx（JWT 校验）
        ├─► 未初始化 → /auth/login（初始化模式）
        ├─► 无 Token → /auth/login（登录模式）
        └─► 通过 → 渲染主界面 + AiSidebar

POST /auth/login
  └─► CryptoManager.authenticate()（bcrypt 校验）
        └─► create_access_token()（JWT HS256，7天）
              └─► localStorage.token
```

### 4.4 并发防重链路（v1.0.0-Stable 新增）

```
每个任务模块独立的 threading.Lock
  1. 非阻塞获取锁（blocking=False）
  -> 2. 失败 → 立即返回，丢弃冗余请求，不排队不等待
  -> 3. 成功 → 执行任务，finally 块无条件释放锁（系统自愈）
```

## 五、模块职责速览

### 后端

| 模块 | 职责 | 禁止越界 |
|------|------|---------|
| `api/` | 接收 HTTP、参数验证、调用 Services、返回响应 | 禁止包含业务逻辑 |
| `services/` | 业务逻辑、外部 API 调用、文件操作 | 禁止直接操作数据库表结构 |
| `infra/` | 数据库连接、加密、配置管理 | 禁止包含业务规则 |
| `models/` | Pydantic 数据模型、类型定义 | 禁止包含方法逻辑 |

### 前端

| 模块 | 职责 |
|------|------|
| `app/` | Next.js App Router（页面路由、布局）|
| `components/` | React 组件（容器 + 展示分离）|
| `lib/api.ts` | 唯一合法 HTTP 出口（secureFetch + 鉴权注入）|
| `lib/i18n.ts` | 中英双语翻译字典（自研，< 100 行）|
| `types/index.ts` | 前端唯一类型来源，与后端 Pydantic 严格对齐 |
| `context/` | 全局状态（NetworkContext 断链通知）|

## 六、设计哲学

| 原则 | 具体体现 |
|------|----------|
| **数据库即真相源** | 所有正则规则、AI 规则均存储于 SQLite，严禁硬编码 |
| **默认拦截，显式豁免** | 所有 `/api/v1/*` 路由默认要求 JWT |
| **零依赖降级** | 各服务步骤独立容错，单步失败不崩溃整个任务 |
| **极简部署** | AIO 模式单进程、单端口，适合 NAS 场景 |
| **防重叠调度** | threading.Lock 防止同类任务并发执行 |
