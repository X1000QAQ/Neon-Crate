## 二、应用入口与生命周期（main.py）

### 2.1 启动序列

```
应用启动
  ├─► 日志系统初始化（RotatingFileHandler，10MB × 5）
  ├─► DatabaseManager 懒初始化
  │     ├─► SQLite WAL 模式
  │     ├─► 自动建表（tasks / media_archive / config）
  │     ├─► _migrate_plaintext_keys()   历史明文密钥迁移
  │     └─► _inject_ai_defaults()       注入 AI 规则 + 15条正则
  ├─► 路由注册（auth / public / api_router）
  ├─► 静态资源挂载（/api/v1/assets → /storage）
  ├─► AIO 前端挂载（static/ → /，html=True）
  ├─► SPA 404 回退处理器注册
  └─► asyncio.create_task(cron_scanner_loop())  后台巡逻启动
```

### 2.2 自动巡逻流水线（cron_scanner_loop）

```
每轮：读取 cron_enabled
  = False → 跳过
  = True  → 1. 物理扫描
          → 读取 auto_scrape = True → 2. 全量刮削
          → 读取 auto_subtitles = True → 3. 字幕查找
等待 cron_interval_min 分钟后下一轮
```

| 配置键 | 默认值 | 说明 |
|--------|--------|------|
| `cron_enabled` | False | 巡逻总开关 |
| `cron_interval_min` | 60 | 间隔（分钟，≤0 回退到 60）|
| `auto_scrape` | False | 扫描后自动刮削 |
| `auto_subtitles` | False | 刮削后自动找字幕 |

### 2.3 路由优先级

```
1. /api/v1/auth/*    鉴权路由（显式豁免 JWT）
2. /api/v1/public/*  公共路由（JWT 保护）
3. /api/v1/*         业务路由（全局 JWT）
4. /health           健康检查（豁免）
5. /*                静态文件兜底（最低优先级）
```

### 3.1 数据库模块（db_manager.py）

#### 核心特性

| 特性 | 实现 |
|------|------|
| WAL 模式 | `PRAGMA journal_mode=WAL` |

#### tasks 表核心字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | INTEGER PK | 自增主键 |
| `path` | TEXT UNIQUE | 原始文件路径（去重键）|
| `status` | TEXT | `pending/scraped/archived/failed/ignored` |
| `type` | TEXT | `movie/tv` |
| `season` | INTEGER | 季号（剧集专用）|

#### 双表删除机制（v1.0.0 修复）

```
delete_task(task_id):
  1. 先查 tasks 中该 id 对应的 path
  -> 2. DELETE FROM tasks WHERE id = ?
  -> 3. DELETE FROM media_archive WHERE path = ?       （按路径，最准确）
  -> 4. DELETE FROM media_archive WHERE original_task_id = ?  （兜底）
```

⚠️ `tasks.id` 与 `media_archive.id` 各自独立自增，禁止用 id 交叉对删。

#### 原子操作：mark_task_as_ignored_and_inherit

```

1. 跨表检索：从 media_archive（冷表）或 tasks（热表）检索同源海报路径
```

### 3.2 安全模块（security/crypto.py）

**神盾计划（Project Aegis）** — 四项安全能力：

| 能力 | 实现 | 用途 |
|------|------|------|
| Fernet 对称加密 | `cryptography.fernet` | 加密存储 API 密钥 |
| Bcrypt 密码哈希 | `passlib[bcrypt]` | 管理员账号密码 |
| JWT Token | `PyJWT` HS256 | 无状态会话（7天）|
| 管理员账号管理 | `data/auth.json` | 初始化/验证/读取 |

```python
crypto = get_crypto_manager()
crypto.encrypt_api_key("sk-xxx")     # → 密文
crypto.decrypt_api_key(cipher)       # → 明文，失败返回空字符串
crypto.create_access_token("admin")  # → JWT（7天）
crypto.verify_token(token)           # → username 或 None
crypto.init_admin(username, pwd)     # 仅允许执行一次
```

### 4.3 任务状态机

```
pending
  ├─► scraped   （刮削成功）
  │     └─► archived  （归档成功）
  ├─► failed    （刮削/归档失败）
  │     └─► pending   （retry 重置）
  └─► ignored   （手动跳过）
```

### 4.4 并发防重机制（v1.0.0-Stable 核心加固）

每个任务模块独立维护 `threading.Lock` + 状态字典：

```python
# 物理级并发防重逻辑：
# 1. 非阻塞获取锁（blocking=False），失败立即返回丢弃冗余请求
# -> 2. 检查 is_running 内存标记（双重防护）
# -> 3. finally 块无条件释放锁（系统自愈，防按钮永久僵死）
if not _scan_entry_lock.acquire(blocking=False):
    logger.warning("[SCAN] 拦截并发请求，本次触发已丢弃")
    return
```

## 五、服务层（Services）

### 5.1 AI 决策层（services/ai/）

**AIAgent** — 所有配置从数据库动态读取，WebUI 修改无需重启。

意图常量：

| 常量 | 含义 |
|------|------|
| `ACTION_SCAN` | 物理扫描 |
| `ACTION_SCRAPE` | 全量刮削 |
| `ACTION_SUBTITLE` | 字幕查找 |
| `DOWNLOAD` | 下载指令 |
| `CHAT` | 普通聊天 |

`process_message()` 流程：
```
1. 候选等待状态拦截（读取 DB _pending_candidates）
-> 2. LLMClient.call_llm() 意图识别（JSON 输出）
-> 3. 解析失败 → 关键词规则引擎兜底
-> 4. _generate_llm_response() 生成响应
     ├─► DOWNLOAD（模糊）→ TMDB 候选 → 写入 _pending_candidates
     ├─► DOWNLOAD（精准）→ ServarrClient 直接下载
     └─► 其他 → LLM 生成回复
```

**LLMClient** — 兼容所有 OpenAI 格式 API，三连击指数退避重试（1s→2s→4s）。

### 5.2 扫描引擎（services/scraper/）

**ScanEngine** 防护机制：

| 防护 | 实现 |
|------|------|
| 软链接死循环 | `os.walk(followlinks=False)` |
| 超深目录剪枝 | `MAX_SCAN_DEPTH=5` |
| NAS 系统目录 | 跳过 `/@eadir/`、`.deletedbytmm`、隐藏目录 |
| 并发去重 | `Set[str]` 路径去重 |
| 物理 inode 熔断 | (inode, size) 指纹集合防硬链接重复入库 |

**MediaCleaner** — 纯执行层，不硬编码任何过滤正则，`filename_clean_regex` 是系统唯一正则真相源。

清洗步骤：
```
1. 去除文件扩展名
-> 2. 去除首部方括号组名（20字以内）
-> 3. 执行数据库 filename_clean_regex 过滤正则
-> 4. 中文冒号→英文冒号 / 下划线/点/横线→空格
-> 5. 清理多余空格 + 首尾特殊符号
```

### 5.3 元数据服务（services/metadata/）

**TMDBAdapter** 精确匹配策略：
```
results = search_tv(query, year)
  ├─ 优先：original_name == query（严格）
  ├─ 其次：name == query（严格）
  ├─ 宽松：去末尾集号后匹配
  └─ 兜底：results[0]
```

429 限流自动重试：最多 3 次，指数退避 2s→4s→8s。

**MetadataManager** — 生成 NFO + 海报 + Fanart，各步骤独立容错。

### 5.4 归档组织器（services/organizer/hardlinker.py）

**SmartLink** 三级降级策略：
```
1. 硬链接 os.link()      （同分区，零额外空间，首选）
-> 2. 软链接 os.symlink()  （跨分区，依赖源文件）
-> 3. 文件复制 shutil.copy2 （兜底，占用双倍空间）
```

归档路径规范：
```
电影：{library_root}/{title} ({year})/{filename}
剧集：{library_root}/{title} ({year})/Season {season:02d}/{filename}
```

### 5.5 字幕引擎（services/subtitle/engine.py）

- 语言评分：`zh-CN`=100 / `zh-TW`=50 / `en`=30
- 本地检测：已有同名字幕直接跳过，节省 API 配额
- 429 冷却：等待 15 秒重试
- 命名规范：`{video_stem}.ai.{lang_code}.srt`

## 六、并发与任务调度模型

| 任务类型 | 执行方式 | 原因 |
|----------|----------|------|
| 物理扫描 | BackgroundTasks（asyncio）| I/O 密集，异步友好 |
| 全量刮削 | threading.Thread | 同步库调用，避免阻塞事件循环 |
| 字幕查找 | threading.Thread | 同步 HTTP 调用 |
| AI 对话 | asyncio 协程 | 天然异步 |
| 定时巡逻 | asyncio.create_task | 后台持续运行 |

**SQLite 线程安全**：`threading.local()` 每线程独立连接 + WAL 模式支持并发读写。

## 七、安全架构

```
第一层：网络层   → CORS 白名单
第二层：路由层   → 默认拦截 /api/v1/*，显式豁免白名单
第三层：认证层   → JWT HS256 + 7天过期，全局 Depends 注入
第四层：数据层   → Fernet 加密 6个 API Key，Bcrypt 哈希密码
第五层：文件层   → Path.resolve() + 目录黑名单 + 后缀白名单
```

### 鉴权豁免清单

| 路由 | JWT | 原因 |
|------|-----|------|
| `/api/v1/auth/status` | 无 | 登录前检查初始化状态 |
| `/api/v1/auth/init` | 无 | 首次初始化无账号可验证 |
| `/api/v1/auth/login` | 无 | 登录本身需要豁免 |
| `/health` | 无 | 外部健康检查 |
| `/*`（静态文件）| 无 | 前端静态资源公开访问 |

## 八、架构约束与红线

| 约束 | 说明 |
|------|------|
| 正则唯一真相源 | 所有过滤正则必须来自数据库，严禁硬编码 |
| AI 规则来自数据库 | `expert_archive_rules`、`master_router_rules` 不得写死 |
| 业务 API 走 api_router | 禁止绕过全局 JWT 依赖 |
| Services 不操作 DDL | 只能通过 DatabaseManager 方法操作数据 |
| 各步骤独立容错 | 单步失败不得崩溃整个任务 |
| 刮削/字幕用线程 | 禁止在 asyncio 事件循环中调用阻塞同步 HTTP |
| 双表删除按 path | 禁止用 id 交叉对删 tasks 和 media_archive |
