## 一、后端系统定位与设计哲学

### 1.1 系统定位
Neon Crate 后端是一套面向 NAS / 家庭媒体服务器场景的**全自动影音库管理引擎**。它以 FastAPI 为 Web 框架骨干，承担以下核心职责：
- **AI 总控神经**：接收自然语言指令，通过 LLM 进行意图路由，自动触发扫描、刮削、下载、字幕等任务
- **物理文件编排**：扫描下载目录、清洗文件名、TMDB 刮削元数据、SmartLink 硬链接归档
- **数据持久化**：SQLite WAL 模式主数据库，管理所有媒体任务状态、配置、归档记录
- **安全网关**：JWT 无状态鉴权 + Fernet 加密配置，保护所有 API 密钥和用户数据
- **AIO 静态服务**：生产模式下同时托管前端静态资源，实现单端口、单进程的极简部署
### 1.2 设计哲学
| 原则 | 具体体现 |
|------|----------|
| **数据库即真相源** | 所有正则规则、AI 规则、配置均存储于 SQLite，严禁硬编码 |
| **默认拦截，显式豁免** | 所有 `/api/v1/*` 路由默认要求 JWT，仅白名单端点豁免 |
| **零依赖降级** | 各服务独立容错：海报下载失败不影响 NFO 生成，字幕失败不影响归档 |
| **极简部署** | AIO 模式单进程、单端口、无需 CORS 配置，适合 NAS 场景 |
| **数据库驱动行为** | AI 人格、路由规则、归档规则均可通过 WebUI 热更新，无需重启 |
| **防重叠调度** | 全局任务状态字典防止同类任务并发执行，避免资源竞争 |

## 四、应用入口与生命周期管理

> 核心文件：`backend/app/main.py`
### 4.1 启动序列
```
应用启动（uvicorn app.main:app）
  │
  ├─► 日志系统初始化（RotatingFileHandler，10MB × 5 备份）
  ├─► DatabaseManager 懒初始化
  │     ├─► SQLite WAL 模式开启
  │     ├─► 自动建表（tasks / media_archive / config）
  │     ├─► _migrate_plaintext_keys()   历史明文密钥迁移
  │     └─► _inject_ai_defaults()       注入 AI 规则 + 15 条正则默认值
  ├─► 路由注册
  │     ├─► /api/v1/auth   → auth_router（无 JWT）
  │     ├─► /api/v1/public → public_system_router（有 JWT）
  │     └─► /api/v1        → api_router（全局 JWT）
  ├─► 静态资源挂载
  │     ├─► /api/v1/assets → Docker /storage 或 data/posters/
  │     └─► /             → static/（AIO 模式，可选）
  ├─► SPA 404 回退处理器注册
  └─► asyncio.create_task(cron_scanner_loop())   后台巡逻协程启动
```
### 4.2 自动巡逻流水线（`cron_scanner_loop`）
后台持续运行的 `asyncio` 协程，是 Neon Crate 全自动化能力的核心调度器。
```
每轮开始
  ├─► 读取 cron_enabled
  │     = False → 跳过本轮，等待下一个间隔
  │     = True  ↓
  ├─► 步骤 1：await perform_scan_task()              物理扫描 + 智能入库
  ├─► 读取 auto_scrape
  │     = False → 跳过步骤 2 和 3
  │     = True  ↓
  ├─► 步骤 2：run_in_executor(perform_scrape_all_task_sync)
  │     ├─► 检查 scrape_all_status["is_running"] 防重叠
  │     └─► 刮削完成 ↓
  ├─► 读取 auto_subtitles
  │     = False → 跳过步骤 3
  │     = True  ↓
  └─► 步骤 3：run_in_executor(perform_find_subtitles_task_sync)
        └─► 检查 find_subtitles_status["is_running"] 防重叠
等待 cron_interval_min 分钟后进入下一轮
```
**巡逻配置项：**
| 配置键 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `cron_enabled` | bool | False | 巡逻总开关 |
| `cron_interval_min` | int | 60 | 巡逻间隔（≤0 时回退到 60）|
| `auto_scrape` | bool | False | 扫描后自动刮削 |
| `auto_subtitles` | bool | False | 刮削后自动找字幕（依赖 auto_scrape）|
### 4.3 路由优先级
```
优先级从高到低：
  1. /api/v1/auth/*       鉴权路由（显式豁免 JWT）
  2. /api/v1/public/*     公共路由（JWT 保护）
  3. /api/v1/*            业务路由（全局 JWT）
  4. /health              健康检查（豁免）
  5. /*                   静态文件兜底（最低优先级）
```

### 5.1 配置模块（`app/infra/config/`）
基于 **pydantic-settings** 的环境变量配置管理，提供全局单例 `settings`，带 `lru_cache` 缓存避免重复解析。
**关键配置字段：**
| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|

| `CORS_ORIGINS` | list | `["http://localhost:3000"]` | CORS 白名单 |

| `CONFIG_PATH` | str | `"data/config.json"` | 配置文件路径 |

| `JWT_ALGORITHM` | str | `"HS256"` | JWT 算法 |
| `JWT_EXPIRE_DAYS` | int | `7` | Token 过期天数 |

### 5.2 数据库模块（`app/infra/database/db_manager.py`）
#### 5.2.1 核心特性
| 特性 | 实现 |
|------|------|
| WAL 模式 | `PRAGMA journal_mode=WAL`，提升并发读取性能 |

| 敏感密钥加密 | 6 个 API Key 字段自动加密存储，读取时自动解密 |
| 自动明文迁移 | 首次启动检测并迁移历史明文密钥 |

SENSITIVE_KEYS = [

#### 5.2.4 双表删除机制（V1.1 修复）
历史版本存在 `tasks.id` 误删 `media_archive.id` 的 Bug，导致归档记录残留后文件永远无法重新刮削。修复后逻辑：
```
delete_task(task_id):
  1. 先查 tasks 中该 id 对应的 path
  2. DELETE FROM tasks WHERE id = ?
  3. DELETE FROM media_archive WHERE path = ?              ← 按路径（最准确）
  4. DELETE FROM media_archive WHERE original_task_id = ?  ← 兜底
```
#### 5.2.5 AI 规则注入（`_inject_ai_defaults`）
服务首次启动时自动执行，只在字段为空时注入工业默认值，不覆盖用户已有内容：
| 配置键 | 说明 |
|--------|------|
| `ai_name` | AI 助手名称 |
| `ai_persona` | AI 人格 System Prompt |
| `expert_archive_rules` | 归档专家规则（JSON 输出约束）|
| `master_router_rules` | 总控路由规则（意图识别 JSON 指令）|
| `filename_clean_regex` | 15 条正则清洗规则（全局唯一真相源）|

### 5.3 安全模块（`app/infra/security/crypto.py`）
**神盾计划（Project Aegis）**，提供四项安全能力：
| 能力 | 实现 | 用途 |
|------|------|------|
| Fernet 对称加密 | `cryptography.fernet` | 加密存储 API 密钥等敏感配置 |
| Bcrypt 密码哈希 | `passlib[bcrypt]` | 管理员账号密码安全存储 |
| JWT Token | `PyJWT` HS256 | 无状态会话管理（7 天过期）|
| 管理员账号管理 | `data/auth.json` | 初始化 / 验证 / 读取 |
**文件依赖：**
| 文件 | 用途 | 权限 |
|------|------|------|
| `data/secret.key` | Fernet 主密钥（同时作为 JWT 签名密钥）| `0o600` |
| `data/auth.json` | 管理员用户名 + bcrypt 哈希密码 | `0o600` |
**关键方法：**
```python
crypto = get_crypto_manager()  # 懒初始化全局单例
crypto.encrypt_api_key("sk-xxx")        # → 密文字符串
crypto.decrypt_api_key(cipher_text)     # → 明文，失败返回空字符串
crypto.get_password_hash("123456")      # → bcrypt 哈希
crypto.verify_password(plain, hashed)   # → bool
crypto.create_access_token("admin")     # → JWT 字符串（7天）
crypto.verify_token(token)              # → username 或 None
crypto.init_admin(username, password)   # 仅允许执行一次
crypto.authenticate(username, password)  # → bool
```

### 6.2 鉴权端点（`app/api/auth.py`）
路由前缀：`/api/v1/auth`
| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/auth/status` | 检查系统是否已初始化管理员账号 |
| `POST` | `/auth/init` | 首次初始化管理员（仅允许一次）|
| `POST` | `/auth/login` | 登录，返回 JWT Token |
| `GET` | `/auth/verify` | 验证 Token 有效性 |
**全局依赖注入 `get_current_user()`**：注入到 `api_router` 的 `dependencies`，对所有业务路由全局生效。Token 无效或过期返回 `HTTP 401`。
**调用链：**
```
POST /auth/login
  └─► CryptoManager.authenticate()         # bcrypt 校验
        └─► CryptoManager.create_access_token()  # 生成 JWT
所有受保护路由
  └─► get_current_user() [Depends]
        └─► CryptoManager.verify_token()   # JWT 校验
```

### 7.1 AI 决策层（`app/services/ai/`）
#### 7.1.1 `AIAgent`——总控神经

**意图常量：**
| 常量 | 含义 |
|------|------|
| `ACTION_SCAN` | 物理扫描 |
| `ACTION_SCRAPE` | 全量刮削 |

| `LOCAL_SEARCH` | 本地搜索 |
| `SYSTEM_STATUS` | 系统状态查询 |
| `CHAT` | 普通聊天 |
**`process_message()` 总控流程：**
```
1. 读取 master_router_rules 和 ai_persona
2. 候选等待拦截（读取数据库 _pending_candidates）
   ├─► 匹配成功 → ServarrClient(tmdb_id) 直接下载，清除状态
   ├─► 匹配失败（输入像候选选择）→ 保留状态，返回提示+候选
   └─► 匹配失败（其他意图）→ 清除状态，放行至意图识别
3. LLMClient.call_llm() 意图识别（JSON 输出）
4. 解析失败 → 关键词规则引擎兜底
5. _generate_llm_response() 生成响应
   ├─► SYSTEM_STATUS → 注入真实数据库统计（防 LLM 编造）
   ├─► DOWNLOAD（模糊）→ _tmdb_search_candidates() 前5候选
   │                      写入 _pending_candidates，引导用户选择
   ├─► DOWNLOAD（精准）→ 直接调用 ServarrClient 下载
   └─► CHAT → 注入系统快报 + 人格，LLM 生成回复
6. 响应含 __CANDIDATES__ → 强制 action_code=None
7. 返回 (响应文本, 意图代码)
```
**候选匹配白名单反转逻辑：** 只有输入明确像候选选择（纯数字 / `序号.` 格式 / 含候选片名前4字）才保留状态；其余一律清除，避免黑名单覆盖不全问题。

| 防护 | 实现 |
|------|------|

#### 7.2.2 `MediaCleaner`——数据库驱动清洗器
`MediaCleaner` 是纯执行层，**不硬编码任何过滤正则**，`filename_clean_regex` 是系统唯一正则真相源。
**`clean_name()` 清洗步骤：**
1. 去除文件扩展名
2. 去除首部方括号组名（`[HbT]`、`[SubsPlease]` 等，20字以内）
3. 去除所有剩余方括号内容（兜底）
4. 执行数据库 `filename_clean_regex` 过滤正则（逐条 `sub(' ')`）
5. 中文冒号 → 英文冒号 / 下划线/点/横线 → 空格
6. 清理多余空格 + 首尾特殊符号
**固定内置正则（结构化提取专用，不受 RegexLab 管理）：** `_YEAR_PATTERN`（年份提取）/ `_SEASON_EPISODE_PATTERNS`（S01E01 等）/ `_ANIME_EPISODE_PATTERN`（动漫格式）
**默认15条过滤规则（存储于数据库，用户可在 RegexLab 增删）：** 分辨率 / 编码格式 / 方括号技术标签 / 花括号标签 / 广告词 / 音视频特性 / 语言标签 / 制作组后缀 / 年份清除 / 季集标签（5条）/ 动漫集数格式。一键重置：`POST /tasks/settings/reset {"target": "regex"}`

## 十、安全架构

### 10.1 纵深防御体系
```
第一层：网络层   → CORS 白名单
第二层：路由层   → 默认拦截 /api/v1/*，显式豁免白名单
第三层：认证层   → JWT HS256 + 7天过期，全局 Depends 注入
第四层：数据层   → Fernet 加密 6个 API Key，Bcrypt 哈希密码
第五层：文件层   → Path.resolve() + 目录黑名单 + 后缀白名单
```
### 10.2 鉴权豁免清单
| 路由 | JWT | 原因 |
|------|-----|------|
| `/api/v1/auth/status` | 无 | 登录前检查初始化状态 |
| `/api/v1/auth/init` | 无 | 首次初始化无账号可验证 |
| `/api/v1/auth/login` | 无 | 登录本身需要豁免 |
| `/api/v1/public/image` | 有 | 路径安全机制保护 |
| `/health` | 无 | 外部健康检查 |
| `/*`（静态文件）| 无 | 前端静态资源公开访问 |
### 10.3 Token 生命周期
```
生成：POST /auth/login → create_access_token() → HS256 → 7天过期
传输：Authorization: Bearer <token>
存储：前端 localStorage.token
校验：每次受保护请求 → get_current_user() → verify_token()
失效：过期或密钥轮换 → HTTP 401 → 前端跳转登录页
```

## 十一、并发与任务调度模型

### 11.1 并发模型总览
| 任务类型 | 执行方式 | 原因 |
|----------|----------|------|
| 物理扫描 | BackgroundTasks（asyncio）| I/O 密集，异步友好 |
| 全量刮削 | threading.Thread | 同步库调用，避免阻塞事件循环 |
| 字幕查找 | threading.Thread | 同步 HTTP 调用 |
| AI 对话 | asyncio 协程 | 天然异步 |
| 定时巡逻 | asyncio.create_task | 后台持续运行 |
### 11.2 防重叠机制
三个模块级全局字典在任务启动时设 is_running=True，完成或异常后重置为 False，再次触发时直接返回提示，不重复执行。
### 11.3 SQLite 线程安全
DatabaseManager 使用 threading.local() 为每个线程维护独立连接，配合 WAL 模式支持多线程并发读写。

## 十二、数据库架构

### 12.1 设计原则
- 单文件部署：data/media_database.db，无需额外数据库服务
- WAL 模式：读写并发性能优于 DELETE 模式
- 配置与数据分离：业务配置存 config.json（原子写入），媒体数据存 SQLite
- 唯一真相源：filename_clean_regex 等规则只在数据库中存在，代码中严禁备份副本
### 12.2 统计查询模式
实时统计（pending/completed）：直接 SQL COUNT 查询 tasks 表。
缓存统计（movies/tv_shows）：扫描/刮削完成后由 _update_library_counts() 物理统计并写入 config 表，接口直接读缓存，避免 NAS 高频 I/O。
### 12.3 配置存储格式
config.json 原子写入，包含：settings（所有系统配置，敏感键为 Fernet 密文）、paths（路径配置数组）、_pending_candidates（AI 候选临时状态，选择完成后清除）。

## 十三、AIO 单进程部署架构

### 13.1 AIO 模式原理
传统分离模式：两个进程（Next.js + FastAPI），需要 CORS，内存约 500MB。
AIO 单进程模式：单一 FastAPI 进程同时提供 API 和静态文件，无 CORS，内存约 300MB。
### 13.2 核心实现要点
前端构建：next.config.js 设置 output: 'export' 生成纯静态文件，禁用图片优化，移除 rewrites。
后端挂载：app.mount("/", StaticFiles(directory="static", html=True))。
SPA 回退：404 处理器对 /api/* 返回 JSON 404，其他路径返回 static/index.html。
### 13.3 部署流程
1. cd frontend && npm run build
2. mv out ../backend/static
3. cd backend && python -m uvicorn app.main:app --host 0.0.0.0 --port 8000
### 13.4 Docker Compose 极简编排
services.backend: build ./backend，端口 8000:8000，挂载 data 和 storage 目录，注入 DOCKER_STORAGE_PATH 和 JWT_SECRET_KEY 环境变量。

## 十六、架构约束与红线清单

### 16.1 数据库约束
- 所有过滤正则必须来自数据库 filename_clean_regex，严禁在代码中硬编码
- 所有 AI 规则（expert_archive_rules、master_router_rules、ai_persona）必须来自数据库
- filename_clean_regex 是系统唯一正则真相源，MediaCleaner 和刮削任务均从此读取
- 新功能若需扩展配置字段，必须在 domain_system.py 的 SystemSettings 中添加并评审
### 16.2 API 约束
- 所有新增业务 API 必须通过 api_router 挂载，不得绕过全局 JWT 依赖
- 禁止在随机 Handler 内重复手写认证逻辑
- 图片访问必须通过 /api/v1/public/image 代理，禁止直接暴露物理路径
- 设置保存前必须执行 1+1 路径约束校验
### 16.3 服务层约束
- Services 层禁止直接操作数据库表结构（DDL），只能通过 DatabaseManager 的方法操作
- 各服务步骤必须独立容错，单步失败不得导致整个任务崩溃
- 刮削和字幕任务使用 threading.Thread，禁止在 asyncio 事件循环中调用阻塞同步 HTTP
- 硬链接归档要求源文件和目标在同一分区，Docker 单存储挂载可确保此条件
### 16.4 安全约束
- JWT_SECRET_KEY 默认值仅用于开发，生产环境必须通过 .env 替换
- 6 个敏感 API Key 必须通过 CryptoManager 加密存储，禁止明文写入配置文件
- 图片代理必须经过 Path.resolve() 防御和目录黑名单校验
- secret.key 文件权限必须为 0o600，禁止 Git 提交
### 16.5 前端约束
- 所有受保护接口通过 lib/api.ts 的 secureFetch() 调用，禁止组件直接 fetch
- 所有 UI 文本使用 i18n key，禁止硬编码中文字符串
- 图片加载使用 getPosterUrl() 封装，本地路径必须 encodeURIComponent 编码
- 前端 Task 接口字段 season/episode 必须为整数类型，status 枚举无 success
