## 核心架构约束（红线）

| # | 约束 | 违反后果 |
|---|------|----------|
| 1 | 所有过滤正则必须来自数据库 `filename_clean_regex`，严禁硬编码 | 正则规则与 RegexLab 脱钩，用户设置失效 |
| 2 | 所有 AI 规则必须来自数据库 | AI 行为无法通过 WebUI 热更新 |
| 3 | 所有业务 API 必须通过 `api_router` 挂载，不得绕过全局 JWT | 未鉴权端点暴露 |
| 4 | 前端所有 HTTP 调用只能经过 `lib/api.ts` | 401 处理、超时熔断失效 |
| 5 | 6个敏感 API Key 必须通过 `CryptoManager` 加密存储 | 配置文件明文泄露密钥 |
| 6 | 删除操作必须按 `path` 双重清理 `tasks` 和 `media_archive` | 归档记录残留，文件永远无法重新刮削 |
| 7 | `mark_task_as_ignored_and_inherit` 不可拆分（原子操作） | 前端 VHS 特效失效 |
| 8 | 前端所有 UI 文本通过 `t(key)` 获取，禁止硬编码字符串 | i18n 切换失效 |
| 9 | `getPosterUrl` 只返回原始物理路径，严禁提前拼接 `/api/v1/public/image?path=` | SecureImage 防套娃误判 → 404 |
| 10 | 禁止 `sudo npm run dev` | `.next` 缓存 root 锁死，代码修改静默失效 |
| 11 | `NEXT_PUBLIC_API_BASE` 绝对路径只能存放于 `.env.development.local` | 编译期 bake 进 bundle，AIO 局域网设备图片失败 |
| 12 | `SecureImage` 本站图片代理必须走 `fetch + token`，不能 `<img src>` 直接渲染 | AIO 模式 401 |
