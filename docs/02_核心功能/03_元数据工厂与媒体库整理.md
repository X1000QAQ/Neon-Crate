# 元数据工厂：TMDB 识别与媒体库整理

---

## 一句话理解

系统拿到「蝙蝠侠：黑暗骑士」这个片名，去 TMDB 查到完整信息，生成海报、背景图、NFO 文件，并把原始文件无损整理进媒体库。

---

## TMDB 匹配流程

```
清洗后的片名 + 年份
  ↓
搜索 TMDB（电影走 /search/movie，剧集走 /search/tv，language=zh-CN）
  ↓
精确匹配：优先选 original_title/original_name == query 的结果
  ↓ 若无精确匹配：宽松匹配（去末尾集号后再比对，如 "The Boys 02" → "The Boys"）
  ↓ 最终兜底：results[0]
  ↓
获得 tmdb_id / 标题 / 年份 / 海报路径
  ↓
调用 /external_ids 获取 IMDb ID
  ↓
下载海报 poster.jpg + 背景图 fanart.jpg
  ↓
生成 NFO 元数据文件
  ↓
任务状态更新为 archived，season 回写数据库
```

**精确匹配设计原因：** TMDB zh-CN 搜索「The Boys」时，`results[0]` 可能是同名的「哈迪兄弟（2020）」而非「黑袍纠察队（2019）」。精确匹配优先选 `original_name == "The Boys"` 的结果，彻底解决同名歧义问题。

## 生成的本地文件

每部电影/剧集归档后，媒体库目录里会有：

```
电影示例：
/媒体库/电影/蝙蝠侠：黑暗骑士 (2008)/
  ├── 蝙蝠侠：黑暗骑士 (2008).mkv   ← 硬链接（零空间占用）
  ├── poster.jpg                    ← TMDB 海报
  ├── fanart.jpg                    ← TMDB 背景图（1280宽）
  └── movie.nfo                     ← 元数据（Emby/Jellyfin/Plex 可读）

剧集示例：
/媒体库/剧集/黑袍纠察队 (2019)/
  ├── tvshow.nfo
  ├── poster.jpg
  ├── fanart.jpg
  ├── Season 2/
  │   └── 黑袍纠察队 (2019) - S02E02.mkv
  └── Season 3/
      └── 黑袍纠察队 (2019) - S03E01.mkv
```

---

## NFO 文件包含什么

NFO 是 Kodi/Emby/Jellyfin/Plex 通用的元数据格式（XML），包含：

- 标题 / 原始标题 / 简介 / 标语
- 评分 / 投票数 / 年份 / 首播日期
- TMDB ID / IMDb ID
- 类型标签（动作/剧情/科幻...）
- 演员列表（前10位，含角色名和头像URL）
- 导演（电影）
- 海报和背景图 URL

## 匹配失败怎么办

- TMDB 没有结果 → 任务标记为 `failed`，可以在媒体墙点「重试」
- AI 提炼失败 → 自动降级用正则清洗名直接搜索
- 剧集匹配失败 → 自动尝试只用第一个词重新搜索

## Servarr 下载联动（寻猎者计划）

当你对 AI 说「下载流浪地球」时，不是扫描本地文件，而是：

```
AI 识别意图 DOWNLOAD
  ↓
先去 TMDB 确认片名和 tmdb_id（防止名称歧义）
  ↓
用 tmdb:12345 精确查询 Radarr/Sonarr
  ↓
如果已在库中 → 触发搜索补全（不重复添加）
如果不在库中 → 添加并立即触发下载
```

这个「先 TMDB 确认再提交 Radarr/Sonarr」的设计，避免了同名电影歧义导致的错误下载。
