# 全自动流水线：扫描 → 刮削 → 字幕

---

## 一句话理解

下载好的文件放在指定目录，系统定时自动发现、识别、整理、补充元数据和字幕，全程无需人工干预。

---

## 完整流水线

```
定时巡逻触发（cron_interval_min 分钟一次）
  ↓
步骤1：物理扫描 — 发现下载目录里的新文件，入库
  ↓
步骤2：全量刮削 — 去 TMDB 查电影/剧集信息，归档整理（需开启 auto_scrape）
  ↓
步骤3：字幕搜索 — 去 OpenSubtitles 下载字幕（需开启 auto_subtitles）
```

这三步严格串行，上一步完成才进入下一步。

---

## 控制开关（基础设置页）

| 开关 | 作用 |
|---|---|
| 定时巡逻 `cron_enabled` | 总开关，关闭则不执行任何自动任务 |
| 巡逻间隔 `cron_interval_min` | 多少分钟执行一次，默认 60 分钟 |
| 自动刮削 `auto_scrape` | 扫描后自动去 TMDB 识别并归档 |
| 自动字幕 `auto_subtitles` | 刮削后自动搜索字幕（依赖刮削开启）|

---

## 步骤1：物理扫描

**做什么：** 扫描下载目录，把新文件登记进数据库（`status=pending`）

**过滤规则：**
- 文件小于 `min_size_mb`（默认50MB）→ 直接忽略（过滤样片/预告片）
- 隐藏目录、sample 目录 → 跳过
- 非视频格式文件 → 跳过
- `tasks` 表或 `media_archive` 表中已有该路径 → 跳过（不重复入库）

**类型判断（电影 or 剧集）：**
1. `MediaCleaner` 正则提取（文件名含 S01E01 等 → 剧集）
2. 路径配置可以强制覆盖（配置为 `movie` 目录的文件一律算电影）
3. `mixed` / `library` 等混合目录维持正则猜测不变

**防重复机制：** `check_task_exists_by_path()` 同时检查 `tasks` 和 `media_archive` 两张表，已归档的文件不会被重新扫描入库。若需要重新处理，必须先从前端删除该任务（删除操作会同时清理两张表的记录）。

---

## 步骤2：全量刮削

**做什么：** 对所有 `status=pending` 的任务，去 TMDB 查信息，归档到媒体库

**文件名清洗流程（数据库驱动，无硬编码）：**

```
原始文件名：「[HbT] The Boys - 02 [2160p][HEVC].mkv」
  ↓ MediaCleaner.clean_name()（全程调用数据库 filename_clean_regex）
      1. 去扩展名
      2. 去首部方括号组名 [HbT]
      3. 去剩余方括号内容 [2160p][HEVC]
      4. 执行 15 条数据库正则（去分辨率/编码/季集标签）
      5. 符号清理
  ↓ 清洗后：「The Boys 02」
  ↓ AI 语义提炼（提取片名+年份+类型+季集）
  ↓ { query: "The Boys", year: "2019", type: "tv", season: 2, episode: 2 }
  ↓ TMDB 精确匹配（优先 original_name == query，宽松匹配去末尾集号）
  ↓ 命中「黑袍纠察队」TMDB_ID=76479
```

**季号补充机制（防止无 S01E01 格式文件季号丢失）：**

```
归档路径构建时：
  season_num = db.tasks.season or 1
  if season_num == 1:
      从原始路径目录名搜索 Season XX
      若发现非1的季号 → 覆盖 season_num
  update_task_title_year(season=season_num)  → 写回数据库
  → media_archive.season 与归档路径保持一致
```

**归档动作：**
- 在媒体库目录创建硬链接（零空间占用）
- 下载海报（poster.jpg）和背景图（fanart.jpg）
- 生成 NFO 元数据文件（Jellyfin/Emby/Plex 可直接读取）
- 更新任务状态为 `archived`，season 同步回写

---

## 步骤3：字幕搜索

**做什么：** 对所有已归档但没有字幕的任务，去 OpenSubtitles 下载字幕

**7天冷却机制：** 如果上次搜索结果是「没有字幕」（missing），7天内不会再重复搜索同一部片，避免浪费 API 配额。

**字幕状态流转：**

```
pending → 搜到了 → scraped（字幕已就绪）
pending → 没找到 → missing（7天后重试）
pending → 出错了 → failed（下次任务直接重试）
```

## 任务进度条（前端可见）

每个任务卡片底部有一条进度条，显示流水线进度：

| 状态 | 进度 | 含义 |
|---|---|---|
| `pending` | 30% | 已扫描入库，等待刮削 |
| `archived`，字幕未就绪 | 60% | 刮削完成，字幕搜索中 |
| `archived` + 字幕 `scraped` | 100% | 全部完成 |

## 手动触发

不想等定时器，可以在 Dashboard 或媒体墙点击按钮手动触发：

- **Scan**：立即扫描
- **Scrape**：立即全量刮削
- **Find Subtitles**：立即搜索字幕
