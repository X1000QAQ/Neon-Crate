**高频轮询指数退避策略（Exponential Backoff）**：

> **[2026-03-15 优化]** 降低 NAS 磁盘唤醒频率，保证响应速度的同时保护硬件。

| 阶段 | 时间范围 | 刷新间隔 | 目的 |
|------|----------|----------|------|
| 极速响应 | 0 – 10s | 1.5s | 用户点击后快速感知扫描结果 |
| 平稳监控 | 10 – 30s | 5s | 进入稳态，降低磁盘唤醒频率 |
| 休眠保护 | 30s 后 | 停止 | 完全静默，零磁盘唤醒 |

```typescript
// 🚀 指数退避轮询策略：
// 1. 扫描触发后立即进入极速阶段（1.5s/次）
// -> 2. 10s 后降速进入平稳阶段（5s/次）
// -> 3. 30s 后停止轮询，进入休眠保护（零磁盘唤醒）
const startedAt = Date.now();
let phase = 0; // 0: 极速, 1: 平稳

scanBoostTimerRef.current = setInterval(() => {
  const elapsed = Date.now() - startedAt;
  if (elapsed < 10000) {
    void loadStats();          // 极速阶段：直接刷新
  } else if (elapsed < 30000) {
    if (phase !== 1) {         // 平稳阶段：重置为 5s 间隔
      phase = 1;
      clearInterval(scanBoostTimerRef.current!);
      scanBoostTimerRef.current = setInterval(() => void loadStats(), 5000);
    }
  } else {
    clearInterval(scanBoostTimerRef.current!);  // 休眠保护：停止轮询
    scanBoostTimerRef.current = null;
  }
}, 1500);
```

**其他组件的高频轮询基准**：此策略作为前端高频轮询的性能标准，其他需要轮询的组件应参照此三阶段模型实现。
