### NeuralLinkAlert.tsx
- 监听 `NetworkContext.isLinkDown`，全屏覆盖（`z-[9999]`）
- 「重连」→ `setLinkDown(false)` + `window.location.reload()`

### SecureImage.tsx
- Props: `src: string, alt: string, width?, height?, className?, fallback?`
- 外部链接（`http://` / `https://` 且非本站代理路径）直通，不做鉴权处理
- 本地物理路径：使用 `API_BASE`（来自 `lib/config.ts`）构建代理 URL，**无论开发还是 AIO 生产模式，均走 `fetch + Authorization: Bearer token` 获取 Blob，再转为 ObjectURL 渲染**
- 加入 `res.ok` 检查，后端返回 4xx/5xx 时显示 `fallback`
- 卸载时自动 `URL.revokeObjectURL`，防止内存泄漏

**双模式自适应逻辑：**

| 环境 | `API_BASE` 值 | `finalUrl` 形式 | fetch 行为 |
|------|-------------|----------------|----------|
| 本地开发（3000+8000） | `http://localhost:8000/api/v1` | 绝对路径 | fetch + token ✅ |
| AIO 生产（单端口 8000） | `/api/v1` | 相对路径 | fetch + token ✅ |
| 外部图片（TMDB） | — | `https://...` | 直接 `setBlobUrl`，无 token ✅ |

> **⚠️ 关键约束 1**：上游调用方（如 `MediaTable.tsx` 的 `UniversalMediaRow`）必须传入**原始物理路径**（通过 `posterSrc` prop），
> 严禁提前拼接 `/api/v1/public/image?path=...`，否则防套娃逻辑会误判。
>
> **⚠️ 关键约束 2**：AIO 模式下图片代理路由有 JWT 鉴权，**不能**用 `<img src>` 直接渲染相对路径，
> 必须走 fetch 携带 token，否则 401。
>
> 详见故障排查报告：`docs/06_故障排查/01_海报图片404修复报告.md`

**递归 setTimeout 指数退避（标准实现）**：

```typescript
const pollStats = async (startedAt: number) => {
  const elapsed = Date.now() - startedAt;
  if (elapsed > 30000) return;       // 30s 后自然退出，零泄漏
  await loadStats();                  // 等请求完成再调度，天然防堆积
  const nextDelay = elapsed < 10000 ? 1500 : 5000;
  scanBoostTimerRef.current = setTimeout(() => pollStats(startedAt), nextDelay);
};
```

| 阶段 | 时间范围 | 间隔 |
|------|----------|------|
| 极速响应 | 0 – 10s | 1.5s |
| 平稳监控 | 10 – 30s | 5s |

### 高频输入框“状态隔离缓冲”标准（v1.0.0）

> **结论**：所有会写入全局 `ConfigContext` 的高频输入框，统一采用 **Local State Buffer + onBlur Sync**。

**标准模式**：
- 输入过程中：只更新本地 `useState`（避免每个击键触发 Context 全树重渲染）
- 失焦时：`onBlur` 再调用 `updateSetting(key, value)` 一次性提交（批量同步）

**必须遵守的原因**：
- 可免疫高频输入/粘贴时的渲染抖动与崩溃风险
- 避免 React 合成事件回收（SyntheticEvent pooling）在错误绑定姿势下触发的“内容截断/光标跳位/偶发崩溃”
