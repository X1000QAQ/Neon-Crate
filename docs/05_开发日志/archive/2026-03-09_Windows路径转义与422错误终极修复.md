# Windows 路径转义与 422 错误终极修复

## 修复日期
2026-03-09

## 问题背景

在 Windows 环境下，AI Media Master 遭遇三个关键问题：

1. **路径转义刺客**：Windows 反斜杠路径导致扫描失败
2. **422 错误**：数据库重置接口使用 DELETE 方法导致请求体被忽略
3. **50MB 阈值顽固回退**：配置为 0 时仍回退到 50MB 默认值

## 问题分析

### 问题一：Windows 路径转义地狱

**现象**：
- 前端配置路径：`D:\Downloads\Movies`
- 后端读取路径：`D:\Downloads\Movies`（包含转义字符）
- 物理验证失败：`os.path.exists()` 返回 `False`

**根本原因**：
Windows 使用反斜杠 `\` 作为路径分隔符，在 Python 字符串中会被解析为转义字符，导致路径解析错误。

### 问题二：422 Unprocessable Entity

**现象**：
- 前端调用 `DELETE /tasks/purge` 并传递 `{ confirm: "CONFIRM" }`
- 后端收到请求但 `payload.confirm` 为空
- 返回 422 错误

**根本原因**：
HTTP DELETE 方法的请求体在某些框架和浏览器中会被忽略或不规范处理，FastAPI 无法正确解析请求体。

### 问题三：50MB 阈值顽固回退

**现象**：
- 前端配置 `min_size_mb: 0`（不过滤文件大小）
- 后端读取配置后仍使用 50MB 作为阈值

**根本原因**：
```python
min_size_mb = int(settings.get("min_file_size_mb", 50))
```
当配置值为 `0` 时，Python 的 `dict.get()` 会返回 `0`，但后续逻辑未正确处理 `0` 这个有效值。

## 解决方案

### 修复一：强制路径规范化

**修改文件**：`backend/app/api/v1/endpoints/tasks.py`

**修改位置**：`perform_scan_task()` 函数

**修改内容**：
```python
from pathlib import Path
import os

# 筛选出 enabled=True 的下载目录并强制规范化路径
download_paths = [
    str(Path(p.get("path")).resolve()) for p in paths 
    if p.get("enabled", False) and p.get("type") == "download" and p.get("path")
]
print(f"[DEBUG] [API] 启用的下载目录（已规范化）: {download_paths}")

# 物理验证路径是否存在
for path in download_paths:
    exists = os.path.exists(path)
    print(f"[DEBUG] [API] 物理检查路径是否存在: {path} -> {exists}")
    if not exists:
        logger.warning(f"[API] 路径不存在，跳过: {path}")

# 过滤掉不存在的路径
download_paths = [p for p in download_paths if os.path.exists(p)]
```

**核心改进**：
1. 使用 `Path(path).resolve()` 强制规范化路径，自动处理 Windows 反斜杠
2. 增加物理路径验证打印，便于调试
3. 过滤掉不存在的路径，避免后续扫描失败

### 修复二：将 DELETE 改为 POST

**修改文件**：
- `backend/app/api/v1/endpoints/tasks.py`
- `frontend/lib/api.ts`

**后端修改**：
```python
@router.post("/purge")  # 从 @router.delete("/purge") 改为 @router.post("/purge")
async def purge_all_tasks(payload: PurgeRequest):
    """
    POST /tasks/purge - 全量清空 tasks 表（核弹按钮）
    """
    if payload.confirm.strip().upper() != "CONFIRM":
        raise HTTPException(status_code=400, detail="请正确输入 CONFIRM")
    
    try:
        db = get_db_manager()
        deleted = db.clear_all_tasks()
        logger.info(f"[OK] [API] 已清空任务表，删除 {deleted} 条记录")
        return {"success": True, "deleted": deleted, "message": f"已清空 {deleted} 条任务记录"}
    except Exception as e:
        logger.error(f"[ERROR] [API] 清空任务表失败: {str(e)}")
        raise HTTPException(status_code=500, detail=str(e))
```

**前端修改**：
```typescript
async purgeAllTasks(): Promise<{ success: boolean; deleted: number; message: string }> {
  const res = await secureFetch(`${API_BASE}/tasks/purge`, {
    method: 'POST',  // 从 'DELETE' 改为 'POST'
    headers: getHeaders(),
    body: JSON.stringify({ confirm: 'CONFIRM' }),
  });
  if (!res.ok) throw new Error('Failed to purge tasks');
  return res.json();
}
```

**核心改进**：
- 使用 POST 方法确保请求体被正确解析
- 前后端同步修改，保持接口一致性

### 修复三：正确处理 0 值阈值

**修改文件**：`backend/app/api/v1/endpoints/tasks.py`

**修改位置**：`perform_scan_task()` 函数

**修改内容**：
```python
# 读取最小体积过滤阈值，正确处理 0 值
settings = all_config.get("settings", {})
settings_dict = settings if isinstance(settings, dict) else {}

# 优先读取 min_size_mb，如果不存在则读取 min_file_size_mb，最后才回退到 50
min_size_mb = settings_dict.get("min_size_mb")
if min_size_mb is None:
    min_size_mb = settings_dict.get("min_file_size_mb")
if min_size_mb is None:
    min_size_mb = 50

# 确保即便是 0 也不会被回退
min_size_mb = int(min_size_mb)
print(f"[DEBUG] [API] 最小文件体积阈值: {min_size_mb}MB")
```

**核心改进**：
1. 使用 `is None` 判断而非 `or` 运算符，避免 `0` 被误判为假值
2. 分步读取配置，优先级：`min_size_mb` > `min_file_size_mb` > 默认值 50
3. 确保 `0` 这个有效值不会被回退到默认值

## 技术要点

### Windows 路径处理最佳实践

1. **使用 pathlib.Path**：
   ```python
   from pathlib import Path
   path = Path(raw_path).resolve()  # 自动规范化
   ```

2. **避免硬编码路径分隔符**：
   ```python
   # 错误
   path = "D:\Downloads\Movies"
   
   # 正确
   path = r"D:\Downloads\Movies"  # 原始字符串
   path = Path("D:/Downloads/Movies")  # 正斜杠也支持
   ```

3. **物理验证路径**：
   ```python
   if not os.path.exists(path):
       logger.warning(f"路径不存在: {path}")
   ```

### HTTP 方法选择原则

1. **GET**：查询数据，无副作用
2. **POST**：创建资源或执行操作（推荐用于带请求体的操作）
3. **PUT**：完整更新资源
4. **PATCH**：部分更新资源
5. **DELETE**：删除资源（避免使用请求体）

### Python 零值判断陷阱

```python
# 错误：0 会被误判为 False
value = config.get("key", 50)  # 如果 key=0，返回 0
if not value:  # 0 被判断为 False
    value = 50  # 错误地回退到 50

# 正确：使用 is None
value = config.get("key")
if value is None:
    value = 50
```
