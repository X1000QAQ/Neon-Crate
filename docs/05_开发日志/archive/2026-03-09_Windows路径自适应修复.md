# Windows 路径自适应修复

**日期**: 2026-03-09  
**架构师**: 首席系统架构师  
**状态**: 已完成

---

## 问题描述

后端在 Windows 启动时报 `/storage` 挂载目录未找到。日志显示 `APP_LOG_PATH` 指向 `/app/data`。这会导致 Windows 环境下海报显示、日志读取功能失效。

### 问题根源

1. `main.py` 中硬编码 Docker 路径 `/storage`，Windows 环境下不存在
2. `system.py` 中硬编码日志路径 `/app/data/logs/app.log`，Windows 无法访问
3. 缺乏跨平台路径适配机制

---

## 解决方案

### 第一步：修复静态资源挂载 (main.py)

**修改位置**: `backend/app/main.py`

**修改前**:
```python
if os.path.isdir(settings.DOCKER_STORAGE_PATH):
    app.mount("/api/v1/assets", StaticFiles(directory=settings.DOCKER_STORAGE_PATH), name="assets")
    print(f"[OK] 静态资源已挂载: {settings.DOCKER_STORAGE_PATH} -> /api/v1/assets")
else:
    print(f"[WARN] 未找到挂载目录 {settings.DOCKER_STORAGE_PATH}，本地海报将不可通过 /api/v1/assets 访问")
```

**修改后**:
```python
# 优先使用 Docker 挂载点，Windows 环境自动回退到项目根目录下的 data/posters
assets_dir = None
if os.path.isdir(settings.DOCKER_STORAGE_PATH):
    assets_dir = settings.DOCKER_STORAGE_PATH
    print(f"[OK] 静态资源已挂载: {settings.DOCKER_STORAGE_PATH} -> /api/v1/assets")
else:
    # Windows 环境回退方案：使用项目根目录下的 data/posters
    fallback_dir = Path(__file__).resolve().parent.parent / "data" / "posters"
    fallback_dir.mkdir(parents=True, exist_ok=True)
    assets_dir = str(fallback_dir)
    print(f"[OK] 静态资源已挂载（Windows 回退）: {assets_dir} -> /api/v1/assets")

if assets_dir:
    app.mount("/api/v1/assets", StaticFiles(directory=assets_dir), name="assets")
```

**核心改进**:
- 检测 `/storage` 不存在时，自动回退到 `data/posters`
- 使用 `Path(__file__)` 计算相对路径，跨平台兼容
- 自动创建目录，确保挂载成功

---

### 第二步：修复日志路径硬编码 (system.py)

**修改位置**: `backend/app/api/v1/endpoints/system.py`

**修改前**:
```python
# 日志文件路径
APP_LOG_PATH = Path("/app/data/logs/app.log")
VALID_TAGS = {"SCAN", "TMDB", "SUBTITLE", "ERROR", "API", "ORGANIZER"}
```

**修改后**:
```python
# 日志文件路径 - 自适应路径（支持 Windows 和 Docker）
BASE_DIR = Path(__file__).resolve().parent.parent.parent.parent
APP_LOG_PATH = BASE_DIR / "data" / "logs" / "app.log"
# 确保日志目录存在
APP_LOG_PATH.parent.mkdir(parents=True, exist_ok=True)

VALID_TAGS = {"SCAN", "TMDB", "SUBTITLE", "ERROR", "API", "ORGANIZER"}
```

**核心改进**:
- 使用 `Path(__file__)` 计算项目根目录
- 相对路径 `data/logs/app.log`，Windows 和 Docker 均可访问
- 自动创建日志目录，避免文件不存在错误

### 第三步：校准海报生成逻辑

**检查结果**:

- 前端模型 `TaskResponse` 已定义这两个字段（见 `domain_media.py`）
- 海报下载功能尚未完全实现，暂无需修改

**预留规范**:
当未来实现海报下载时，必须遵循以下规范：
1. 海报保存路径：`data/posters/{tmdb_id}.jpg`
2. 数据库存储：相对路径或能被 `/api/v1/assets` 正确映射的路径
3. 禁止硬编码 `/storage` 等绝对路径
