## 一、核心改进概览

### 1. 全时态感知注入 ✅

**修改文件**: `backend/app/services/ai/agent.py`

**改进内容**:
- 在 `_generate_llm_response` 方法顶部注入系统运行快报
- 为所有对话（包括 CHAT）提供实时系统状态
- 实现真正的"全时态感知"，AI 始终了解当前系统状况

**代码范式**:
```python
# 为所有对话注入系统运行快报
stats = self._get_system_stats()
status_summary = f"[实时现状] 总文件:{stats['total']}, 已归档:{stats['archived']}, 磁盘占用:{stats['disk_usage_percent']}%"

# CHAT 分支也注入系统状态
prompt = f"{persona}\n\n当前系统状态：{status_summary}\n\n执行专业闲聊回复。"
```

### 2. 意图路由强化 ✅

**修改文件**: 
- `backend/app/services/ai/agent.py` (_recognize_intent 方法)
- `backend/data/config.json` (master_router_rules)

**改进内容**:

#### 2.1 扩充关键词库
- **ACTION_SCAN**: 新增"物理扫描"、"找新电影"、"找新剧"、"detect"
- **ACTION_SCRAPE**: 新增"全量刮削"、"匹配封面"、"找元数据"、"获取信息"
- **SYSTEM_STATUS**: 新增"日志"、"运行记录"、"报错信息"、"错误日志"、"log"

#### 2.2 优先级排序
确保意图识别按以下顺序检查，避免误判：
1. ACTION_SCAN
2. ACTION_SCRAPE
3. SYSTEM_STATUS
4. ACTION_SUBTITLE
5. DOWNLOAD
6. LOCAL_SEARCH
7. CHAT（兜底）

#### 2.3 数据库配置同步
在 `config.json` 的 `master_router_rules` 中添加强制映射规则，确保 LLM 意图识别与规则引擎保持一致。

### 3. 日志透明度修复 ✅

**修改文件**: `backend/app/services/ai/agent.py`

**改进内容**:

#### 3.1 路径诊断增强
```python
tried_paths = []  # 记录所有尝试过的路径

# 返回详细诊断信息
diagnostic_msg = (
    f"[诊断] 未能找到日志文件\n"
    f"已尝试路径：\n"
    f"  1. {tried_paths[0]} (Docker路径)\n"
    f"  2. {tried_paths[1]} (Windows路径)\n"
    f"提示：日志文件可能尚未生成，或路径配置有误。"
)
```

#### 3.2 编码容错修复
```python
# 使用 errors='ignore' 避免 Windows 环境编码崩溃
with open(log_path, "r", encoding="utf-8", errors="ignore") as f:
    log_content = "".join(f.readlines()[-lines:])
```

#### 3.3 透明度提升
- 不再返回模糊的"暂无日志"
- 明确告知用户尝试了哪些路径
- 提供具体的失败原因和建议
