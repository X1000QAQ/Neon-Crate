# 全栈日志监控系统修复记录

## 问题描述

物理扫描任务执行成功，但前端日志监控面板始终为空，无法实时查看系统运行状态。

## 根本原因

1. 系统未配置磁盘日志持久化，所有日志仅输出到终端
2. 大量调试信息使用 `print()` 而非标准 `logging` 模块
3. 日志解析器对非标准格式日志直接丢弃，导致信息丢失
4. **关键问题**：`system.py` 中的 `BASE_DIR` 路径计算错误，只使用了 4 层 parent，导致无法正确定位到项目根目录的 `data/logs/app.log`

## 解决方案

### 第一步：修正日志检索路径（核心修复）

**修改文件**: `backend/app/api/v1/endpoints/system.py`

**问题分析**:

`system.py` 位于 `app/api/v1/endpoints/system.py`，从文件位置到项目根目录需要 5 层 parent：
- system.py (当前文件)
- endpoints/ (parent 1)
- v1/ (parent 2)
- api/ (parent 3)
- app/ (parent 4)
- backend/ (parent 5) ← 项目根目录

**核心修复**:

```python
# 修改前（错误）
BASE_DIR = Path(__file__).resolve().parent.parent.parent.parent

# 修改后（正确）
BASE_DIR = Path(__file__).resolve().parent.parent.parent.parent.parent
APP_LOG_PATH = BASE_DIR / "data" / "logs" / "app.log"
```

**增加调试验证**:

在 `get_logs` 接口开头增加路径打印：

```python
@router.get("/logs")
async def get_logs(...):
    print(f"[DEBUG] [API] 网页后端正在尝试读取日志文件: {APP_LOG_PATH}")
    
    if not APP_LOG_PATH.exists():
        return {"logs": [], "source": str(APP_LOG_PATH), "message": "日志文件不存在"}
```

### 第二步：增强日志解析宽容度

**修改文件**: `backend/app/api/v1/endpoints/system.py`

**改进内容**:

1. 将 `DEBUG` 加入到 `VALID_TAGS` 中：

```python
VALID_TAGS = {"SCAN", "TMDB", "SUBTITLE", "ERROR", "API", "ORGANIZER", "DEBUG"}
```

2. 优化 `_parse_log_line` 方法，确保即便没有时间戳的日志（如初次启动时的日志），也能被作为 INFO 级别正常解析并显示。

**核心逻辑**:

```python
def _parse_log_line(line: str) -> dict:
    """
    解析单行日志（兼容标准格式和非标准格式）
    
    标准格式：2026-03-09 12:34:56,789 - app.module - INFO - [TAG] message
    非标准格式：任意文本（如 print 输出）
    
    改进点：
    - 如果不符合标准正则，不丢弃，而是作为 INFO 级别保留
    - 尝试从文本中提取 [SCAN]、[API] 等标签
    """
    # 先尝试提取标签
    tag = None
    for t in VALID_TAGS:
        if f"[{t}]" in line.upper():
            tag = t
            break
    
    # 尝试匹配标准日志格式
    log_pattern = re.compile(
        r"^(\d{4}-\d{2}-\d{2}\s+\d{2}:\d{2}:\d{2},\d{3})\s+-\s+\S+\s+-\s+(INFO|WARNING|ERROR|DEBUG)\s+-\s+(.+)$"
    )
    match = log_pattern.match(line.strip())
    if match:
        ts, level, msg = match.groups()
        return {"raw": line, "timestamp": ts, "level": level, "message": msg, "tag": tag}
    
    # 不符合标准格式，作为 INFO 级别保留（兼容 print 输出）
    return {"raw": line, "timestamp": None, "level": "INFO", "message": line.strip(), "tag": tag}
```

**改进效果**:

- 标准格式日志：正常解析时间戳、级别、消息
- 非标准格式日志：保留原始文本，级别设为 INFO
- 智能标签提取：即使格式不标准，也能识别 [SCAN]、[API] 等标签

### 第三步：校准 main.py 日志初始化

**修改文件**: `backend/app/main.py`

**验证内容**:

确保 `lifespan` 中的日志初始化代码块在 print 之前执行，且 RotatingFileHandler 的 encoding 设为 "utf-8"。

**核心代码**:

```python
# 初始化 Python 标准 logging 系统
log_file = log_dir / "app.log"

# 配置根日志记录器
root_logger = logging.getLogger()
root_logger.setLevel(logging.INFO)
root_logger.handlers.clear()

# 创建 RotatingFileHandler（磁盘日志持久化）
file_handler = RotatingFileHandler(
    filename=str(log_file),
    maxBytes=10 * 1024 * 1024,  # 10MB
    backupCount=5,
    encoding="utf-8",
    delay=True  # Windows 并发访问兼容
)
file_handler.setLevel(logging.INFO)

# 创建 StreamHandler（终端输出）
console_handler = logging.StreamHandler()
console_handler.setLevel(logging.INFO)

# 统一日志格式（必须与 system.py 的解析正则对齐）
log_format = "%(asctime)s - %(name)s - %(levelname)s - %(message)s"
formatter = logging.Formatter(log_format)
file_handler.setFormatter(formatter)
console_handler.setFormatter(formatter)

# 注册处理器
root_logger.addHandler(file_handler)
root_logger.addHandler(console_handler)
```

**关键特性**:

- 使用 `delay=True` 确保 Windows 环境下的并发访问兼容性
- 强制 UTF-8 编码，避免中文乱码
- 日志轮转机制（10MB 单文件，保留 5 个备份）
- 双通道输出（磁盘 + 终端）

## 技术细节

### 日志格式规范

```
%(asctime)s - %(name)s - %(levelname)s - %(message)s
```

**示例输出**:

```
2026-03-09 14:23:45,123 - app.api.v1.endpoints.tasks - INFO - [DEBUG] [API] 开始执行异步物理扫描任务...
2026-03-09 14:23:46,456 - app.services.scraper - INFO - [SCAN] 发现有效文件: The.Matrix.1999.1080p.mkv
```

### 日志文件路径

- 绝对路径: `{PROJECT_ROOT}/data/logs/app.log`
- 自动创建目录: `log_dir.mkdir(parents=True, exist_ok=True)`
- 轮转策略: 10MB 单文件，保留 5 个备份

### Windows 兼容性保障

1. 强制 UTF-8 编码: `encoding="utf-8"`
2. 延迟文件打开: `delay=True`（避免并发冲突）
3. 禁止 Emoji: 所有日志输出使用纯 ASCII 字符
