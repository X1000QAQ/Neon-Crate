DEFAULT_AI_PERSONA = """你是一个负责调度工具的智能中枢。你是行动派，严禁复读，每一条指令都视为全新任务。

核心原则：
1. 无工具不开口：严禁在未调用工具的情况下直接回复。必须先看到工具调用的结果，才能说话。
2. 强制动作：每一次用户的输入，无论是否与上一句重复，都必须视为一个全新的、独立的指令。
3. 传声筒原则：调用工具时，参数必须一字不差地复制用户的原话。严禁总结、严禁提取关键词。"""

DEFAULT_AI_PERSONA = """你是一个负责媒体归档的智能大脑。你必须直接输出结果，严禁说废话，严禁任何解释性语言。"""

#### 3. 净化 `master_router_rules`
彻底删除【铁腕控制逻辑】部分：

**修改前**：
```python
【铁腕控制逻辑】
1. 物理级去噪优先：在 AI 介入前，通过正则表达式物理剥离分辨率、编码格式、压制组标签等冗余信息。
2. 识别失败自动进入人工审核：如果 TMDB 搜索失败或匹配度过低，自动标记为 pending 状态，等待人工介入。
3. 数据库写入受 db_lock 保护：所有 SQLite 写入操作受全局线程锁保护，杜绝 database is locked 崩溃。
```

**修改后**：完全删除此部分

**理由**：AI 是大语言模型，它只负责推理文本，给它讲 SQLite 线程锁和正则剥离毫无意义。

### 第三步：修复 PersonaSettings.tsx 重置后的数据同步问题

修改文件：`frontend/components/settings/PersonaSettings.tsx`

**修改前**：
```typescript
if (result.success) {
  alert('AI 规则已重置为工业级默认值，正在刷新...');
  window.location.reload();
}
```

**修改后**：
```typescript
if (result.success) {
  alert('AI 规则已重置为工业级默认值');
  const freshConfig = await api.getSettings();
  setConfig(freshConfig);
}
```

**优化点**：
- 不再使用硬刷新 `window.location.reload()`
- 改为调用 `api.getSettings()` 重新拉取配置
- 通过 `setConfig()` 更新上层状态，触发页面重新渲染
