### 第二步：创建智能链接引擎

**文件**: `backend/app/services/organizer/hardlinker.py`

#### 2.1 核心特性

1. **首选硬链接**: 调用 `os.link(src, dst)`，零空间占用
2. **智能兜底**: 捕获 `errno.EXDEV`（跨分区错误），自动切换为 `os.symlink`
3. **Windows 特性**: 在 Windows 下执行 `os.symlink` 时，带上 `target_is_directory` 参数
4. **日志审计**: 记录每次链接操作的类型和结果

#### 2.2 核心代码

```python
class SmartLink:
    """智能链接引擎"""
    
    @staticmethod
    def create_link(src: str, dst: str) -> Tuple[bool, str]:
        """
        创建智能链接（硬链接优先，跨盘自动回退到软链接）
        
        Returns:
            Tuple[bool, str]: (是否成功, 链接类型或错误信息)
        """
        # 尝试硬链接
        try:
            os.link(src, dst)
            logger.info(f"[ORG] Hardlink created: {src} -> {dst}")
            return True, "hardlink"
        except OSError as e:
            # 检查是否是跨分区错误
            if e.errno == errno.EXDEV:
                logger.warning(f"[ORG] Cross-disk detected, fallback to Symlink")
                
                # 回退到软链接
                try:
                    is_dir = os.path.isdir(src)
                    
                    if os.name == 'nt':  # Windows
                        os.symlink(src, dst, target_is_directory=is_dir)
                    else:  # Linux/macOS
                        os.symlink(src, dst)
                    
                    logger.info(f"[ORG] Symlink created (cross-disk fallback)")
                    return True, "symlink"
                except Exception as symlink_error:
                    logger.error(f"[ORG] 软链接创建失败: {symlink_error}")
                    return False, f"error: {symlink_error}"
```

#### 2.3 使用场景

- 同盘归档：使用硬链接，零空间占用
- 跨盘归档：自动回退到软链接，确保归档成功
- Windows 环境：自动适配 Windows 软链接特性

#### 3.1 核心功能

#### 3.2 NFO 生成逻辑

```python
class MetadataManager:
    """元数据管理器"""
    
    def generate_nfo(
        self,
        tmdb_id: str,
        media_type: str,
        output_path: str,
        title: str = None,
        year: str = None
    ) -> bool:
        """生成 NFO 元数据文件"""
        try:
            # 从 TMDB 获取详细信息
            if media_type == "movie":
                details = self._fetch_movie_details(tmdb_id)
            else:
                details = self._fetch_tv_details(tmdb_id)
            
            # 生成 XML
            if media_type == "movie":
                xml_content = self._build_movie_nfo(details)
            else:
                xml_content = self._build_tv_nfo(details)
            
            # 写入文件
            with open(output_path, 'w', encoding='utf-8') as f:
                f.write(xml_content)
            
            logger.info(f"[META] NFO 生成成功: {output_path}")
            return True
        except Exception as e:
            logger.error(f"[META] NFO 生成失败: {e}")
            return False
```

#### 3.3 海报下载逻辑

```python
def download_poster(
    self,
    tmdb_id: str,
    media_type: str,
    output_dir: str,
    title: str = None
) -> Optional[str]:
    """下载 TMDB 海报"""
    try:
        # 获取海报 URL
        details = self._fetch_movie_details(tmdb_id) if media_type == "movie" else self._fetch_tv_details(tmdb_id)
        poster_path = details.get("poster_path")
        
        if not poster_path:
            return None
        
        poster_url = f"{self.image_base_url}{poster_path}"
        
        # 下载海报
        response = requests.get(poster_url, timeout=30)
        response.raise_for_status()
        
        # 保存海报
        output_path = os.path.join(output_dir, "poster.jpg")
        with open(output_path, 'wb') as f:
            f.write(response.content)
        
        logger.info(f"[META] 海报下载成功: {output_path}")
        return output_path
    except Exception as e:
        logger.error(f"[META] 海报下载失败: {e}")
        return None
```

### 第五步：前端日期解析最终加固

**文件**: `frontend/lib/utils.ts`

#### 5.1 问题

后端返回的时间格式多样：
- SQLite CURRENT_TIMESTAMP: `2024-03-09 14:30:00`
- 带逗号格式: `2024-03-09, 14:30:00`
- 带毫秒格式: `2024-03-09 14:30:00.123`

#### 5.2 修复

```typescript
export function formatDate(dateString: string): string {
  if (!dateString) return '刚刚';
  
  try {
    // 多重兼容策略：处理后端返回的各种时间格式
    let cleanStr = dateString;
    
    // 策略 1：处理带逗号的格式
    cleanStr = cleanStr.replace(/,\s*/g, ' ');
    
    // 策略 2：处理空格分隔的日期时间
    if (/^\d{4}-\d{2}-\d{2}\s+\d{2}:\d{2}:\d{2}/.test(cleanStr)) {
      cleanStr = cleanStr.replace(' ', 'T');
    }
    
    // 策略 3：处理 SQLite CURRENT_TIMESTAMP 格式
    if (/^\d{4}-\d{2}-\d{2}\s+\d{2}:\d{2}:\d{2}\.\d+/.test(cleanStr)) {
      cleanStr = cleanStr.replace(' ', 'T');
    }
    
    const date = new Date(cleanStr);
    
    // 验证日期有效性
    if (isNaN(date.getTime())) {
      console.warn(`[formatDate] 无法解析日期: ${dateString}`);
      return '格式错误';
    }
    
    return date.toLocaleString('zh-CN', {
      year: 'numeric',
      month: '2-digit',
      day: '2-digit',
      hour: '2-digit',
      minute: '2-digit'
    });
  } catch (error) {
    console.error(`[formatDate] 日期解析异常: ${dateString}`, error);
    return '格式错误';
  }
}
```

## 安全红线

### 绝对禁止

1. **禁止使用 Emoji**: 所有日志输出严禁使用 Emoji，必须使用纯 ASCII 字符
2. **强制 UTF-8 编码**: 所有文件操作必须使用 `encoding="utf-8"`

### 原因

- Windows PowerShell 默认使用 GBK 编码
- Emoji 字符无法用 GBK 编码，导致 `UnicodeEncodeError` 崩溃
