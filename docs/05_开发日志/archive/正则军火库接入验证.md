## 🛠️ 实现方案

### 核心改动

在 AI 调用之前，插入正则预处理引擎，实现物理级去噪：

```python
# ==========================================
# 正则军火库：物理级前置拦截器
# ==========================================
raw_filename = file_name or file_path.split('\\')[-1].split('/')[-1]
cleaned_filename = raw_filename

# 1. 获取系统正则规则
regex_rules_str = db.get_config("filename_clean_regex", "")

# 2. 逐行解析并应用正则
if regex_rules_str:
    for line in regex_rules_str.splitlines():
        rule = line.strip()
        # 跳过空行和注释行
        if not rule or rule.startswith('#'):
            continue
        try:
            # 忽略大小写，执行全局替换为空格
            cleaned_filename = re.sub(rule, ' ', cleaned_filename, flags=re.IGNORECASE)
        except re.error as e:
            logger.warning(f"[RegexLab] 无效的正则规则跳过: {rule}, 错误: {e}")

# 3. 基础标点符号清理（将点、下划线替换为空格，清理多余空格）
cleaned_filename = re.sub(r'[._\[\]\(\)]+', ' ', cleaned_filename)
cleaned_filename = ' '.join(cleaned_filename.split()).strip()

# 4. 兜底策略：如果正则洗得太猛把名字洗没了，退回使用原始名
if not cleaned_filename:
    cleaned_filename = raw_filename

logger.info(f"[RegexLab] 物理正则去噪完成: '{raw_filename}' -> '{cleaned_filename}'")

# AI 决策层强制注入：禁止直接搜索 TMDB
logger.info(f"[AI] 调用 AI Agent 分析文件名: {cleaned_filename}")
ai_result = loop.run_until_complete(ai_agent.ai_identify_media(
    cleaned_name=cleaned_filename,
    full_path=file_path,
    type_hint=media_type
))
```

## 🔧 技术细节

### 1. 正则规则来源

- 配置键: `filename_clean_regex`
- 存储位置: `data/config.json` (通过 `db_manager.get_config()` 获取)
- 格式: 多行文本，每行一条正则规则，支持 `#` 注释和空行

### 2. 解析逻辑

- **逐行解析**: 按 `\n` 分割规则字符串
- **注释过滤**: 跳过以 `#` 开头的行
- **空行过滤**: 跳过空白行
- **异常容错**: 捕获 `re.error`，记录警告但不中断流程

### 3. 清洗策略

- **正则替换**: 使用 `re.sub(rule, ' ', filename, flags=re.IGNORECASE)` 全局替换为空格
- **标点清理**: 移除 `._[]()` 等符号
- **空格压缩**: 将多个连续空格压缩为单个空格
- **兜底保护**: 如果清洗后为空，回退到原始文件名

### 4. 日志输出

```
[RegexLab] 物理正则去噪完成: '[1080p][x265]复仇者联盟4.mkv' -> '复仇者联盟4'
[AI] 调用 AI Agent 分析文件名: 复仇者联盟4
```
