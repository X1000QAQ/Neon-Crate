## 📝 后续建议

### 1. 标签使用规范
建议在开发文档中明确各标签的使用场景：
- `[SCAN]`: 仅用于物理扫描任务
- `[TMDB]`: 仅用于 TMDB API 调用和刮削
- `[AI]` vs `[AI-EXEC]`: 前者用于分析，后者用于执行

### 2. 标签审计机制
建议定期（每月）执行标签审计，确保新增功能的日志标签及时同步到前端。

### 3. 日志级别规范
建议统一日志级别使用：
- `INFO`: 正常业务流程
- `WARNING`: 可恢复的异常
- `ERROR`: 需要人工介入的错误
