# 2026-03-09 422错误与后台任务崩溃修复记录

## 问题描述

### 问题 1：重置数据库报 422 Unprocessable Entity

前端点击"重置数据库"按钮时，后端返回 422 错误，导致操作失败。

**根本原因**：
- 后端 `purge_all_tasks` 接口未定义请求体模型
- 前端发送的 `{ confirm: "CONFIRM" }` 无法被后端解析
- FastAPI 自动返回 422 错误（请求体验证失败）

### 问题 2：物理扫描返回 200 但后台任务无任何输出

前端点击"触发物理扫描"后，接口返回 200 成功，但后台终端无任何日志输出，任务完全"哑火"。

**根本原因**：
- 后台任务在 Windows 环境下执行时，所有 `logger.info()` 输出被 PowerShell 的 GBK 编码拦截
- 中文日志触发 `UnicodeEncodeError`，导致任务静默崩溃
- 缺少强制 `print()` 调试日志，无法追踪任务执行流程

## 修复方案

### 第一步：修复 422 错误

**文件**：`backend/app/api/v1/endpoints/tasks.py`

**修改内容**：

1. 定义 Pydantic 请求模型：

```python
class PurgeRequest(BaseModel):
    """重置数据库请求体"""
    confirm: str
```

2. 修改接口签名，接收请求体：

```python
@router.delete("/purge")
async def purge_all_tasks(payload: PurgeRequest):
    if payload.confirm.strip().upper() != "CONFIRM":
        raise HTTPException(status_code=400, detail="请正确输入 CONFIRM")
    # ... 执行清理逻辑 ...
```

**前端修改**：`frontend/lib/api.ts`

```typescript
async purgeAllTasks(): Promise<{ success: boolean; deleted: number; message: string }> {
  const res = await secureFetch(`${API_BASE}/tasks/purge`, {
    method: 'DELETE',
    headers: getHeaders(),
    body: JSON.stringify({ confirm: 'CONFIRM' }),  // 添加请求体
  });
  if (!res.ok) throw new Error('Failed to purge tasks');
  return res.json();
}
```

### 第二步：修复扫描"哑火"

**文件**：`backend/app/api/v1/endpoints/tasks.py`

**核心修复**：在 `perform_scan_task()` 函数的每个关键步骤添加强制 `print()` 日志

**修改示例**：

```python
async def perform_scan_task():
    global scan_status
    
    try:
        scan_status["is_running"] = True
        scan_status["error"] = None
        
        print("[DEBUG] [API] 开始执行异步物理扫描任务...")
        logger.info("[API] 开始执行异步物理扫描任务...")
        
        # 导入扫描器
        from app.services.scraper import ScanEngine
        print("[DEBUG] [API] ScanEngine 导入成功")
        
        db = get_db_manager()
        print("[DEBUG] [API] 数据库管理器获取成功")
        
        # 读取配置
        all_config = db.get_all_config()
        print(f"[DEBUG] [API] 配置读取成功，配置键: {list(all_config.keys())}")
        
        paths = all_config.get("paths", [])
        print(f"[DEBUG] [API] 路径配置数量: {len(paths)}")
        
        # 筛选下载目录
        download_paths = [
            p.get("path") for p in paths 
            if p.get("enabled", False) and p.get("type") == "download" and p.get("path")
        ]
        print(f"[DEBUG] [API] 启用的下载目录: {download_paths}")
        
        if not download_paths:
            print("[DEBUG] [API] 未配置任何启用的下载目录，扫描终止")
            logger.warning("[API] 未配置任何启用的下载目录，扫描终止")
            scan_status["is_running"] = False
            scan_status["error"] = "未配置启用的下载目录"
            return
        
        # 读取最小体积阈值
        settings = all_config.get("settings", {})
        min_size_mb = int(settings.get("min_file_size_mb", 50))
        print(f"[DEBUG] [API] 最小文件体积阈值: {min_size_mb}MB")
        
        # 初始化扫描引擎
        print("[DEBUG] [API] 正在初始化 ScanEngine...")
        scan_engine = ScanEngine(max_workers=4, min_size_mb=min_size_mb)
        print("[DEBUG] [API] ScanEngine 初始化完成")
        
        logger.info(f"[API] 扫描配置: 路径数={len(download_paths)}, 最小体积={min_size_mb}MB")
        
        # 触发扫描
        discovered_files = []
        for path in download_paths:
            print(f"[DEBUG] [API] 开始扫描路径: {path}")
            logger.info(f"[API] 扫描路径: {path}")
            files = scan_engine.scan_directory(path, recursive=True)
            print(f"[DEBUG] [API] 路径 {path} 发现 {len(files)} 个文件")
            discovered_files.extend(files)
        
        print(f"[DEBUG] [API] 扫描完成，总计发现 {len(discovered_files)} 个有效文件")
        logger.info(f"[API] 扫描完成，发现 {len(discovered_files)} 个有效文件")
        
        # 智能入库
        new_count = 0
        for file_info in discovered_files:
            file_path = file_info.get("path")
            
            if db.check_task_exists_by_path(file_path):
                logger.debug(f"[API] 文件已存在，跳过: {file_path}")
                continue
            
            task_data = {
                "path": file_path,
                "file_name": file_info.get("file_name"),
                "size": file_info.get("size"),
                "clean_name": file_info.get("clean_name"),
                "year": file_info.get("year"),
                "type": "tv" if file_info.get("is_tv") else "movie",
                "season": file_info.get("season"),
                "episode": file_info.get("episode"),
                "status": "pending"
            }
            
            db.insert_task(task_data)
            new_count += 1
            print(f"[DEBUG] [API] 新文件入库: {file_info.get('clean_name')}")
            logger.debug(f"[API] 新文件入库: {file_info.get('clean_name')}")
        
        # 更新状态
        scan_status["is_running"] = False
        scan_status["last_scan_count"] = new_count
        scan_status["last_scan_time"] = asyncio.get_event_loop().time()
        
        print(f"[DEBUG] [API] 扫描任务完成，新增 {new_count} 条任务记录")
        logger.info(f"[API] 扫描任务完成，新增 {new_count} 条任务记录")
        
    except Exception as e:
        scan_status["is_running"] = False
        scan_status["error"] = str(e)
        print(f"[DEBUG] [API] 扫描任务执行失败: {str(e)}")
        logger.error(f"[API] 扫描任务执行失败: {str(e)}")
```

**关键改进**：
1. 每个关键步骤都添加 `print("[DEBUG] ...")` 强制输出
2. 所有 `print()` 输出严格使用纯 ASCII 字符，避免 Emoji
3. 双重日志：`print()` + `logger.info()` 确保在任何环境下都能看到输出
4. 异常捕获时同时输出到 `print()` 和 `logger.error()`

## 技术要点

### Windows 编码陷阱

**问题**：PowerShell 默认使用 GBK 编码，无法处理 Unicode Emoji 和特殊字符。

**解决方案**：
1. 所有 `print()` 输出严格使用纯 ASCII 字符
2. 禁止使用 Emoji（如 ✅、❌、🚀）
3. 使用文本标记替代：`[OK]`、`[ERROR]`、`[DEBUG]`

### FastAPI 请求体验证

**问题**：FastAPI 自动验证请求体，不匹配时返回 422。

**解决方案**：
1. 为所有接收请求体的接口定义 Pydantic 模型
2. 前端确保发送的 JSON 结构与模型匹配
3. 使用 `BaseModel` 定义清晰的数据契约

### 后台任务调试

**问题**：FastAPI 的 `BackgroundTasks` 在后台执行，异常不会抛到前端。

**解决方案**：
1. 使用全局状态字典（如 `scan_status`）跟踪任务状态
2. 在任务函数中捕获所有异常，记录到状态字典
3. 提供独立的状态查询接口（如 `/scan/status`）
4. 使用 `print()` 强制输出调试信息到终端
