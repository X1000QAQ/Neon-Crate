### 第二步：修复时间戳兼容性

**修改文件**: `backend/app/api/v1/endpoints/system.py`

**核心修改**:
在 `_parse_log_line` 函数中，将提取出的 timestamp 字符串中的空格替换为 `T`，使其符合 ISO 8601 标准。

**修改内容**:

```python
def _parse_log_line(line: str) -> dict:
    """
    解析单行日志（兼容标准格式和非标准格式）
    
    标准格式：2026-03-09 12:34:56,789 - app.module - INFO - [TAG] message
    非标准格式：任意文本（如 print 输出）
    
    改进点：
    - 如果不符合标准正则，不丢弃，而是作为 INFO 级别保留
    - 尝试从文本中提取 [SCAN]、[API] 等标签
    - 时间戳格式化为 ISO 8601（空格替换为 T）
    """
    # 先尝试提取标签
    tag = None
    for t in VALID_TAGS:
        if f"[{t}]" in line.upper():
            tag = t
            break
    
    # 尝试匹配标准日志格式
    log_pattern = re.compile(
        r"^(\d{4}-\d{2}-\d{2}\s+\d{2}:\d{2}:\d{2},\d{3})\s+-\s+\S+\s+-\s+(INFO|WARNING|ERROR|DEBUG)\s+-\s+(.+)$"
    )
    match = log_pattern.match(line.strip())
    if match:
        ts, level, msg = match.groups()
        # 关键修复：将空格替换为 T，符合 ISO 8601 标准
        ts = ts.replace(" ", "T")
        return {"raw": line, "timestamp": ts, "level": level, "message": msg, "tag": tag}
    
    # 不符合标准格式，作为 INFO 级别保留（兼容 print 输出）
    return {"raw": line, "timestamp": None, "level": "INFO", "message": line.strip(), "tag": tag}
```

**效果**:
- 时间戳从 `2026-03-09 12:34:56,789` 转换为 `2026-03-09T12:34:56,789`
- 前端 JavaScript 的 `new Date()` 可以正确解析
- 不再显示 Invalid Date

### 第三步：后端响应层强制格式化

**修改文件**: `backend/app/infra/database/db_manager.py`

**核心修改**:
在 `get_all_data` 方法中，确保 `created_at` 字段在返回给前端前，经过 `replace(" ", "T")` 处理。

**修改内容**:

```python
def get_all_data(self, search_keyword: Optional[str] = None) -> List[Dict[str, Any]]:
    """获取所有任务数据"""
    with self.db_lock:
        conn = sqlite3.connect(self.db_path)
        
        if search_keyword:
            cursor = conn.execute("""
                SELECT id, path, file_name, clean_name, type, status, 
                       tmdb_id, imdb_id, title, year, target_path, 
                       sub_status, created_at
                FROM tasks
                WHERE file_name LIKE ? OR clean_name LIKE ? OR title LIKE ?
                ORDER BY created_at DESC
            """, (f"%{search_keyword}%", f"%{search_keyword}%", f"%{search_keyword}%"))
        else:
            cursor = conn.execute("""
                SELECT id, path, file_name, clean_name, type, status, 
                       tmdb_id, imdb_id, title, year, target_path, 
                       sub_status, created_at
                FROM tasks
                ORDER BY created_at DESC
            """)
        
        rows = cursor.fetchall()
        conn.close()
        
        return [
            {
                "id": r[0],
                "path": r[1],
                "file_name": r[2],
                "clean_name": r[3],
                "type": r[4],
                "status": r[5],
                "tmdb_id": r[6],
                "imdb_id": r[7],
                "title": r[8],
                "year": r[9],
                "target_path": r[10],
                "sub_status": r[11],
                "time": r[12].replace(" ", "T") if r[12] and " " in r[12] else r[12]  # 关键修复
            }
            for r in rows
        ]
```

**效果**:
- 100% 解决 Invalid Date 问题
- 所有任务记录的 created_at 字段都符合 ISO 8601 标准
- 前端无需额外处理
