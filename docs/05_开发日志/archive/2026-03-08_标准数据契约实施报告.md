## 一、任务背景

### 问题描述

1. 前端设置界面的"文件名清洗正则"字段为空
2. AI 规则中的变量（如 `query`、`chinese_title`）与当前 Python 后端的字典键名存在不一致
3. 缺乏统一的数据契约规范，导致各模块间数据传递混乱

### 影响范围

- AI 识别模块（`engine.py`）
- TMDB 刮削模块（`engine.py`）
- 数据库管理模块（`db_manager.py`）
- 前端设置界面

---

## 二、解决方案

### 2.1 建立标准数据契约

定义了统一的键名规范，所有模块必须遵守：

| 键名 | 类型 | 说明 |
|------|------|------|
| `clean_name` | string | 纯净名（替代旧的 `query`） |
| `chinese_title` | string | 中文官方译名 |
| `original_title` | string | 原产地名称 |
| `year` | string | 年份（4位数字） |
| `type` | string | 类型（movie/tv/IGNORE） |
| `season` | integer | 季数 |
| `episode` | integer | 集数 |

**废弃变量**:
- ❌ `query` → ✅ `clean_name`
- ❌ `media_name` → ✅ `clean_name`

temp_cleaner = MediaCleaner()

regex_rules = [
    ("分辨率标签过滤", temp_cleaner.resolution_pattern.pattern),
    ("编码格式过滤", temp_cleaner.codec_pattern.pattern),
    ("方括号技术标签过滤", temp_cleaner.bracket_tags_pattern.pattern),

DEFAULT_FILENAME_CLEAN_REGEX = "# 物理级正则去噪规则（从 MediaCleaner 自动提取）\n"
for i, (comment, pattern) in enumerate(regex_rules, 1):
