## 修复目标

## 修复内容

### 1. 修复物理盘点判断逻辑

**问题描述**：
- 原代码使用 `p.get("type")` 判断是否为 library，导致统计逻辑错误
- 统计结果始终为 0

**修复方案**：
- 直接使用配置的 `category` 字段进行统计
- 判断 `category in ["movie", "tv"]` 而不是判断 `type == "library"`

**修改文件**：`backend/app/api/v1/endpoints/tasks.py`

```python
# 修复前
if p.get("type") == "movie":
    movie_count += len(items)
elif p.get("type") == "tv":
    tv_count += len(items)

# 修复后
category = p.get("category", "").lower()
if os.path.exists(folder) and category in ["movie", "tv"]:
    items = [name for name in os.listdir(folder) if os.path.isdir(os.path.join(folder, name))]
    if category == "movie":
        movie_count += len(items)
    elif category == "tv":
        tv_count += len(items)
```

### 3. 修复剧集海报的前端路径层级

**问题描述**：
- 剧集的海报存储在剧集根目录（如 `tv/绝命毒师/poster.jpg`）
- 但 `target_path` 指向的是具体剧集文件（如 `tv/绝命毒师/Season 1/S01E01.mkv`）
- 直接截取导致路径包含 `Season 1`，无法找到海报

**修复方案**：
- 在截取文件夹路径后，检测是否包含 `Season` 字样
- 如果包含，再往上退一级目录

**修改文件**：`frontend/components/media/MediaWall.tsx`

```typescript
// 去除末尾的具体文件名（如 .mkv），只保留文件夹路径
let finalDir = dirPath.split(/[/\\]/).slice(0, -1).join('/');

// 修复：如果路径包含 Season 字样（剧集），海报在上一级目录
if (/Season\s*\d+/i.test(finalDir)) {
  finalDir = finalDir.split(/[/\\]/).slice(0, -1).join('/');
}

return `/api/v1/assets/${finalDir}/poster.jpg`;
```

### 4. 修复剧集外部链接跳转

**问题描述**：
- 原代码在没有 IMDb ID 时，会显示一个额外的 "TMDB TV" 链接
- 实际上第一个 TMDB 链接已经根据 `media_type` 正确跳转了

**修复方案**：
- 移除冗余的 TMDB TV 链接逻辑
- 确保 TMDB 链接根据 `task.media_type` 正确跳转到 movie 或 tv 页面

**修改文件**：`frontend/components/media/MediaWall.tsx`
