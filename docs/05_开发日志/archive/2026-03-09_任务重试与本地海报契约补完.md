### 第二步：后端增加重试接口

修改文件：`backend/app/api/v1/endpoints/tasks.py`

增加 POST 路由用于重试单个失败的任务：

```python
@router.post("/{task_id}/retry")
async def retry_task(task_id: int):
    """
    POST /tasks/{task_id}/retry - 重试单个失败的任务
    
    功能说明：
    - 将任务状态重置为 pending
    - 清空错误信息，以便扫描引擎再次处理
    
    参数：
    - task_id: 任务 ID
    
    返回：
    - success: 是否成功
    - message: 操作结果描述
    """
    try:
        db = get_db_manager()
        
        # 检查任务是否存在
        task = db.get_task_by_id(task_id)
        if not task:
            logger.warning(f"[API] 重试任务失败: 任务 {task_id} 不存在")
            raise HTTPException(status_code=404, detail="任务不存在")
        
        # 重置任务状态为 pending
        db.update_task_status(task_id, "pending")
        
        logger.info(f"[API] 任务 {task_id} 已重置为 pending 状态，等待重新处理")
        return {"success": True, "message": "任务已重置，等待重新处理"}
    except HTTPException:
        raise
    except Exception as e:
        logger.error(f"[API] 重试任务失败: {str(e)}")
        raise HTTPException(status_code=500, detail=f"重试任务失败: {str(e)}")
```

**逻辑说明**：
1. 检查任务是否存在
2. 将任务状态重置为 `pending`
3. 扫描引擎会在下次运行时重新处理该任务

  async retryTask(id: number): Promise<{ success: boolean; message: string }> {
    const res = await secureFetch(`${API_BASE}/tasks/${id}/retry`, {

const getPosterUrl = (task: Task): string => {
  // 优先使用本地 /storage 路径，拼接 /api/v1/assets/ 前缀供静态服务与缓存
  if (task.local_poster_path) {
    const pathUnderStorage = task.local_poster_path.replace(/^\/storage\/?/, '/');

  const posterPath = task.poster_path;

  if (posterPath.startsWith('http')) return posterPath;

};

## 技术细节

### 海报路径优先级

1. **local_poster_path**（最高优先级）
   - 本地缓存的海报文件
   - 路径格式：`/storage/posters/movie_123.jpg`
   - 前端拼接为：`/api/v1/assets/posters/movie_123.jpg`

2. **poster_path**（次优先级）
   - TMDB 返回的海报路径
   - 路径格式：`/abc123.jpg`
   - 前端拼接为：`https://image.tmdb.org/t/p/w500/abc123.jpg`

3. **placeholder**（兜底）
   - 占位图：`/placeholder-poster.jpg`

### 重试机制

1. 用户点击重试按钮
2. 前端调用 `POST /tasks/{id}/retry`
3. 后端将任务状态重置为 `pending`
4. 扫描引擎在下次运行时重新处理该任务
5. 前端刷新任务列表，显示最新状态
