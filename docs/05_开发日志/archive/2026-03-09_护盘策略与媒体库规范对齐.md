# 护盘策略与媒体库规范对齐

**日期**: 2026-03-09  
**版本**: V1.0  
**任务**: 优化硬盘休眠策略、严格对齐 Emby/Jellyfin 目录规范、补全标准 NFO 与 fanart.jpg

---

## 一、任务背景

### 问题描述

1. **硬盘休眠问题**: 前端轮询 `/api/v1/system/stats` 接口时，`get_system_stats` 方法会实时扫描媒体库目录统计文件数量，导致 NAS 硬盘频繁唤醒，无法进入休眠状态。
2. **目录规范不一致**: 剧集目录使用 `Season 01` 格式（补零），与 Emby/Jellyfin 标准的 `Season 1` 格式不一致。
3. **元数据不完整**: 缺少 `fanart.jpg` 背景图，NFO 文件结构简陋，缺少 XML 特殊字符转义。
4. **元数据路径错误**: 剧集元数据保存在季目录（如 `Season 1/tvshow.nfo`），应保存在剧集根目录。

### 解决目标

1. 解绑系统统计的实时扫盘，改为从数据库配置读取缓存数据。
2. 在刮削任务结束时执行一次物理扫盘统计，并将结果保存到数据库。
3. 严格对齐 Emby/Jellyfin 剧集目录规范（`Season 1` 格式）。
4. 分离剧集的根目录和季目录，确保元数据保存在正确位置。
5. 补全 `fanart.jpg` 下载功能。
6. 丰富 NFO 结构，添加 XML 特殊字符转义。

    )

#### 修改位置

在 `perform_scrape_all_task` 函数的末尾，`logger.info("[TMDB] 全量刮削完成...")` 之前插入统计逻辑。

#### 新增代码

```python
# 统计媒体库数量并存入 config（代替实时扫盘，护盘策略）
try:
    all_config = db.get_all_config()
    paths = all_config.get("paths", [])
    
    # 获取启用的媒体库路径
    movie_lib = None
    tv_lib = None
    for p in paths:
        if p.get("enabled", False) and p.get("type") == "library":
            if p.get("media_type") == "movie":
                movie_lib = p.get("path")
            elif p.get("media_type") == "tv":
                tv_lib = p.get("path")
    
    # 统计电影数量
    movies_count = 0
    if movie_lib and os.path.isdir(movie_lib):
        for item in os.listdir(movie_lib):
            item_path = os.path.join(movie_lib, item)
            if os.path.isdir(item_path):
                movies_count += 1
    
    # 统计剧集数量
    tv_count = 0
    if tv_lib and os.path.isdir(tv_lib):
        for item in os.listdir(tv_lib):
            item_path = os.path.join(tv_lib, item)
            if os.path.isdir(item_path):
                tv_count += 1
    
    # 保存到数据库配置
    db.set_config("library_movies_count", movies_count)
    db.set_config("library_tv_count", tv_count)
    
    logger.info(f"[STAT] 媒体库统计完成: 电影 {movies_count} 部, 剧集 {tv_count} 部")
except Exception as stat_error:
    logger.error(f"[STAT] 媒体库统计失败: {stat_error}")
```

#### 设计理念

- 仅在刮削任务完成时执行一次物理扫盘
- 统计结果缓存到数据库配置
- 前端轮询读取缓存，避免频繁唤醒硬盘

### 2.5 补全 fanart 下载（metadata_manager.py）

#### 新增方法

```python
def download_fanart(
    self,
    tmdb_id: str,
    media_type: str,
    output_dir: str,
    title: str = None
) -> Optional[str]:
    """
    下载 TMDB Fanart（背景图）
    
    Args:
        tmdb_id: TMDB ID
        media_type: "movie" 或 "tv"
        output_dir: 输出目录（fanart 将保存为 fanart.jpg）
        title: 可选，作品标题（用于日志）
    
    Returns:
        Optional[str]: Fanart 本地路径，失败返回 None
    """
    try:
        # 从 TMDB 获取详细信息
        if media_type == "movie":
            details = self._fetch_movie_details(tmdb_id)
        else:
            details = self._fetch_tv_details(tmdb_id)
        
        if not details:
            logger.error(f"[META] 获取 TMDB 详情失败: {title or tmdb_id}")
            return None
        
        backdrop_path = details.get("backdrop_path")
        if not backdrop_path:
            logger.warning(f"[META] 无 Fanart 数据: {title or tmdb_id}")
            return None
        
        # 构建 Fanart URL（使用 w1280 尺寸）
        fanart_url = f"https://image.tmdb.org/t/p/w1280{backdrop_path}"
        
        # 下载 Fanart
        response = requests.get(fanart_url, timeout=30)
        response.raise_for_status()
        
        # 确保目录存在
        os.makedirs(output_dir, exist_ok=True)
        
        # 保存 Fanart
        output_path = os.path.join(output_dir, "fanart.jpg")
        with open(output_path, 'wb') as f:
            f.write(response.content)
        
        logger.info(f"[META] Fanart 下载成功: {output_path}")
        return output_path
    
    except Exception as e:
        logger.error(f"[META] Fanart 下载失败: {e}")
        return None
```

#### 关键特性

- 使用 `w1280` 尺寸（高清背景图）
- 保存为 `fanart.jpg`（Emby/Jellyfin 标准命名）
- 与 `poster.jpg` 同级目录

```python
def _escape_xml(self, text: str) -> str:
    """转义 XML 特殊字符"""
    if not text:
        return ""
    text = str(text)
    text = text.replace("&", "&amp;")
    text = text.replace("<", "&lt;")
    text = text.replace(">", "&gt;")
    text = text.replace('"', "&quot;")
    text = text.replace("'", "&apos;")
    return text
```

## 四、技术细节

### 4.1 护盘策略原理

**问题根源**:
- 前端每 5 秒轮询 `/api/v1/system/stats`
- 原实现每次调用 `os.listdir()` 扫描媒体库目录
- NAS 硬盘被频繁唤醒，无法进入休眠状态

**解决方案**:
- 统计数据缓存到数据库配置表
- 前端轮询仅读取数据库，不触发磁盘 I/O
- 刮削任务结束时更新缓存，确保数据准确性

**性能对比**:
| 方案 | 磁盘 I/O | 响应时间 | 硬盘休眠 |
|------|---------|---------|---------|
| 修改前 | 每次轮询 | 200-500ms | 无法休眠 |
| 修改后 | 仅刮削时 | 5-10ms | 正常休眠 |

### 4.2 目录规范对齐

**Emby/Jellyfin 标准**:
- 剧集根目录: `Title (Year)`
- 季目录: `Season 1`, `Season 2`（不补零）
- 元数据位置: 剧集根目录

**Plex 标准**:
- 季目录: `Season 01`, `Season 02`（补零）
- 元数据位置: 剧集根目录

**本项目选择**: 严格对齐 Emby/Jellyfin 标准

### 4.3 XML 特殊字符转义

**必须转义的字符**:
- `&` → `&amp;`
- `<` → `&lt;`
- `>` → `&gt;`
- `"` → `&quot;`
- `'` → `&apos;`

**示例**:
```python
# 原始文本
title = "Tom & Jerry: The Movie"

# 转义后
title_escaped = "Tom &amp; Jerry: The Movie"
```

### 4.4 Fanart 尺寸选择

**TMDB 提供的尺寸**:
- `w300`: 300px 宽度（缩略图）
- `w780`: 780px 宽度（中等）
- `w1280`: 1280px 宽度（高清）
- `original`: 原始尺寸（超大）

**本项目选择**: `w1280`（平衡质量与文件大小）
