## 修复方案

### 第一步: 加固数据库清理逻辑

**文件**: `backend/app/infra/database/db_manager.py`

**新增方法**:

1. **`clear_all_tasks(self) -> int`**
   - 功能: 清空所有任务记录并重置自增 ID 计数器
   - 核心逻辑:
     - 使用 `DELETE FROM tasks` 清空任务表
     - 执行 `DELETE FROM sqlite_sequence WHERE name='tasks'` 重置自增 ID
     - 确保重置后任务从 ID 1 开始
     - 必须在 `with self.db_lock:` 保护下执行，防止并发写入冲突
   - 返回值: 删除的记录数

2. **`delete_task(self, task_id: int) -> bool`**
   - 功能: 删除单个任务记录
   - 返回值: 是否删除成功

3. **`delete_tasks_by_ids(self, ids: List[int])`**
   - 功能: 批量删除任务记录
   - 参数: 任务 ID 列表

4. **`check_task_exists_by_path(self, file_path: str) -> bool`**
   - 功能: 检查指定路径的任务是否已存在
   - 用途: 扫描时避免重复入库

5. **`insert_task(self, task_data: Dict[str, Any])`**
   - 功能: 插入新任务记录
   - 参数: 任务数据字典

**关键代码片段**:

```python
def clear_all_tasks(self) -> int:
    """清空所有任务记录（核弹按钮）"""
    with self.db_lock:
        conn = sqlite3.connect(self.db_path)
        conn.execute("PRAGMA journal_mode=WAL")
        
        # 统计删除前的记录数
        cursor = conn.execute("SELECT COUNT(*) FROM tasks")
        count = cursor.fetchone()[0]
        
        # 清空任务表
        conn.execute("DELETE FROM tasks")
        
        # 重置自增 ID 计数器（关键步骤）
        conn.execute("DELETE FROM sqlite_sequence WHERE name='tasks'")
        
        conn.commit()
        conn.close()
        
        print(f"[OK] [DB] 已清空任务表，删除 {count} 条记录，自增 ID 已重置")
        
        return count
```

**安全保障**:
- 绝对不触碰 `paths` 配置表，只清理任务记录
- 使用线程锁保护，确保并发安全
- 采用 WAL 模式，提升并发性能

## 技术要点

### 1. SQLite 自增 ID 重置机制

SQLite 使用 `sqlite_sequence` 表来跟踪自增主键的当前值：

```sql
-- 查看自增计数器
SELECT * FROM sqlite_sequence WHERE name='tasks';

-- 重置自增计数器
DELETE FROM sqlite_sequence WHERE name='tasks';
```

**注意事项**:
- 必须先清空表数据，再删除 `sqlite_sequence` 记录
- 如果表中仍有数据，重置计数器可能导致主键冲突
- 重置后，下次插入的 ID 从 1 开始

### 2. 线程安全保障

使用 `threading.Lock()` 确保数据库操作的线程安全：

```python
with self.db_lock:
    # 数据库操作
    conn.execute("DELETE FROM tasks")
    conn.execute("DELETE FROM sqlite_sequence WHERE name='tasks'")
    conn.commit()
```

**优势**:
- 防止并发写入冲突
- 确保事务完整性
- 避免数据不一致

### 3. WAL 模式优化

启用 SQLite 的 WAL (Write-Ahead Logging) 模式：

```python
conn.execute("PRAGMA journal_mode=WAL")
```

**优势**:
- 提升并发读写性能
- 减少锁竞争
- 支持多读单写

### 4. 前端样式对比度优化

**修改前**:
- 背景: `bg-charcoal-lighter` (浅灰色)
- 文字: `text-white` (白色)
- 对比度: 不足，难以辨识

**修改后**:
- 背景: `bg-black/60` (暗色半透明)
- 文字: `text-white` (白色)
- 对比度: 显著提升，清晰可见
