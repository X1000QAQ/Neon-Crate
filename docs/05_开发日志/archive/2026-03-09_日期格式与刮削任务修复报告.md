## 问题描述

### 问题 1：全线 Invalid Date

- 日志面板和任务列表中的所有时间戳显示为 `Invalid Date`
- 根本原因：Python 日志格式使用逗号分隔毫秒（如 `2026-03-09 12:34:56,789`），而 JavaScript 的 `Date` 构造函数要求使用点号（ISO 8601 标准）

### 问题 2：刮削任务瞬间结束且无状态变化

- 点击全量刮削按钮后，任务瞬间结束
- 数据库中任务状态未更新
- 缺乏详细的匹配日志和错误反馈

### 第四步：前端日期解析兜底（utils.ts）

**文件**: `frontend/lib/utils.ts`

**修改位置**: `formatDate` 函数

**修改内容**:

```typescript
export function formatDate(dateString: string): string {
  if (!dateString || dateString.trim() === '') {
    return '刚才';
  }
  
  // 尝试标准解析
  let date = new Date(dateString);
  
  // 如果解析失败，尝试手动处理字符串
  if (isNaN(date.getTime())) {
    // 尝试替换空格为 T，逗号为点号（兼容后端可能的格式问题）
    const normalized = dateString.replace(' ', 'T').replace(',', '.');
    date = new Date(normalized);
    
    // 如果依然失败，返回兜底值
    if (isNaN(date.getTime())) {
      return '刚才';
    }
  }
  
  return date.toLocaleString('zh-CN', {
    year: 'numeric',
    month: '2-digit',
    day: '2-digit',
    hour: '2-digit',
    minute: '2-digit'
  });
}
```

**修复效果**:
- 增加强力解析：如果 `new Date()` 解析失败，尝试手动处理字符串
- 如果依然失败，显示 `刚才` 作为兜底值
- 确保前端永远不会显示 `Invalid Date`

```

  ↓

  ↓
.replace(" ", "T").replace(",", ".")
  ↓
2026-03-09T12:34:56.789
  ↓
前端接收（utils.ts）
  ↓
new Date("2026-03-09T12:34:56.789")
  ↓
正确解析 ✓
```
