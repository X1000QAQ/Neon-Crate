## 一、任务背景

### 1.1 问题诊断

系统在 AI 决策层注入过程中存在以下关键缺陷：

1. **配置接口缺失**: `DatabaseManager` 缺少 `get_agent_config` 方法，导致 AI Agent 无法集中提取 LLM 配置
2. **刮削逻辑不完整**: `perform_scrape_all_task` 中 AI 分析结果未按照 N8N V9.1 标准进行物理级去噪
3. **硬链接逻辑缺失**: TMDB 匹配成功后缺少归档预览日志，无法追踪文件搬运路径

### 1.2 N8N V9.1 核心智慧

参考 N8N 刮削器 V4.9.2 工作流，系统必须遵循以下铁律：

- **AI 不翻译原则**: AI 仅负责语义清洗，禁止自行翻译英文片名，将翻译权交还给 TMDB 数据库
- **物理级去噪**: 在发起 TMDB 搜索前，必须使用正则表达式物理剥离 query 中的 4 位数年份
- **硬链接透明化**: 每次匹配成功后必须输出预览日志，明确告知用户文件将被搬运到何处

## 二、核心修复内容

### 2.1 第一步：补完数据库配置接口

**文件**: `backend/app/infra/database/db_manager.py`

**新增方法**: `get_agent_config`

```python
def get_agent_config(self, key: str = None) -> Dict[str, Any]:
    """
    获取 AI Agent 运行所需的完整配置
    
    Args:
        key: 可选，指定获取某个配置项
    
    Returns:
        Dict: AI Agent 配置字典
    """
    settings = self.get_all_config().get("settings", {})
    
    agent_config = {
        "provider": settings.get("llm_provider", "cloud"),
        "cloud_url": settings.get("llm_cloud_url", ""),
        "cloud_key": settings.get("llm_cloud_key", ""),
        "cloud_model": settings.get("llm_cloud_model", ""),
        "local_url": settings.get("llm_local_url", ""),
        "local_key": settings.get("llm_local_key", ""),
        "local_model": settings.get("llm_local_model", ""),
        "ai_persona": settings.get("ai_persona", ""),
        "expert_archive_rules": settings.get("expert_archive_rules", ""),
        "master_router_rules": settings.get("master_router_rules", ""),
        "ai_name": settings.get("ai_name", "AI 影音大师")
    }
    
    # 如果指定了 key，返回对应值
    if key:
        return agent_config.get(key, "")
    
    return agent_config
```

**核心功能**:
- 集中提取 AI 运行所需的 11 个核心配置项
- 支持全量获取或单项查询
- 自动处理敏感密钥解密（通过 `get_all_config` 实现）

### 2.2 第二步：加固刮削编排层

**文件**: `backend/app/api/v1\endpoints\tasks.py`

**修改函数**: `perform_scrape_all_task`

#### 关键改进点

##### 1. AI 配置正确注入

```python
# 初始化刮削器和 AI Agent
scraper = TMDBAdapter(api_key=tmdb_api_key)
ai_agent = AIAgent(db)

logger.info("[AI] AI 决策层已激活，将对所有文件名进行智能提炼")
```

确保 `AIAgent` 能够通过 `db.get_agent_config()` 获取完整 LLM 配置。

##### 2. N8N V9.2 物理级去噪补丁

```python
# 物理级去噪：剥离 query 中残留的年份（N8N V9.2 核心补丁）
import re
refined_query_clean = re.sub(r'\b(19|20)\d{2}\b', '', refined_query).strip()
if refined_query_clean != refined_query:
    logger.info(f"[CLEAN] 物理去噪: '{refined_query}' -> '{refined_query_clean}'")
    refined_query = refined_query_clean
```

**解决问题**: 修复《人生复印件》(Click 2006) 等带年份后缀的片名匹配失败问题。

**原理**: AI 可能在 query 中保留年份，导致 TMDB 搜索关键词过载。通过正则表达式物理剥离 4 位数年份，确保搜索纯净度。

##### 3. 硬链接逻辑预埋

```python
# 硬链接逻辑预埋（第三步）
logger.info(f"[ORG] [PREVIEW] 准备执行硬链接: {file_path} -> /media/{'Movies' if refined_type == 'movie' else 'TV'}/{title} ({year})/{title} ({year}).mkv")
```

**功能**: 在 TMDB 匹配成功后，输出预览日志，明确告知用户文件将被搬运到何处。

**格式**: `[ORG] [PREVIEW] 准备执行硬链接: 源路径 -> 目标路径`

## 三、N8N 逻辑深度对齐

### 3.1 AI 分析流程对齐

| 阶段 | N8N V4.9.2 | AI Media Master V9.1 |
|------|-----------|---------------------|
| 物理扫描 | Pre-Processor (Scan ALL) | ScanEngine.scan_directory |
| AI 清洗 | AI Agent (System Message) | AIAgent.ai_identify_media |
| 物理去噪 | 正则剥离年份 | re.sub(r'\b(19|20)\d{2}\b', '', query) |
| TMDB 搜索 | OneClick_Media_Processor | TMDBAdapter.search_movie/tv |
| 硬链接 | ln 命令 | logger.info (预埋) |

## 四、技术细节与安全保障

### 4.1 编码安全

**强制规范**: 所有文件操作必须使用 `encoding="utf-8"`

**实现位置**:
- `db_manager.py`: 所有 `open()` 调用均指定 `encoding='utf-8'`
- `tasks.py`: 日志输出使用 UTF-8 编码

### 4.2 日志格式规范

**标签系统**:
- `[TMDB]`: TMDB API 相关操作
- `[AI]`: AI Agent 决策层操作
- `[CLEAN]`: 物理级去噪操作
- `[ORG]`: 归档与硬链接操作
- `[PREVIEW]`: 预览模式标记

**示例**:
```
[2026-03-09 14:30:15] [AI] AI 提炼结果: query='Dune Part Two', year='2024', type='movie'
[2026-03-09 14:30:16] [CLEAN] 物理去噪: 'Dune Part Two 2024' -> 'Dune Part Two'
[2026-03-09 14:30:17] [TMDB] 匹配成功: Dune Part Two (TMDB ID: 693134, IMDB ID: tt15239678)
[2026-03-09 14:30:17] [ORG] [PREVIEW] 准备执行硬链接: /downloads/ready_for_ai_movie/Dune.Part.Two.2024.mkv -> /media/Movies/Dune Part Two (2024)/Dune Part Two (2024).mkv
```

### 4.3 Emoji 禁令

**绝对红线**: 禁止在任何日志输出中使用 Emoji

**原因**: Windows PowerShell 默认使用 GBK 编码，Emoji 会导致 `UnicodeEncodeError`

**参考文档**: `docs/03_开发日志/Windows环境避坑指南.md`
