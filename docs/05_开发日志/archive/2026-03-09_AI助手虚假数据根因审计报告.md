```398:437:d:\SoftWare\test\ai-media-master\backend\app\services\ai\agent.py
    def _get_system_stats(self) -> Dict:
        """
        获取系统统计数据（真实数据源，绝对禁止硬编码）
        
        数据来源：
        1. 任务统计：从 SQLite 数据库实时查询
        2. 磁盘占用：使用 shutil.disk_usage 获取真实磁盘使用率
        
        Returns:
            Dict: 包含 total, success, scraped, archived, pending, failed, disk_usage_percent
        """
        try:
            # 第一步：从数据库获取真实任务统计
            all_tasks = self.db.get_all_data()
            total = len(all_tasks)
            success = len([t for t in all_tasks if t.get("status") == "success"])
            scraped = len([t for t in all_tasks if t.get("status") == "scraped"])
            archived = len([t for t in all_tasks if t.get("status") == "archived"])
            pending = len([t for t in all_tasks if t.get("status") == "pending"])
            failed = len([t for t in all_tasks if t.get("status") == "failed"])
            
            # 第二步：获取真实磁盘占用率（Windows/Linux 通用）
            import shutil
            try:
                # 使用根目录或当前工作目录作为磁盘检测点
                disk_stat = shutil.disk_usage("/")
                disk_usage_percent = round((disk_stat.used / disk_stat.total) * 100, 2)
            except Exception as disk_err:
                logger.warning(f"[AIAgent] 磁盘占用率获取失败，使用降级方案: {disk_err}")
                # Windows 降级方案：检测当前工作目录所在磁盘
                import os
                cwd = os.getcwd()
                disk_stat = shutil.disk_usage(cwd)
                disk_usage_percent = round((disk_stat.used / disk_stat.total) * 100, 2)
```

**审计结论**: ✅ **数据获取逻辑正确**，调用 `self.db.get_all_data()` 获取真实数据。

## 🎯 根因锁定

基于以上审计，**AI 助手虚假数据的根本原因**可能是以下三者之一或组合：

### 根因假设 A: `llm_client.call_llm()` 方法未正确传递 `system_prompt`

**可能性**: ⭐⭐⭐⭐⭐ (最高)

**证据**:
- `context_report` 组装逻辑正确
- 数据库查询正确
- 但最终 AI 回复仍然是虚假数据

**验证方法**: 必须审计 `llm_client.py` 的 `call_llm()` 方法，检查：
1. 参数顺序是否正确
2. 是否有覆盖 `system_prompt` 的逻辑
3. API 请求体是否正确构建

### 根因假设 B: LLM 模型忽略了 `system_prompt` 中的约束

**可能性**: ⭐⭐⭐ (中等)

**证据**:
- 某些 LLM 模型对 `system_prompt` 的遵守程度较低
- 尤其是在 `system_prompt` 过长时

**验证方法**: 在 `llm_client.py` 中添加日志，打印实际发送给 LLM 的完整请求体。

### 根因假设 C: 日志文件缺失导致上下文不完整

**可能性**: ⭐ (较低)

**证据**:
- 日志文件确实不存在
- 但核心统计数据仍然完整

**影响**: 次要因素，不是主要原因。
