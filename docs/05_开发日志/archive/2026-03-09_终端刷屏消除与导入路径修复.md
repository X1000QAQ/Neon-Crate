# 终端刷屏消除与导入路径修复

## 修复日期
2026-03-09

## 问题背景

### 问题一：终端刷屏干扰
前端日志面板通过轮询 `/api/v1/system/logs` 接口获取日志，每次请求都会触发后端打印调试信息：
```python
print(f"[DEBUG] [API] 网页后端正在尝试读取日志文件: {APP_LOG_PATH}")
```
该日志随着前端轮询频繁触发，严重干扰开发调试体验。

### 问题二：刮削器导入路径错误
`perform_scrape_all_task` 函数中使用了不存在的导入路径：
```python
from app.services.scraper import TMDBScraper
result = await scraper.search_and_match(...)
```

实际上：
- `TMDBScraper` 类不存在
- 正确的类名是 `TMDBAdapter`，位于 `app.services.metadata.adapters`
- `TMDBAdapter` 没有 `search_and_match` 方法，需要分别调用 `search_movie` 和 `search_tv`

## 修复方案

### 第一步：消除刷屏干扰

**文件**: `backend/app/api/v1/endpoints/system.py`

**修改内容**:
删除 `get_logs` 函数中的调试打印语句：
```python
# 删除前
print(f"[DEBUG] [API] 网页后端正在尝试读取日志文件: {APP_LOG_PATH}")

# 删除后
# 直接进入逻辑，不再打印
```

**理由**: 路径已经确认正确，该调试语句已完成历史使命，不再需要。

### 第二步：修复刮削导入路径

**文件**: `backend/app/api/v1/endpoints/tasks.py`

**修改内容**:

1. 修正导入语句：
```python
# 修改前
from app.services.scraper import TMDBScraper

# 修改后
from app.services.metadata.adapters import TMDBAdapter
```

2. 重构刮削逻辑：
```python
# 修改前
result = await scraper.search_and_match(
    query=clean_name or file_name,
    media_type=media_type,
    year=task.get("year")
)

if result and result.get("tmdb_id"):
    # 处理结果...

# 修改后
if media_type == "movie":
    results = scraper.search_movie(
        query=clean_name or file_name,
        year=task.get("year")
    )
else:
    results = scraper.search_tv(
        query=clean_name or file_name,
        year=task.get("year")
    )

if results and len(results) > 0:
    best_match = results[0]
    tmdb_id = best_match.get("id")
    title = best_match.get("title") or best_match.get("name")
    release_date = best_match.get("release_date") or best_match.get("first_air_date", "")
    year = release_date.split("-")[0] if release_date else None
    # 处理结果...
```

## 技术细节

### TMDBAdapter API 说明

**search_movie 方法**:
```python
def search_movie(self, query: str, year: Optional[str] = None) -> List[Dict]
```
返回电影搜索结果列表，每个结果包含：
- `id`: TMDB ID
- `title`: 电影标题
- `release_date`: 发行日期（YYYY-MM-DD）
- 其他元数据字段

**search_tv 方法**:
```python
def search_tv(self, query: str, year: Optional[str] = None) -> List[Dict]
```
返回剧集搜索结果列表，每个结果包含：
- `id`: TMDB ID
- `name`: 剧集名称
- `first_air_date`: 首播日期（YYYY-MM-DD）
- 其他元数据字段

### 匹配策略

采用"取第一个结果"策略：
```python
if results and len(results) > 0:
    best_match = results[0]
```

**理由**:
- TMDB API 返回结果已按相关度排序
- 第一个结果通常是最佳匹配
- 简化逻辑，避免复杂的相似度计算
