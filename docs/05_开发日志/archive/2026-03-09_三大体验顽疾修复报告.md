### 第一步：物理探针级扫盘统计

**文件**: `backend/app/api/v1/endpoints/tasks.py`

**问题**: 原有统计逻辑依赖 `type="library"` 和 `media_type` 字段，导致统计数始终为 0。

**修复方案**:
```python
# --- 真正的物理扫盘统计 ---
try:
    movie_count, tv_count = 0, 0
    paths = db.get_all_config().get("paths", [])
    for p in paths:
        if p.get("enabled"):
            folder = p.get("path")
            if os.path.exists(folder):
                # 统计第一层子文件夹数量（排除文件）
                items = [name for name in os.listdir(folder) if os.path.isdir(os.path.join(folder, name))]
                if p.get("type") == "movie":
                    movie_count += len(items)
                elif p.get("type") == "tv":
                    tv_count += len(items)
    db.set_config("library_movies_count", movie_count)
    db.set_config("library_tv_count", tv_count)
    logger.info(f"[ORG] 物理媒体库盘点完成: 电影 {movie_count} 部, 剧集 {tv_count} 部")
except Exception as stat_err:
    logger.error(f"[ORG] 物理盘点失败: {stat_err}")
```

**核心改进**:
- 直接使用 `type` 字段匹配（"movie" / "tv"）
- 物理扫描第一层子文件夹，精确统计媒体数量
- 在全量刮削完成后执行，确保数据实时更新

### 第二步：海报双写持久化机制

**文件**: `backend/app/services/metadata/metadata_manager.py`

**问题**: 海报仅保存在作品目录，TMDB 链接失效后出现裂图。

**修复方案**:
```python
# 海报双写持久化机制：复制到系统静态资源目录
import shutil
from pathlib import Path
system_poster_dir = Path(__file__).resolve().parent.parent.parent.parent / "data" / "posters"
system_poster_dir.mkdir(parents=True, exist_ok=True)
system_poster_path = system_poster_dir / f"{tmdb_id}.jpg"
shutil.copy2(output_path, system_poster_path)
logger.info(f"[META] 海报双写完成: {system_poster_path}")

# 返回相对路径供前端拼接
return f"{tmdb_id}.jpg"
```

**核心改进**:
- 海报保存到 `data/posters/{tmdb_id}.jpg`
- 返回相对路径而非绝对路径
- 配合后端静态资源路由 `/api/v1/assets`

### 第三步：强力补全剧集 IMDb

**文件**: `backend/app/api/v1/endpoints/tasks.py`

**问题**: 剧集的 IMDb ID 经常缺失，影响字幕匹配和外部链接。

**修复方案**:
```python
# 强力补全剧集 IMDb
if refined_type == "tv" and not imdb_id:
    try:
        import requests
        resp = requests.get(
            f"https://api.themoviedb.org/3/tv/{tmdb_id}/external_ids",
            params={"api_key": tmdb_api_key},
            timeout=10
        )
        if resp.status_code == 200:
            imdb_id = resp.json().get("imdb_id", "")
    except Exception as e:
        logger.warning(f"[TMDB] 获取剧集IMDb失败: {e}")
```

**核心改进**:
- 在常规详情接口失败后，强制调用 `external_ids` 接口
- 仅针对剧集类型（`refined_type == "tv"`）
- 确保 IMDb ID 完整性

### 第四步：前端海报渲染护航

**文件**: `frontend/components/media/MediaWall.tsx`

**问题**: 前端未优先使用本地海报路径，导致 TMDB 链接失效时裂图。

**修复方案**:
```typescript
const getPosterUrl = (task: Task): string => {
  // 优先使用本地海报路径
  if (task.local_poster_path) {
    return `/api/v1/assets/${task.local_poster_path}`;
  }
  const posterPath = task.poster_path;
  if (!posterPath) return '/placeholder-poster.jpg';
  if (posterPath.startsWith('http')) return posterPath;
  return `https://image.tmdb.org/t/p/w500${posterPath}`;
};
```

**核心改进**:
- 优先使用 `local_poster_path`（本地持久化海报）
- 回退到 TMDB CDN 链接
- 最终回退到占位图
