# RegexLab 状态绑定修复报告

## 问题描述

RegexLab.tsx 组件在执行重置操作后，虽然后端成功返回了 15 条工业级默认正则规则，但前端 Textarea 文本框中依然没有显示内容，处于"隐身"状态。

## 问题根因分析

### 核心问题

在 `handleReset` 函数中，状态更新逻辑存在时序问题：

```typescript
if (result.success) {
  alert('正则清洗规则已重置为工业级默认值（15条规则）');
  await loadRegexRules();  // 问题：alert 阻塞了用户交互，但状态更新在 alert 之后
}
```

### 问题分析

1. **Alert 阻塞机制**：`alert()` 是同步阻塞调用，会暂停 JavaScript 执行直到用户点击确认
2. **状态更新延迟**：虽然使用了 `await loadRegexRules()`，但用户在看到 alert 时，状态实际上还未更新到 UI
3. **用户体验断层**：用户点击确认后才触发状态更新，导致视觉上感觉"没有反应"

### 正确的执行顺序

应该先完成状态更新，再通知用户：

```typescript
if (result.success) {
  // 1. 先拉取最新配置
  const freshConfig = await api.getSettings();
  // 2. 立即更新 React 状态
  setRegexRules(freshConfig.settings.filename_clean_regex || '');
  // 3. 最后通知用户（此时 UI 已经更新）
  alert('正则清洗规则已重置为工业级默认值（15条规则）');
}
```

## 修复方案

### 修改文件

`frontend/components/settings/RegexLab.tsx`

### 修改内容

```typescript
const handleReset = async () => {
  if (!confirm('确定要重置正则清洗规则吗？这将覆盖您当前的所有自定义修改。')) {
    return;
  }

  setResetting(true);
  try {
    const result = await api.resetSettings('regex');
    
    if (result.success) {
      // 关键修复：先更新状态，再弹出提示
      const freshConfig = await api.getSettings();
      setRegexRules(freshConfig.settings.filename_clean_regex || '');
      alert('正则清洗规则已重置为工业级默认值（15条规则）');
    } else {
      alert(`重置失败: ${result.message || '未知错误'}`);
    }
  } catch (error) {
    console.error('Reset failed:', error);
    const errorMessage = error instanceof Error ? error.message : '未知错误';
    alert(`重置失败: ${errorMessage}`);
  } finally {
    setResetting(false);
  }
};
```

### 修复要点

1. **直接调用 API**：不再依赖 `loadRegexRules()` 函数，直接在 `handleReset` 中调用 `api.getSettings()`
2. **显式状态更新**：明确调用 `setRegexRules(freshConfig.settings.filename_clean_regex || '')`
3. **正确的时序**：确保状态更新在 alert 之前完成

## 技术细节

### React 状态更新机制

```typescript
// 状态定义
const [regexRules, setRegexRules] = useState('');

// Textarea 绑定
<textarea
  value={regexRules}  // 受控组件，完全由 state 控制
  onChange={(e) => setRegexRules(e.target.value)}
  ...
/>
```

### 数据流

```
用户点击重置
  ↓
调用 api.resetSettings('regex')
  ↓
后端重置配置文件
  ↓
调用 api.getSettings() 拉取最新配置
  ↓
setRegexRules(freshConfig.settings.filename_clean_regex)
  ↓
React 触发重新渲染
  ↓
Textarea 显示最新内容
  ↓
弹出 alert 通知用户
```
