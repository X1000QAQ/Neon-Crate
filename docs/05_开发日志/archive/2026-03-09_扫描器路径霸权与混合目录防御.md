## 问题背景

### 原始问题

物理扫描引擎在将新文件入库时，过于信任文件名的正则匹配结果，导致以下问题：

**典型案例**：
- 文件路径：`D:\Movies\Dune.Part.Two.2024.S01E01.mkv`
- 路径配置：该目录被明确配置为 `category: movie`
- 正则引擎判断：因文件名包含 `S01E01`，被误判为 `tv`
- 入库结果：**错误入库为剧集**

### 核心矛盾

1. **正则引擎的局限性**：仅基于文件名模式匹配，无法理解目录语义
2. **用户配置的权威性**：用户明确指定的目录分类应具有最高优先级
3. **混合目录的合理性**：下载目录可能同时包含电影和剧集，不应强制覆盖

---

## 解决方案：路径配置霸权机制

### 核心设计原则

**"路径霸权"**：当且仅当目录被用户明确配置为纯粹的 `movie` 或 `tv` 时，强制覆盖正则引擎的判断。

**"混合防御"**：对于 `download`、`library`、`mixed` 等类型的目录，保留正则引擎的智能判断。

### 实现逻辑

```python
# 1. 获取正则引擎（MediaCleaner）给出的初步猜测
is_tv_guess = file_info.get("is_tv", False)
task_type = "tv" if is_tv_guess else "movie"

# 2. 查找当前文件所属的路径配置
file_path_normalized = os.path.normpath(file_path).lower()
matched_path_config = None

for path_cfg in paths:
    if not path_cfg.get("enabled", False):
        continue
    cfg_path = os.path.normpath(path_cfg.get("path", "")).lower()
    if file_path_normalized.startswith(cfg_path):
        matched_path_config = path_cfg
        break

# 3. 路径霸权与混合防御逻辑
if matched_path_config:
    folder_category = str(matched_path_config.get("category") or "").strip().lower()
    
    # 只有当路径被明确配置为纯粹的 "movie" 或 "tv" 时，才强制覆盖正则结果
    if folder_category in ["movie", "tv"]:
        original_type = task_type
        task_type = folder_category
        if original_type != task_type:
            logger.info(f"[SCAN] [路径霸权] 文件 {file_info.get('file_name')} 原类型={original_type}, 路径强制={task_type}")

# 4. 如果因路径霸权被强制认定为 movie，必须清空剧集信息
season_val = file_info.get("season") if task_type == "tv" else None
episode_val = file_info.get("episode") if task_type == "tv" else None
```

## 决策表

| 路径配置 category | 正则判断 | 最终入库类型 | 是否触发霸权 | 说明 |
|------------------|---------|------------|------------|------|
| `movie` | `movie` | `movie` | ❌ | 一致，无需干预 |
| `movie` | `tv` | `movie` | ✅ | **霸权生效**，强制覆盖为 movie |
| `tv` | `movie` | `tv` | ✅ | **霸权生效**，强制覆盖为 tv |
| `tv` | `tv` | `tv` | ❌ | 一致，无需干预 |
| `download` | `movie` | `movie` | ❌ | 混合目录，保留正则判断 |
| `download` | `tv` | `tv` | ❌ | 混合目录，保留正则判断 |
| `library` | `movie` | `movie` | ❌ | 混合目录，保留正则判断 |
| `mixed` | `tv` | `tv` | ❌ | 混合目录，保留正则判断 |

## 防御机制

### 1. 剧集信息清空

当文件被路径霸权强制认定为 `movie` 时，必须清空可能被正则误提取的剧集信息：

```python
season_val = file_info.get("season") if task_type == "tv" else None
episode_val = file_info.get("episode") if task_type == "tv" else None
```

**防御场景**：
- 文件名：`Dune.Part.Two.2024.S01E01.mkv`
- 正则提取：`season=1, episode=1`
- 路径霸权：强制为 `movie`
- 最终入库：`type=movie, season=None, episode=None`

### 2. 路径匹配规范化

使用 `os.path.normpath()` 统一路径分隔符，避免 Windows/Linux 路径差异导致匹配失败：

```python
file_path_normalized = os.path.normpath(file_path).lower()
cfg_path = os.path.normpath(path_cfg.get("path", "")).lower()
```

### 3. 最长前缀匹配

当文件路径匹配多个配置时，采用**第一个匹配**的配置（建议用户配置时避免路径嵌套）。

## 配置建议

### 推荐配置结构

```json
{
  "paths": [
    {
      "id": 1,
      "type": "download",
      "path": "D:\\Downloads\\Mixed",
      "category": "download",
      "enabled": true
    },
    {
      "id": 2,
      "type": "library",
      "path": "E:\\Media\\Movies",
      "category": "movie",
      "enabled": true
    },
    {
      "id": 3,
      "type": "library",
      "path": "E:\\Media\\TV Shows",
      "category": "tv",
      "enabled": true
    }
  ]
}
```

### 配置原则

1. **下载目录**：使用 `category: download`，保留正则智能判断
2. **媒体库目录**：使用 `category: movie` 或 `category: tv`，启用路径霸权
3. **避免路径嵌套**：不要配置父子关系的路径（如 `D:\Media` 和 `D:\Media\Movies`）

- **额外开销**：每个文件增加约 0.1ms 的路径匹配时间
- **可接受性**：对于千级文件扫描，总开销 < 100ms
